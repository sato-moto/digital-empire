# 📔 大帝国憲章 (Digital Empire Root Directive)

このファイルは、本プロジェクト全体の最上位指針である。
Claude Codeは、すべての作業においてこの憲章を遵守し、帝国の秩序を維持せよ。

## 🏛 帝国構造と統治指針
帝国は以下の7つの専門領域（ディレクトリ）で構成される。各領域の詳細は、それぞれのディレクトリ直下にある `CLAUDE.md` を参照せよ。

1.  **admin/**: 帝国運営・タスク管理・事務全般。
2.  **blog/**: 情報発信・コンテンツ制作の拠点。
3.  **dev/**: 開発・技術検証・ツール作成の工房。
4.  **lectures/**: 学習記録・講義資料・知識の蓄積。
5.  **research/**: 調査・分析・市場動向の探求。
6.  **video/**: 動画企画・制作・メディア管理。
7.  **SHIFT AI/**: 教材制作の全工程（情報整理・アウトライン・スライド・台本）を担う制作拠点。
- **--archive/**: 過去の遺産および凍結されたプロジェクト。

## 📜 統治基本原則
- **自律的提案**: Claudeは受動的な作業に留まらず、各領域の目的達成のために最適な提案を積極的に行え。
- **領域の尊重**: 作業を行う際は、必ず該当するディレクトリへ `cd` 移動してから実行せよ。
- **知識の同期**: 新しい知見やルールが生まれた場合は、速やかに各ディレクトリの `CLAUDE.md` またはRootの本憲章を更新せよ。

## 🚀 GitHub 同期（バックアップ）ルール
帝国の知恵を保護するため、以下のプロセスを厳守せよ。
1.  **タイミング**: ブログ記事の完成、重要な調査の完了、開発のマイルストーン到達など、「一つの成果」が出た際に必ず実行。
2.  **確認フロー**: Claude側から「マスター、成果をGitHubにバックアップ（push）しますか？」と必ず確認せよ。
3.  **実行**: 許可が降りた場合、実施した内容を端的にまとめたコミットメッセージ（日本語）を作成し、`git add .` -> `git commit` -> `git push` を一気通貫で行え。

## 🔐 セキュリティ・プロトコル（MCP運用ガードレール）
- **機密ファイルの直接読み取り禁止**: `.git/config`, `wp-config.json`, `.env` 等の機密ファイルは、AIが直接読み取ってチャットに出力することを禁ずる。
- **MCP経由の操作原則**: 認証情報・外部API・DBへのアクセスが必要な操作は、直接実行せずMCPツールを介して行うこと。
- **外部通信は承認制**: `curl`, `wget`, `ssh` 等の外部通信コマンドは、実行前に必ずマスターの承認を得ること。
- **`ignorePaths` との連動**: `.claude/settings.json` の `ignorePaths` に列挙されたパスは、AIからの読み取りを物理的に遮断している。これを回避しようとする行為は禁止。

## 🛠 開発・運用コマンド
- **ビルド/実行**: 各ディレクトリの個別ルールに従う。
- **全土検品**: `find . -maxdepth 2 -name "CLAUDE.md"` で各領域の憲法が健全か確認せよ。
