# 品質チェック

## Phase 5: セルフレビュー

プロンプト完成後、以下の観点でセルフチェック。問題があれば修正してから提出。

| # | チェック項目 |
|---|------------|
| 1 | ワクワク感: 見た瞬間「行きたい！読みたい！」と思えるか |
| 2 | コピーの質: 企画案の直訳ではなくベネフィット訴求になっているか |
| 3 | 視線誘導: 読む順番が自然で迷わないか |
| 4 | 情報階層: 主役が一目で分かるか |
| 5 | 配色: 3色以内。アクセント文字だけ読んで意味が通るか |
| 6 | タイポグラフィ: サイズ強弱が極端か。スマホで読めるか |
| 7 | 情報正確性: 曜日・日程・名前に誤りがないか |
| 8 | 講座名一致: プロンプトの講座名がPhase 1で確定した講座名と完全一致しているか。訴求コピーに置き換わっていないか |
| 9 | プロンプト密度: デザイン指示を書きすぎていないか。フッターのリサーチ指示が活きる余地があるか |
| 10 | ターゲット明確性: 誰に向けた画像かがプロンプトから読み取れるか |

---

## Phase 7: 画像品質チェック

画像生成後、Read ツールで生成画像を確認し、以下の品質問題がないかチェックする。

| # | チェック項目 | 判定基準 |
|---|-------------|---------|
| 1 | 重複テキスト | 同じ文章が画像内に2箇所以上表示されている |
| 2 | 背景異常 | 背景が黒一色、または透過して不自然になっている |
| 3 | 不要テキスト | プロンプトの指示文言（テキストルール、配置ルール等）が画像内に表示されている |

- 問題なし → Phase 8（テキスト検証・修正）へ
- 問題あり → `--regen` フラグ付きで再生成（1回のみ）。再生成後も問題がある場合は Phase 8 で対応

## NGパターン

| やりがち | 何が起きるか |
|---------|------------|
| 企画テキストそのまま | ダサい。ベネフィットに変換する |
| 抽象的なコピー | 何が学べるか伝わらない。「内容を知らない人が説明できるか」で判定する |
| AIっぽいリアル人物 | 不気味の谷。スタイルを明示指定する |
| テキストだけ | ワクワクしない。必ずビジュアル要素を入れる |
| 全部同じ文字サイズ | 平坦。強弱を極端につける |
| バッジ・タグだらけ | ゴチャゴチャ。最小限に |
| 色が4色以上 | まとまらない。3色以内 |
| デザイン指示の書きすぎ | リサーチ指示が死ぬ。「何を」だけ書いて「どう作るか」は任せる |
| ターゲットが不明確 | 誰に向けた画像か伝わらないと、リサーチの方向が定まらない |
| 講座名を訴求コピーに置き換える | 講座名が消え、何の講座か分からなくなる。訴求はサブタイトルの役割。講座名のまま変更禁止 |
