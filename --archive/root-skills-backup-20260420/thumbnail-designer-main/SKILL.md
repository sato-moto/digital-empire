# Thumbnail Architect

アウトライン → プロの広告設計思考 → Nano Banana Pro プロンプト → 画像生成 → テキスト検証・修正。
テンプレートは使わない。毎回ゼロベースで「なぜこの設計か」を言語化しながら最適解を導く。

**ゴール: 0.5秒で目が止まり、次の0.5秒で「参加したい！」と思わせるサムネイル**

## Phase 1: 企画分析

### 入力ファイルの種別判定

入力ファイル名で種別を判定し、対応する抽出ルールを適用する。

| パターン | 種別 | 抽出ルール |
|----------|------|------------|
| `outline-final.md` | 教養講座アウトライン | 下記「教養講座の場合」を適用 |
| `practice-*.md` / `practice-*-final.md` | 実務講座 収録指示書 | 下記「実務講座の場合」を適用 |

※ 判定できない場合はユーザーに確認する。

### 教養講座（outline-final.md）の場合

アウトライン（outline-final.md）から以下を抽出する。

| 項目 | 内容 |
|------|------|
| 狙い | 何を伝えたいか、ユーザーに何をさせたいか |
| ターゲット | 誰に向けているか（レベル・属性・課題） |
| 用途 | オンライン講座 / eラーニング / コースページ |
| 講座名 | サムネイルで最も大きく目立つメインテキスト |
| 強調ポイント | 一番伝えたい1メッセージ |
| 登壇者/人物 | 名前・肩書き（必要な場合）。肩書きは以下のいずれかを使う。①数字・実績ベース（「受講生3,000名」「法人研修のべ200社」等）②役職・称号（「株式会社〇〇 代表」「〇〇大学 教授」等）。「〇〇のプロ」「〇〇専門家」など誰でも名乗れる表現は避ける。提示された肩書きが弱いと判断した場合は、他に使える実績や役職がないかユーザーに確認する |
| 参照画像 | 人物指定画像（あれば）。ない場合はAI生成。ある場合は配置パターンを確認する: **①丸型で小さく配置**（登壇者の顔を添える用途）/ **②大きく配置**（人物をメインビジュアルにする用途） |
| 日程 | 日付・曜日・時間（曜日は `cal` コマンドで必ず検証） |
| サイズ | X宣伝用: 1920x1080 / コース講座: 1000x300 |
| ブランド | ロゴ・ブランドカラー（あれば） |

### 実務講座（practice md）の場合

収録指示書（practice-{N}.md）から以下のマッピングで抽出する。

| 項目 | 抽出元 | 抽出ロジック |
|------|--------|-------------|
| 講座名 | §0「講座番号 / タイトル」 | タイトル部分をそのまま使用 |
| 狙い | §1「収録の目的」 | 目的の要約（1文に凝縮） |
| ターゲット | §0「コース」+ 講座タイトル | コース名と講座内容からレベル・属性を推定 |
| 用途 | — | 固定値: 「オンライン講座（実務）」 |
| 強調ポイント | §1 目的 + §0 アウトプット | 受講者が得る最大の成果を1メッセージに |
| 使用ツール | §0「使用ツール」 | ビジュアル要素として活用可能（ツールアイコン等） |
| アウトプット | §0「アウトプット」 | 受講者ベネフィットとして訴求に使用 |
| ケース情報 | §2-3「ダミーデータ」 | 実務感を出すビジュアル要素として参考 |
| 登壇者/人物 | — | 確認フローでユーザーに確認 |
| 参照画像 | — | 確認フローでユーザーに確認 |
| 日程 | — | 確認フローでユーザーに確認 |
| サイズ | — | デフォルト: コース講座 1000x300（確認フローでユーザーに確認） |
| ブランド | — | 確認フローでユーザーに確認 |

※ 「—」の項目は収録指示書に含まれないため、確認フローでユーザーに聞く。
※ 抽出後は教養講座と同じ確認フロー（確認0〜4）に合流する。

### Phase 1 確認フロー

**必ず AskUserQuestion ツールのポップアップ選択肢でユーザーに確認する**。
テキスト返答ではなく、選択肢UIで確認すること。以下の項目を順に確認する。

**確認0: 講座名の確認**（最初に必ず実施。他の確認より先に行う）
アウトラインから抽出した講座名をポップアップで提示し、この講座名でよいか確認する。
- A. この講座名でOK
- B. 変更する（正しい講座名を入力）

**確認1: 人物の有無**（AskUserQuestion で選択肢を提示）
- A. 人物あり（参照画像を提供する）
- B. 人物あり（AI生成）
- C. 人物なし

**確認2: 人物ありの場合 — 配置パターン**（確認1で A or B を選んだ場合のみ）
- A. 丸型で小さく配置（登壇者の顔を添える用途）
- B. 大きく配置（人物をメインビジュアルにする用途）

**確認3: 参照画像（デザインスタイル）**
- A. 参照画像あり（パスを指定する）
- B. 参照画像なし（ゼロベースで設計）

**確認4: 抽出内容の最終確認**
- A. OK（Phase 2 へ進む）
- B. 修正あり（修正内容を入力）

※ 確認1〜3は依存関係がなければまとめて1回のポップアップで提示してよい。
※ 確認4は抽出テーブルを提示した上で必ず単独で確認する。
