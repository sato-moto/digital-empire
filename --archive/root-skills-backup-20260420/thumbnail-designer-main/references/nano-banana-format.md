# Nano Banana Pro プロンプト出力フォーマット

Phase 2（ビジュアルコンセプト思考）+ Phase 3（選択されたコピー）を統合し、
Nano Banana Pro に直接投入できるプロンプトを生成する。

**重要**: Nano Banana Pro は日本語を理解する。日本語で出力するのが最も再現性が高い。

---

## 核心原則

**「何を」書いて「どう作るか」は Nano Banana Pro に任せる**

- プロンプトには **目的・ターゲット・テキスト・トーン** を明確に書く
- デザインの詳細（レイアウト、HEXコード、フォント指定、装飾エフェクト等）は **書かない**
- フッターの「リサーチし、勝ちパターンを抽出して」が活きるのは、デザイン余地を残したとき
- 指示を書きすぎると、リサーチ指示が無視されて指定通りの（凡庸な）画像になる

---

## 構成（5セクション + フッター）

| # | セクション | 必須 | 役割 |
|---|-----------|------|------|
| 1 | 冒頭文 | 必須 | 何の画像か（目的 + 形式）を1文で |
| 2 | ターゲット | 必須 | 誰に向けているか（属性・課題・心理状態） |
| 3 | テキスト | 必須 | 講座名・サブタイトル（訴求コピー）・その他 |
| 4 | 人物 | 任意 | 人物を含む場合の方向性 |
| 5 | トーン | 必須 | 色の方向 + 雰囲気キーワード + サイズ |
| - | フッター | 必須 | リサーチ + CV最大化 + サイズ指定 |

---

## 出力テンプレート

```
[何の画像かを1文で。目的 + 形式を明確に]

# ターゲット
- [属性・レベル・課題を箇条書きで]
- [心理状態や動機があればここに]

# テキスト
- 講座名: [講座名]（最も大きく目立つメインテキスト）
- サブタイトル: [訴求コピー]（講座名を補強する1行）
- その他: [日程 / 登壇者名 等]

# ビジュアル
- 人物は原則使わない。講座の内容を象徴する抽象的・概念的なビジュアルで表現する
- 例: データが収束するイメージ、ツールのUI風グラフィック、インフォグラフィック風の図解など
- 人物が必要な場合のみ: [「このサムネにふさわしい〇〇を生成」+ 最低限の属性]

# トーン
- [色の方向を色名レベルで。HEXコード不要]
- [雰囲気キーワード（カンマ区切り）]

# サイズ
- [16:9 / 1:1 / 9:16]

---
2026年現在の日本市場で同ジャンルの広告・サムネイルの最新トレンドを反映し、現在バズっている・伸びているデザインの勝ちパターンを抽出して、[媒体名]広告を作って。商品のCV最大化を狙う広告画像を生成して。[サイズ指定]で作って。煽り系・クリック誘発型の低品質なバナー広告のスタイルは避け、洗練されたプロフェッショナルなデザインで今すぐ画像を生成して。
```

## 各セクションの書き方ガイド

### 冒頭文

- 1文で「何の画像か」を伝える
- 良い例: `G検定対策のオンライン合宿（1日完結）を告知するサムネイル画像を生成してください。`
- 悪い例: `サムネイル画像を作ってください。`（何の？が抜けている）

### ターゲット

- **最も重要なセクション**。リサーチの方向を決める
- 属性だけでなく「課題」「心理状態」まで書くと精度が上がる
- 良い例: `周囲のAI活用に焦りを感じ始めている30代〜50代`
- 悪い例: `ビジネスパーソン`（広すぎて方向が定まらない）

### テキスト

- 講座名 = そのまま入れる（**.claude/rules/text-hierarchy.md** 参照）
- サブタイトル = Phase 3 で選ばれた訴求コピーをそのまま入れる。強調フレーズが指定されている場合は「『〇〇』をアクセントカラーで強調すること」を併記する
- 日程・登壇者名は「その他」にまとめて書く
- **位置指定は書かない**（Nano Banana Pro のリサーチに任せる）
- **プロンプトに含めない情報**:
  - 参加形式（「オンライン」「Zoom」「リアル」等）→ ターゲットへの訴求に不要
  - 主催者側の目的（「会員サイト告知」「リスト獲得」等）→ ターゲットへのメッセージではない

### 人物

- 入れる場合は「このサムネにふさわしい〇〇を生成」と書く
- 最低限の属性: 年代・性別・雰囲気（1〜2行で十分）
- 服装・ライティング・構図の詳細は書かない（リサーチに任せる）
- 人物なしの場合はセクションごと省略

### トーン

- **色は色名レベルで書く**（HEXコード不要）
- 良い例: `エメラルドグリーン × ネイビー × 白を基調`
- 悪い例: `#2ECC71（エメラルドグリーン）をアクセントに使用`
- 雰囲気キーワードを3〜5個。一文の補足があるとなお良い
- サイズもここで指定
- **視認性の確保**: テキストの可読性が確保できない場合は、背景全体に軽くブラーをかけること。強度はテキストが読める最低限にとどめる

## プロンプト品質チェック

| チェック | 基準 |
|---------|------|
| 冒頭文 | 何の画像かが1文で伝わるか |
| ターゲット | 属性 + 課題が具体的か。リサーチの方向が定まるか |
| サブタイトルの質 | 企画案の直訳ではなくベネフィット訴求になっているか |
| テキスト網羅性 | 講座名・サブタイトル・日程等が漏れていないか |
| 情報正確性 | 曜日・日程・名前に誤りがないか |
| デザイン余地 | レイアウト・色・フォントを書きすぎていないか |
| トーンの方向性 | 色名 + キーワードでイメージが伝わるか |
| フッター | 媒体名とサイズが正しく埋まっているか |

---

## 書きすぎ判定の目安

プロンプトが以下に該当したら削る:

| 書きすぎサイン | 対処 |
|--------------|------|
| HEXコードが入っている | 色名に置き換える |
| フォントの形状・ウェイトが指定されている | 削除（トーンのキーワードで十分） |
| レイアウトの具体的な配置（「左1/3に人物」等）がある | 削除 |
| 装飾・エフェクトの詳細がある | 削除 |
| ライティングの方向や色が指定されている | 削除 |
| サイズ比率（5:1等）がある | 削除 |
| プロンプト全体が20行を超えている | 削って15行以内に |

---

## 注意事項

- **日本語のまま出力してOK**。Nano Banana Pro は日本語を理解する
- テキスト描画は不完全な場合がある。生成後に Canva / Figma 等で追い込み仕上げする前提
- デザインの「正解」はプロンプトで指定するのではなく、Nano Banana Pro のリサーチに委ねる
- プロンプトの役割は「何を・誰に・どんな空気感で」を正確に伝えること
