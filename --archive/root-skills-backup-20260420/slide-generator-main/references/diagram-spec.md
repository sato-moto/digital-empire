# Diagram Integration

## 図解生成コマンド

```bash
NODE_PATH="$(npm root -g)" \
  node scripts/generate_diagrams.js <slides.json> <output_dir> \
  [--env <.env_path>] [--author <name>]
```

- slides.json の各コンテンツスライドに対して Gemini API で図解 PNG を生成
- 命名規則: `slide-{NNN}.png` (NNN = ゼロ埋め3桁、1-indexed)
- カバー/チャプタースライドはスキップ
- API キー: `GOOGLE_API_KEY` 環境変数 or `--env` で `.env` ファイル指定
- `--author`: キャラクタースライド(Tier 3)で使用する参照画像の選択に必要
- エラー時は該当スライドをスキップして続行
- **既存ファイルスキップ**: 出力先に `slide-{NNN}.png` が存在する場合は自動スキップ
- **逐次生成 (CONCURRENCY=1)**: レート制限対策のため1枚ずつ順次生成
- API レート制限対策: 15秒間隔 + 自動リトライ (最大5回、429時は累進待機)
- **白背景必須**: 図解は必ず白背景(#ffffff)のPNGで生成
- **解像度**: プロンプトでは 2400x840px を指定（2x）、PPTX上は 1200x420px 相当で配置

## ビジュアルティアシステム

各コンテンツスライドは3つのティアに分類され、ティアごとに異なるプロンプトで図解を生成する。

| ティア | 名称 | 割合 | プロンプト | 特徴 |
|--------|------|------|-----------|------|
| Tier 1 | Simple Flat | 80% | `buildFlatPrompt()` | シェイプ・矢印・ボックスのみ。人物・アイコン完全禁止 |
| Tier 2 | Diagram/Icon | 15% | `buildDiagramPrompt()` | 詳細インフォグラフィック。アイコン・人物1人まで |
| Tier 3 | Character | 5% | `buildCharacterPrompt()` | キャラクター参照画像をマルチモーダル入力 |

### ティア判定ロジック (`classifyVisualTier`)

1. JSONの `visual_type` フィールドに `flat`/`diagram`/`character` → そのまま使用
2. `visual_type` が空/null → 自動判定:
   - デフォルト: `flat`
   - キーワードヒットで `diagram` に昇格: `フロー`, `仕組み`, `アーキテクチャ`, `ツール連携`, `システム構成`, `API`, `プロセス全体`
   - `character` は自動判定しない（Excel F列で明示指定のみ）

## Tier 1: Simple Flat プロンプト仕様

最もシンプルなビジュアル。講座の80%を占める標準スタイル。

**許可要素:**
- 角丸ボックス、矢印、線、テキストラベル
- グルーピング用の背景色囲み

**禁止要素:**
- 人物イラスト（シルエット含む）
- 装飾アイコン（歯車・電球・雲など一切不可）
- 複雑な図形や装飾

**スタイル:**
- 完全フラットデザイン（影・グラデーション禁止）
- 視覚要素は3〜4個に厳守
- テキストラベルは15文字以内
- 配色は共通カラースキーム（後述）に従う

## Tier 2: Diagram/Icon プロンプト仕様

業務フローやAIの仕組みなど、より詳細な視覚表現が必要なスライド。

**許可要素:**
- Tier 1の全要素に加えて:
- シンプルなフラットアイコン（1図解に最大2個）
- 人物シルエット（1図解に最大1人、シンプルなフラットイラスト）

**スタイル:**
- インフォグラフィック品質
- デュアルコーディング（図形 + テキストラベル）
- サイズ階層（中心概念は大きく、補助要素は小さく）

## Tier 3: Character プロンプト仕様

キャラクターを使った感情的・ストーリー的なスライド。

**キャラクター参照画像:**

| 講師名(--author) | キャラクター | ファイル | 外見 |
|-------------------|-------------|---------|------|
| Shifuko | Shifko | `references/shifko.png` | オレンジのテックスーツ、ボブカット、ガジェット持ち |
| Shifuto | Shifto | `references/shifto.png` | 和装（白い着物）、ピンク髪、合掌ポーズ |

**API呼び出し:**
- Gemini APIにマルチモーダル入力（テキスト + 参照画像PNG）
- 参照画像はbase64エンコードしてinlineDataとして送信
- プロンプトで「この参照画像のキャラクターを図解に自然に配置」と指示

**スタイル:**
- キャラクターは参照画像のスタイルを維持
- 背景は白固定、フラットデザイン
- キャラクター + 吹き出しや簡潔な図解の組み合わせ
- テキストは最小限（キーワードのみ）

## PPTX 統合

`build_slides.js` に `--diagrams <dir>` を指定するとスライドに統合。
ティアに関わらず、同じ位置・サイズでPNGを配置する。

**図解配置仕様:**
- PNG 2400x840px（2x解像度）、白背景必須 (#ffffff)
- PPTX上の配置: x=40pt, y=64pt, w=656pt, h=224pt

## レイアウトパターン（11種 — Tier 1/2 共通）

パターンは `classifyStructure()` でbody本文をキーワード分析し、4構造に自動分類。
`assignPattern()` で型別カウンターにより順番に割当（同一パターンの連続を回避）。

| 構造タイプ | パターン |
|-----------|---------|
| 階層 | ロジックツリー / ピラミッド図 / ネストボックス |
| 手順 | ステップフロー / タイムライン / プロセスチェーン |
| 比較 | 左右パネル / カード並列 |
| 因果 | 因果チェーン / フィッシュボーン / Before→After |

## カラースキーム（全ティア共通、最大4色 + 白背景）

| 用途 | カラー |
|------|--------|
| メインカラー | `#A51E6D`（ボーダー、ヘッダー、強調要素） |
| 文字色 | `#333333` |
| 補助色1 | 薄グレー `#F5F5F5`（ボックス背景） |
| 補助色2 | ティール `#1ABC9C` またはスレートブルー `#5D6D7E`（差し色） |
| 背景 | `#ffffff`（白）— 暗い背景は禁止 |

**アイコン・イラストはカラフル可** (Tier 2/3): 上記4色のトーン違いや他の色も自由に使用可
