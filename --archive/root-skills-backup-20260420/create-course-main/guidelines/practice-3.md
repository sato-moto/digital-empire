## 1. 収録の目的
教養講座で学んだ3FW（As-Is/To-Be、ロジックツリー、5Why）を**実ケースで使う動き**を視聴者に見せる。特に**「自分で考える → Claudeで広げる → 自分で判断する」の順番**を、画面上の手の動きと判断のセリフで体感させる。

## 3. Phase構成（本編 = CSV側に詳細）

| Phase | 内容 | 対応スライド | 目安尺 |
|---|---|---|---|
| Phase 1 | Step1: As-Is/To-Be表をスプレッドシートで作る | 実践動画①（成約率のギャップ） | 2分 |
| Phase 2 | Step2: draw.ioで自作ロジックツリーを作る | 実践動画②（ツリーは3階層で） | 4分 |
| Phase 3 | Step3: Claudeにケースデータを渡し、抜け漏れを補完してもらう | 実践動画③（Claudeで補完） | 4分 |
| Phase 4 | Step4: 問題リストに統合する（採否判断） | 実践動画④（根拠で採否判断） | 3分 |

> **重要**: 各Phaseの冒頭で必ず「次に〇〇をします」と予告 → 操作 → 「これで〇〇ができました」と結果確認、の3拍子で進める。

### Phase 1 補足
- 数字をベタ打ちするのではなく、**To-Beの設定根拠（前年同期）を口頭で必ず言う**こと。視聴者が一番詰まるのは「To-Beってどう決めるの？」という点。

### Phase 2 補足
- 自作ツリーは**完璧でなくてよい**ことを必ず言葉で強調する。「この段階では抜けがあって当然です。あとでClaudeに補ってもらいます」と予告しておくと、Phase 3への接続がスムーズ。
- データから引ける枝（プロセスデータ・失注理由）を先に、推測の枝（組織体制）を後に伸ばす順番を守る。

### Phase 3 補足
- Claudeに渡す前に**「自分のツリーは見せずに、ケースデータだけを渡す」**ことを画面上で示す。これが「自分の思考をAIで上書きしない」という思想の核。
- Claudeの出力が出たら、**自分のツリーと並べて画面分割**して比較する動きを入れる。

### Phase 4 補足
- Claudeの出力をそのまま採用しない判断を見せることが本Phaseの最大の目的。「根拠データがない項目は影響度を下げる or 採用しない」という判断のセリフをはっきり入れる。
