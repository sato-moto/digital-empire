# 実務系 動画講義 設計ガイドライン

## 1. この動画講義が目指すもの

本ガイドラインで設計する動画講義は、**「ツールや知識を教える」ものではなく、「実務をAIで完遂する体験を届ける」もの**である。

受講者は、動画講義を通じて「自分の仕事でAIを使うのが当たり前」という状態に移行することをゴールとする。

---

## 2. 設計の前提

### 2-1. 実務を主語にする

動画講義のテーマは必ず「業務・タスク」を主語にする。ツール名・機能名を主語にしない。

> 良い例：「営業メールをAIで5分で仕上げる」

> NG例：「ChatGPTの使い方マスター」

### 2-2. ロールを与えて仕事をなぞらせる

受講者に具体的な「役割（ロール）」と「ミッション（状況）」を与え、その仕事を完遂するためにAIを使う、という体験を設計する。

> 例：「BtoB営業担当として、明後日の商談までに提案資料を仕上げる必要がある。だからAIで骨子を作る。」

ツールは「目的を達成するための手段」として登場させる。ツールの説明から始めない。

### 2-3. 1本の動画講義＝1つの実務テーマ

1本の動画講義で扱う実務テーマは1つに絞る。複数のテーマを1本の動画講義に詰め込まない。

> 良い例：「議事録作成に特化した1本の動画講義」

> NG例：「メール・議事録・資料・SNS・翻訳をまとめて学ぶ1本の動画講義」

---

## 3. 設計プロセス

### Step 1：動画講義ゴールの設計

動画講義の設計は必ず「ゴールの定義」から始める。コンテンツの内容から考え始めない。

#### ペルソナの定義

受講者を1人の具体的な人物として定義する。

| 項目 | 内容 |
| --- | --- |
| 職種・役職 | 例：中堅メーカーの営業担当・入社3年目 |
| 日常の業務 | 例：顧客訪問、提案資料作成、社内報告 |
| 現在の課題 | 例：メール・議事録・資料作成に時間を取られ、提案活動に集中できない |
| AIの利用経験 | 例：試したことはあるが、業務では使えていない |

#### 行動ゴールの設定

動画講義終了後に受講者が「できるようになること」を、観察可能な行動として定義する。

> 良い例：「取引先へのお礼メールの叩き台を、AIで3分以内に作れる」

> NG例：「AIの使い方を理解している」

#### Before → After の定義

| Before（受講前） | After（受講後） |
| --- | --- |
| メール1通に15〜20分かかる | 叩き台を3分で作り、5分で仕上げられる |
| 議事録作成を後回しにしてしまう | 会議直後にAIで即出力できる |

### Step 2：動画講義のアウトライン設計

動画講義の流れは以下の4パートで構成する。

#### Why → What → How → Wrap の型

| パート | 内容 | 目安の時間 |
| --- | --- | --- |
| Why（なぜ） | この動画でやる実務・状況を提示する | 全体の15% |
| What（何を） | この動画でやること・使うツールを提示する | 全体の10% |
| How（どうやって） | 操作手順を実演する（画面録画） | 全体の60% |
| Wrap（まとめ） | この動画でできるようになったことを確認する | 全体の15% |

#### 各パートの詳細

**Why パート**

- Why の答えは「その実務をやるため」がほとんどの場合の正解。難しく考えない。
- 「この動画では〇〇（実務）をやります」と宣言するだけでよい。

**What パート**

- この動画で使うツール・機能を1〜2個に限定して提示する。
- 「この動画が終わったら何ができるか」を先に宣言する。
- 操作の全体フローをステップで見せる（地図を渡す）。

**How パート**

- 画面録画で実際の操作を見せる。
- 1ステップずつ、受講者が追いつけるペースで進める。
- サンプルプロンプト・入力例は必ず提供する。
- 「なぜこの操作をするのか」の理由を都度説明する。

**Wrap パート**

- 「この動画でできるようになったこと」を1〜2文で言語化する。
- 次の動画への橋渡しを行う（「次はこの課題を解決します」）。

## 4. コース文脈別の設計例

このガイドラインは、以下の2つのコース文脈に応用できる。どちらの文脈でも設計原則は共通である。異なるのは、受講者が持ち込む動機と実務テーマの性質だけである。

### コース文脈①：副業コース

受講者は「AIを使って副業収入を得たい」という動機を持っている。実務テーマは副業として実際に発生する仕事から設定する。ツールの習得が目的ではなく、副業として成果物を納品・完結させることが目的になる。

設計の考え方

- ロールは「副業として活動する自分」
- ミッションは「クライアントから依頼された仕事を完成させる」
- 到達状態は「実際に納品・提出できる状態」

実務テーマの例：ライティング

| 項目 | 内容 |
| --- | --- |
| ロール | 副業ライターとして活動を始めた人 |
| ミッション | 不動産投資に関する記事を書き、読者に面談応募をしてもらう |
| Before | 記事の構成が作れない。何を書けばいいか分からない |
| After | AIを使って構成・本文・見出しを作り、クライアントに納品できる |

### コース文脈②：業務効率化コース

受講者は「日常業務をAIで効率化したい」という動機を持っている。実務テーマは受講者が現在の仕事で実際に行っている業務から設定する。ツールの習得が目的ではなく、今の仕事が速く・楽になることが目的になる。

設計の考え方

- ロールは「現職の自分（職種・役職）」
- ミッションは「今週・今月の業務を完成させる」
- 到達状態は「実際の業務に使える状態」

実務テーマの例：資料作成

| 項目 | 内容 |
| --- | --- |
| ロール | SaaS企業の営業担当 |
| ミッション | 自社SaaSサービスの商談資料を作成し、来週の商談に臨む |
| Before | 資料の構成を考えるのに時間がかかる。毎回ゼロから作っている |
| After | AIで構成・文章・骨子を作り、短時間で商談資料を完成させられる |

## 5. サンプルアウトライン

以下に、2つのコース文脈に対応したサンプルアウトラインを示す。各パートにはミニ台本（セリフ例）を付記しており、台本作成時の参考として活用できる。

---

### サンプル①：副業コース × ライティング

**動画講義タイトル**：「不動産投資記事をAIで書いて、面談応募を獲得する」

**ロール**：副業ライターとして活動を始めた人。クライアントから「不動産投資に関心のある読者に向けた記事を書いてほしい」という依頼を受けた。

**ミッション**：不動産投資に関する記事を1本完成させ、読者に面談応募ボタンを押してもらう。

**Before / After**

| Before（受講前） | After（受講後） |
| --- | --- |
| 記事の構成が作れず、何を書けばいいか分からない | AIで構成・本文・見出しを作り、クライアントに納品できる |
| 書き始めるまでに時間がかかる | 最初のプロンプトで骨子が出て、すぐ書き始められる |
| 「読者に刺さる言葉」が思いつかない | 読者の悩みを起点にした言葉をAIで複数パターン出せる |

---

**Why（なぜこの動画を見るのか）**

副業ライターとして記事を受注した。しかし、ゼロから書こうとすると構成が決まらず、何時間も止まってしまう。この動画では、AIを使って「不動産投資記事」を1本完成させる。

> 💬 ミニ台本

> 「今回は、副業ライターとしてクライアントから依頼を受けた、という設定でやっていきます。テーマは不動産投資。読者に記事を読んでもらって、面談に応募してもらうのがゴールです。

> ライティングって、何を書けばいいか分からないとか、構成が決まらないとか、そういうところで時間を使いすぎてしまうことが多いんですよね。

> この動画では、そこをChatGPTで一気に解決していきます。記事1本、一緒に完成させましょう。」

---

**What（この動画でやること）**

- 読者の悩みを起点にした構成をAIで設計する
- 各セクションの本文をAIで生成し、自分の言葉に仕上げる
- 読者をCTAに誘導する締めくくりをAIで作る

使うツール：ChatGPT

> 💬 ミニ台本

> 「やることは大きく3つです。まず読者がどんな悩みを持っているかをAIに整理してもらいます。次に、その悩みをもとに記事の構成を作ります。そして本文を書いて、最後に読者を面談に誘導するCTAまで仕上げます。

> 使うのはChatGPTだけです。では実際にやっていきましょう。」

---

**How（実際の操作手順）**

Step1：読者の悩みを言語化する

プロンプト例：

```
不動産投資に興味はあるが、まだ始めていない30代会社員が抱える
「不安・疑問・知りたいこと」を10個挙げてください。
```

> 💬 ミニ台本

> 「まず最初にやるのは、読者の悩みを言語化することです。記事を書くとき、いきなり本文から書き始めると、誰に向けて書いているのかがブレてしまいます。

> なので最初に、読者がどんなことで悩んでいるかをChatGPTに出してもらいます。このプロンプトを入力してみてください。

> ——（プロンプト入力・出力を見せる）——

> こんな感じで10個出てきましたね。これが記事全体の設計図になります。この中から、記事で扱うテーマに合うものを選んでいきます。」

Step2：記事の構成を設計する

プロンプト例：

```
以下の読者の悩みをもとに、「不動産投資を始めるべきか迷っている30代会社員」に向けた
記事の構成を作ってください。
ゴール：読者に無料面談への応募ボタンを押してもらう。
構成は見出しH2×4本、各H2の下にH3×2〜3本で作成してください。
[読者の悩みリストをここに貼る]
```

> 💬 ミニ台本

> 「次に、さっき出てきた悩みリストをそのままコピーして、このプロンプトに貼り付けます。ゴールとして『面談応募を促す』というのを明示しているのがポイントです。ゴールを伝えることで、AIが記事の流れをゴール逆算で設計してくれます。

> ——（プロンプト入力・出力を見せる）——

> H2が4本、それぞれにH3が2〜3本という構成が出てきました。これで記事の骨格が完成です。あとはこの骨格に沿って本文を埋めていくだけです。」

Step3：本文を生成する

プロンプト例：

```
以下の見出しに対して、本文を600字で書いてください。
読者は「不動産投資に興味はあるが、リスクが怖くて踏み出せない30代会社員」です。
専門用語は使わず、具体的な数字や事例を交えて書いてください。
[見出しをここに貼る]
```

> 💬 ミニ台本

> 「本文はセクションごとに生成していきます。一気に全部書かせようとすると精度が落ちるので、見出し1本ずつ依頼するのがコツです。

> 読者像と文体の条件を毎回入れているのがポイントです。『専門用語を使わない』『数字や事例を入れる』という条件を入れることで、読みやすくて信頼感のある文章になります。

> ——（プロンプト入力・出力を見せる）——

> こんな感じで本文が出てきます。そのまま使うのではなく、次のステップで自分の言葉に仕上げていきます。」

Step4：CTAの文章を作る

プロンプト例：

```
記事の最後に、読者に無料面談への応募を促すCTAを3パターン書いてください。
読者の不安を和らげながら、行動を後押しするトーンにしてください。
```

> 💬 ミニ台本

> 「記事の最後、CTAは読者が行動するかどうかの分かれ目です。ここを雑に書くと、せっかく記事を読んでもらっても応募につながりません。

> AIに3パターン出してもらって、一番しっくりくるものを選ぶか、いいとこ取りして組み合わせましょう

> ——（プロンプト入力・出力を見せる）——

> 3パターン出てきました。読者の不安を和らげるトーンになっているのが分かると思います。この中から選んで、少し自分の言葉で調整してください。」

Step5：全体を見直し、自分の言葉に仕上げる

- AIの出力をそのまま使わず、1文ずつ読んで違和感のある箇所を修正する
- 自分の体験・意見を1〜2箇所加えてオリジナリティを出す

> 💬 ミニ台本

> 「最後のステップは、全体を通して読み直すことです。AIの文章は整っていますが、どこか無難にまとまりすぎていることがあります。

> 自分の体験や意見を1〜2箇所入れるだけで、一気に『書いた人の顔が見える記事』になります。クライアントからの信頼も上がりますし、読者にも響きやすくなります。

> これで記事1本の完成です。お疲れ様でした。」

---

**Wrap（この動画でできるようになったこと）**

AIを使って不動産投資記事の構成・本文・CTAを作り、クライアントに納品できる状態になった。「書けない」ではなく「AIと一緒に書く」が当たり前になった。

> 💬 ミニ台本

> 「今回の動画では、読者の悩みの言語化から始まって、構成・本文・CTAまで、記事1本をChatGPTで完成させる流れをやりました。

> 最初は難しく感じるかもしれませんが、プロンプトのパターンさえ覚えてしまえば、どんなテーマの記事でも同じ流れで作れます。

> 次の動画では〇〇について扱います。引き続きよろしくお願いします。」

### サンプル②：業務効率化コース × スライド資料作成

**動画講義タイトル**：「自社SaaSの商談資料をAIで作って、来週の商談に臨む」

**ロール**：SaaS企業の営業担当。来週、新規顧客との商談が入った。自社サービスの提案資料を用意しなければならないが、毎回ゼロから作っていて時間がかかっている。

**ミッション**：自社SaaSサービスの商談資料を完成させ、来週の商談で使える状態にする。

**Before / After**

| Before（受講前） | After（受講後） |
| --- | --- |
| 資料の構成を考えるのに半日かかる | AIで構成案を30分で作れる |
| 毎回ゼロから文章を書いている | 各スライドの文章をAIで一気に生成できる |
| 「刺さる言葉」が思いつかず、機能説明になりがち | 顧客視点の訴求文を複数パターンAIで出せる |

---

**Why（なぜこの動画を見るのか）**

来週の商談に向けて提案資料を作る必要がある。しかし、構成を考えるだけで時間がなくなり、中身が薄い資料になってしまう。この動画では、AIを使って商談資料を1本完成させる。

> 💬 ミニ台本

> 「今回は、来週に新規顧客との商談が入っている、という設定でやっていきます。自社のSaaSサービスを提案する資料を作らないといけない。でも毎回ゼロから作っていて、構成を考えるだけで半日かかってしまう。そういう状況です。

> この動画では、ChatGPTを使って商談資料を一気に仕上げる流れをやります。構成から各スライドの文章まで、全部AIと一緒に作っていきましょう。」

---

**What（この動画でやること）**

- 顧客の課題を起点にした資料構成をAIで設計する
- 各スライドの文章をAIで生成し、商談で使える言葉に仕上げる
- 「機能説明」ではなく「顧客ベネフィット」の訴求文をAIで作る

使うツール：ChatGPT

> 💬 ミニ台本

> 「やることは3つです。まず顧客がどんな課題を持っているかをAIに整理してもらいます。次に、その課題をもとに資料の構成を作ります。そして各スライドの文章と、顧客に刺さる訴求文を生成します。

> 資料作成でよくある失敗が、自社の機能説明になってしまうことです。今回は最初から顧客の課題を起点にして設計するので、そこを防げます。では始めましょう。」

---

**How（実際の操作手順）**

Step1：顧客の課題を言語化する

プロンプト例：

```
中堅製造業（従業員300名）の営業部長が、
営業管理ツール（SaaS）の導入を検討している場面で抱える
「課題・懸念・知りたいこと」を10個挙げてください。
```

> 💬 ミニ台本

> 「最初にやるのは、顧客の課題を言語化することです。商談資料って、自社のサービス説明から始めてしまいがちなんですが、それだと顧客には刺さりません。

> まず顧客が何に困っているかを整理してから、そこに対して自社サービスがどう解決するかを見せる。この順番が大事です。

> ——（プロンプト入力・出力を見せる）——

> 10個出てきました。この中から今回の顧客に当てはまりそうなものを3〜4個選んで、次のステップに使います。」

Step2：資料の構成を設計する

プロンプト例：

```
以下の顧客課題をもとに、BtoB SaaSの商談資料の構成を作ってください。
顧客：中堅製造業の営業部長
ゴール：次回の商談でトライアル申し込みを獲得する
スライド枚数：8〜10枚
構成は「課題提起→解決策の提示→自社サービスの説明→導入事例→次のステップ」の流れで作成してください。
[顧客課題リストをここに貼る]
```

> 💬 ミニ台本

> 「さっき選んだ課題をここに貼り付けます。ゴールとして『トライアル申し込みを獲得する』と明示しているのがポイントです。

> 構成の流れも指定しています。課題提起から始まって、解決策、自社サービス、事例、次のステップという順番です。これは商談資料の王道の流れなので、このまま使えます。

> ——（プロンプト入力・出力を見せる）——

> スライドのタイトルが8枚分出てきました。この構成をベースに、次は各スライドの文章を作っていきます。」

Step3：各スライドの文章を生成する

プロンプト例：

```
以下のスライドタイトルに対して、スライドに載せる本文を3〜4行で書いてください。
読者は「営業管理ツールの導入を検討している製造業の営業部長」です。
機能の説明ではなく、顧客が得られる効果・メリットを中心に書いてください。
[スライドタイトルをここに貼る]
```

> 💬 ミニ台本

> 「スライドの文章は、タイトルを1枚ずつ渡して生成していきます。ここで大事なのが、『機能の説明ではなく、顧客が得られる効果を書いてください』という指示を入れることです。

> これを入れないと、どうしても機能説明になってしまいます。顧客は機能を知りたいのではなく、自分の課題が解決するかどうかを知りたいので、そこを意識した文章にします。

> ——（プロンプト入力・出力を見せる）——

> こんな感じで各スライドの文章が出てきます。そのまま使うのではなく、自社の実際の数字や事例を入れて仕上げてください。」

Step4：訴求文を複数パターン出す

プロンプト例：

```
以下の機能説明を、顧客視点のベネフィット訴求文に書き換えてください。
3パターン出してください。
機能説明：「リアルタイムで営業進捗を可視化できます」
顧客：営業部長（KPIは受注件数と商談数）
```

> 💬 ミニ台本

## 6. NG例と改善基準

### NG例①：機能を主語にした設計

問題のある状態

動画のタイトルや構成が、機能や操作名を中心に組み立てられている。

> NG例：「ロゴ生成機能」「背景削除機能」「テキスト編集機能」

この場合、動画講義は機能の紹介になりやすい。受講者は操作を知ることはできるが、「何のために使うのか」が曖昧になる。

改善基準

動画の主語を「やりたいこと」に置き換える。

> 改善例：「ロゴを作ってみる」「ポスターに使うために写真を整える」

機能は目的を達成するための手段として登場させる。タイトルを見たときに「できるようになること」が分かる状態にする。

### NG例②：1本の動画に複数の目的を詰め込んでいる

問題のある状態

1本の動画の中で複数の目的を同時に扱い、複数の機能を一気に実装している。

> NG例：ロゴ生成 → 背景削除 → 色調整 → モックアップ → テキスト編集 をすべて1本で扱う。

受講者は「何が重要なのか」「何を身につけたのか」が分からなくなる。

改善基準

1本の動画につき、明確な目的を1つに限定する。扱う機能は1〜2個までに抑える。動画の最後に「この動画でできるようになったこと」が1文で言える状態にする。

### NG例③：実務説明が中心になっている

問題のある状態

ツールの操作ではなく、業務知識・理論の説明が動画の大半を占めている。

> NG例：「営業提案書の書き方の基本」「議事録の正しいフォーマット」を長々と解説する。

受講者は「それ、AIと関係あるの？」と感じ、離脱しやすくなる。

改善基準

実務知識は「状況設定（Why）」として最小限に提示する。メインはAIを使った操作（How）に充てる。実務知識の解説は全体の20%以内に収める。

### NG例④：操作説明から始まっている

問題のある状態

動画の冒頭で操作説明から始まる。

> NG例：「背景削除の方法を説明します」

受講者は「なぜそれを学ぶのか」が分からないまま進むことになる。

改善基準

必ず最初に「どんな状況か」「何を実現したいのか」を提示する。操作は、その課題を解決するために登場させる。

### NG例⑤：技術が独立して存在している

問題のある状態

プロンプトの書き方や設定項目の説明などが、単体で解説されている。

> NG例：「良いプロンプトの条件は〜です」（文脈なし）

改善基準

技術は必ず具体的な目的の中で扱う。

> 改善例：「思ったようなロゴが出ない → プロンプトを具体化する」

技術は常に問題解決の文脈で登場させる。
