# モデル役割・実行マッピング

## メインClaude の役割一覧

メインClaude（Opus 4.6）がフェーズ/Stepごとに異なる役割を担う。外部モデル（Gemini/OpenAI）はアサインマネージャー経由で呼び出す。

| Phase | Step | 役割名 | 担当内容 |
|---|---|---|---|
| 全体 | 全Step | **講座プロデューサー** | 進行管理・ユーザー対話・プロンプト設計 |
| Phase 1 | Step 0: リサーチ生成 | **カリキュラムリサーチャー** | カリキュラム設計に必要な市場・競合・素材の調査 |
| Phase 1 | Step 0: 3版統合 | **リサーチインテグレーター** | 3モデルのリサーチ結果を4層構造に統合 |
| Phase 1 | Step 1: 構成提案+壁打ち | **カリキュラムデザイナー** | セット構成・Lv分け・講座配置の設計と壁打ち |
| Phase 2 | Step 3: 追加リサーチ生成 | **講座リサーチャー** | 個別講座に必要なWhy/What/How素材の深掘り調査 |
| Phase 2 | Step 3: 3版統合 | **リサーチインテグレーター** | （Phase 1と同じ役割） |
| Phase 2 | Step 4: アウトライン生成 | **アウトラインライター** | GL準拠のWhy/What/How/Wrapアウトライン起案 |
| Phase 2 | Step 5-A: 相互批評 | **相互レビュアー** | 他モデルのアウトラインを採点表で批評 |
| Phase 2 | Step 5-B: セルフレビュー | **内省レビュアー** | 自分のアウトラインを失敗前提・強み・修正案で検証 |
| Phase 2 | Step 5-C: マージ+構造lint | **アウトラインエディター** | ベース案選定+差分マージ+構造lint（編集長基準） |
| Phase 2 | Step 6: 51項目レビュー | **品質レビュアー** | 51項目チェックリストでの最終品質検査 |
| Phase 3 | Step 7: スライド構成 | **スライドデザイナー** | outline-finalからスライド構成への変換設計 |
| Phase 4 | Step 8-A: 台本設計 | **スクリプトライター** | スライド構成から講師の語り台本への変換設計 |

## モデル別役割（3モデル並列生成時）

### リサーチ生成時
- **Claude（メイン）**: 講座設計観点での情報変換重視（GL逆算）
- **Gemini（API）**: 探索的な情報収集・周辺領域の発見重視
- **OpenAI（API）**: 構造化・分類・整理・MECE性重視

### アウトライン生成時
- **Claude（メイン）**: 教育設計の一貫性重視 — 学習導線・メンタルモデル・章間接続・受講者心理の順序設計を最優先
- **Gemini（API）**: 追加視点・抜け漏れ検出重視 — 別角度からの論点追加・見落としがちな観点・探索的な広がりを最優先
- **OpenAI（API）**: MECE性・構造最適化・命名明瞭性重視 — 粒度の統一・重複排除・章タイトルの明確さ・構造の整理を最優先
