# 実演動画合成スキル（Video Composer）

操作画面キャプチャ・アバター動画・台本テキストを入力として、
スライドテンプレート上に合成した実演パート動画を生成するスキル。

## スキルルート

このスキルのルートディレクトリ（この SKILL.md が存在するディレクトリ）を `{skill_root}` と呼ぶ。

---

## レイアウト定数（確定値）

**根拠：MATH-M-01-A/ch2-final.mp4（マスター承認済み成果物）からのピクセル実測値。推定値ゼロ。**

### キャンバス
```
CANVAS = (1920, 1080)
TEMPLATE = "temp.png"  # slide-generator-main/assets/temp.png
```

### 操作画面
```
SCREEN_OVERLAY = (184, 56)    # ffmpeg overlay座標
SCREEN_SCALE   = (1552, 770)  # スケールサイズ（アスペクト比維持で算出）
```
- 操作画面の元素材をアスペクト比維持でSCREEN_SCALEにリサイズし、SCREEN_OVERLAYの位置に配置
- SCREEN_SCALEはコンテンツエリア高さ(770)基準で算出: `width = 770 * (元幅 / 元高さ)`
- 水平位置はコンテンツエリア内で中央配置: `x = 88 + (1744 - width) / 2`

### アバター
```
AVATAR_OVERLAY = (200, 865)   # ffmpeg overlay座標
AVATAR_SCALE   = (180, 180)   # スケールサイズ
```
- アバター動画（256×256）をAVATAR_SCALEにリサイズし、AVATAR_OVERLAYの位置に配置
- テンプレート下部バー内の白枠に収まる

### テロップ
```
TELOP_POS       = (1115, 955)  # ASS \pos座標
TELOP_FONT      = "Noto Sans JP"
TELOP_SIZE      = 60           # pt
TELOP_BOLD      = true
TELOP_COLOR     = "#303030"    # ASS形式: &H00303030
TELOP_OUTLINE   = 4            # px, 色: #FFFFFF
TELOP_SHADOW    = 2
TELOP_ALIGNMENT = 5            # 垂直中央・水平中央
TELOP_AREA      = (415, 865, 1400, 180)  # x, y, w, h（テロップ描画領域）
```

### ASS字幕スタイル定義（そのまま使用）
```
Style: Telop,Noto Sans JP,60,&H00303030,&H000000FF,&H00FFFFFF,&H80000000,-1,0,0,0,100,100,0,0,1,4,2,5,0,0,0,1
```

## 入力素材

| 素材 | 形式 | 説明 |
|------|------|------|
| 操作画面キャプチャ | mp4 | ブラウザでAIツールを操作した画面録画 |
| アバター動画 | mp4 × N本 | AIアバターのリップシンク付き映像+音声（シーンごとに1本） |
| 台本テキスト | テキスト or xlsx | シーンごとのナレーション全文 |
| OPアニメーション | mp4 | `{skill_root}/assets/opening.mp4` を自動使用。明示指定不要 |
| スライドテンプレート | png | `slide-generator-main/assets/temp.png` |

## ワークフロー

### Phase 1: 素材準備

1. **操作画面のクロップ**

   操作画面のブラウザUIは動画ごとに異なるため、固定値でクロップしない。
   毎回以下の手順で検出・除去する。

   **切り除く項目（上から順に検出）：**

   | # | 項目 | 検出方法 |
   |---|------|---------|
   | 1 | タブバー | 先頭フレームの上端からy方向スキャン。タブの背景色（灰色帯）の下端を検出 |
   | 2 | アドレスバー | タブバー下端の直下。URL文字列を含む帯の下端を検出 |
   | 3 | ブックマークバー | アドレスバー下端の直下。アイコン＋テキストの帯の下端を検出 |
   | 4 | 右端アイコン | 先頭フレームの右端からx方向スキャン。ユーザーアバター等のアイコン左端を検出 |
   | 5 | タスクバー | 先頭フレームの下端からy方向スキャン。タスクバーの上端を検出 |

   **検出手順：**
   1. 先頭フレームを抽出する
   2. 各項目の境界をピクセルスキャンで検出する
   3. 検出結果をクロップテストフレームとして切り出す
   4. **マスターに目視確認を取ってからクロップを確定する**（自己判断で確定しない）

   **禁止：** 過去の動画のクロップ値を流用すること。必ず毎回検出する。

2. **台本のフレーズ分割**
   - 台本テキストをテロップ表示用のフレーズに分割
   - 1フレーズ最大15文字目安
   - 改行（\N）は最大2行まで。3行は極力避ける
   - フレーズ分割はマスターから提供された台本に準拠

3. **OPアニメーションの尺調整**
   - マスター指示に従って尺を切り出し（通常4秒）

### Phase 2: シーン定義

1. **アバター動画とシーンの対応づけ**
   - avatar_slide1.mp4 → シーン1（導入）
   - avatar_slide2.mp4 → シーン2 …
   - 各アバター動画の尺を取得

2. **操作画面のタイムコード割り当て**
   - 各シーンに対応する操作画面の開始秒・終了秒を定義
   - 導入・締めは操作画面の冒頭/末尾フレームをフリーズ

3. **フリーズフレーム計算**
   - アバター尺 > 操作画面区間尺 → 差分をフリーズフレームで引き延ばし
   - フリーズは各区間の末尾フレームを静止画として使用

### Phase 3: 合成

各シーンについて以下を実行：

1. **ASS字幕ファイル生成**
   - レイアウト定数のテロップ仕様を使用
   - タイミングは文字数比例配分: `各フレーズの表示時間 = アバター尺 × (フレーズ文字数 / 全フレーズ文字数合計)`
   - 全Dialogue行に `{\pos(1115,955)}` を付与

2. **ffmpegフィルター合成**
   ```
   [テンプレート] → fps=30, format=yuv420p → [bg]
   [操作画面]     → scale=1552:770        → [screen]
   [アバター]     → scale=180:180         → [av]
   [bg] + [screen] → overlay=184:56       → [tmp1]
   [tmp1] + [av]   → overlay=200:865      → [tmp2]
   [tmp2]          → ass=字幕ファイル      → [out]
   音声は [アバター] の音声トラックを使用
   ```

3. **音声の取り扱い**
   - 音声は必ずアバター元ファイルから直接取得（再エンコードによる劣化を防ぐ）
   - 合成後にアバター元ファイルの音声で差し替え: `ffmpeg -i seg.mp4 -i avatar.mp4 -map 0:v -map 1:a -c:v copy -c:a copy`

### Phase 4: 結合

1. **OP + 全セグメントを結合**
   - OPの音声スペックをセグメントに合わせる（モノラル44100Hz）
   - 全体を再エンコードして結合（`-c copy`はコーデック不一致で音声が壊れるため禁止）
   ```
   ffmpeg -y -f concat -safe 0 -i concat.txt \
     -c:v libx264 -preset fast -crf 23 \
     -c:a aac -b:a 192k -ar 44100 -ac 1 \
     output.mp4
   ```

### Phase 5: 品質検証

合成後、マスターに報告する前に以下を全て自己検証する：

1. **アバター枠内収まり検証**
   - 3つ以上の異なるタイムポイントでフレームを抽出
   - アバター部分を切り出して枠内に収まっているか目視確認
   - 収まっていない場合はマスターに報告しない

2. **テロップ位置検証**
   - テロップが下部バー内に収まっているか確認
   - 文字がはみ出していないか確認

3. **音声検証**
   - 音声が途切れ・飛びなく再生されるか確認
   - シーン間の音声接続が自然か確認

---

## 禁止事項

- **座標の推定・解析による算出は禁止**。レイアウト定数を使うこと
- **`-c copy`での異なるスペックの結合は禁止**。必ず再エンコード
- **品質検証を省略してマスターに報告することは禁止**
- **見本動画の解析結果を自分の計測値より優先することは禁止**。承認済み成果物の実測値が唯一の正解

## 学習記録

### MATH-M-01-A での教訓

| 失敗 | 原因 | ルール化 |
|------|------|---------|
| テロップサイズを42pxと誤認 | v2動画を解析して間違った値を信じた | 承認済み成果物の実測値 or アプリ設定値を使う。自己解析を過信しない |
| アバターが枠からはみ出した | テンプレートの座標を目視推定した | レイアウト定数を使う。推定禁止 |
| 音声が飛んだ | `-c copy`でステレオ/モノラル混在を結合 | 結合前にスペック統一。全体再エンコード必須 |
| テロップが下寄りになった | ASS Alignment=8（上基準）で\posを中央に置いた | Alignment=5（中央基準）を使う |
| 操作画面が上下圧縮された | アスペクト比無視でスケールした | 高さ基準でアスペクト比維持。水平は中央配置 |
