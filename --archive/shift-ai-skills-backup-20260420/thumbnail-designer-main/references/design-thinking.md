# 広告デザイン思考

テンプレートではなく「考え方」をインストールする。
毎回ゼロベースで最適解を導くための、プロの広告設計原則。

---

## 1. 構図の原則

### なぜ構図が重要か

人間の目は無秩序に画面を見ない。構図は視線の道筋を設計する行為。
構図が良ければ、0.5秒で「何のサムネか」が伝わる。

### 使える構図パターン（状況に応じて自由に選ぶ）

| 構図 | 特性 | 向いている場面 |
|------|------|---------------|
| 三分割法 | 安定感があり目線が自然に流れる | 人物+テキストのバランス型 |
| 黄金比 | 美しさと情報密度の両立 | プレミアム感・信頼性重視 |
| Z字パターン | 左上→右上→左下→右下の順で読ませる | 情報量が多いバナー・広告 |
| F字パターン | 上→左→下の縦スキャン | テキスト中心のサムネ |
| 中央集中 | 一点に視線を集める圧倒的インパクト | ワンメッセージ・衝撃系 |
| 対角線 | 動きとダイナミズム | エネルギッシュ・アクション系 |
| フレーム構図 | 枠で囲んで注目を中央に | 人物ポートレート・プレミアム |

**重要**: パターンを機械的に選ばない。「このターゲットに、このメッセージを届けるには、どう視線を誘導すべきか？」から逆算する。

## 2. 視線誘導

### 視線を操る5つの武器

1. **人物の視線**: 人は写真の人物が見ている方向を追う。人物の目線の先に講座名を置く
2. **サイズの差**: 大きいものに目が行く。ヒーロー語を極端に大きくする理由
3. **色のコントラスト**: 周囲と違う色に目が吸い寄せられる。アクセントカラーの存在理由
4. **方向性のある図形**: 矢印・三角・斜線は視線を誘導する。無意識に導線を作る
5. **余白**: 何もない空間は「そこに大事なものがある」と知らせる。余白の隣が主役

### 読む順番の設計

サムネは「読む」ではなく「見る」メディア。情報を受け取る順番を設計する:

```
第1層（0.5秒）: 目が止まる → 主役ビジュアル or ヒーロー語
第2層（次の0.5秒）: 何の話かわかる → テーマ名 + フック行
第3層（興味を持ったら）: 詳細を確認 → 日程・登壇者・その他
```

## 3. 被写体の設計

### 「普通の写真」で終わらない

サムネイルは広告。報道写真ではない。合成・加工・演出前提で設計する。

### 被写体の選択肢

| タイプ | 効果 | 注意点 |
|--------|------|--------|
| 人物（イラスト） | 親しみやすさ、ターゲットの投影 | AIっぽいリアル人物は不気味の谷。スタイルを明示 |
| 人物（実写・写真素材） | 信頼感、リアリティ | ストックフォト感を避ける。自然なポーズ |
| 抽象グラフィック | 未来感、テック感、概念の視覚化 | 冷たくなりすぎない。温かみのある色と組み合わせる |
| プロダクト/ツール | 具体性、「これを使える」感 | 画面キャプチャそのままはダサい。モックアップ化する |
| アイコン/シンボル | わかりやすさ、シンプルさ | 単調になりやすい。配色と構図で補う |

### 魅力的な人物を主役にする場合

- ポーズ: 正面より斜め。動きがあるとエネルギッシュ
- 表情: テーマに合った表情（学び→真剣、イベント→笑顔、挑戦→力強い）
- 配置: 画面の1/3〜1/2を占める。小さすぎると意味がない
- 視線の先にテキストエリアを確保する

## 4. 背景とグラフィック

### 背景は「空気」を作る

背景は情報ではない。空気・雰囲気・世界観を作る役割。

### 背景の選択肢

| タイプ | 効果 |
|--------|------|
| グラデーション | 奥行き、未来感、プレミアム感 |
| 幾何学パターン | テック感、構造、知的さ |
| テクスチャ（紙・コンクリート等） | 質感、リアリティ、温かみ |
| ぼかし写真 | 文脈を伝えつつテキスト視認性を確保 |
| ベタ塗り + 図形 | ポップ、明快、エネルギッシュ |
| ペイントスプラッシュ/有機的図形 | 創造性、自由さ、ダイナミズム |

### 空間活用の原則

**テキストエリアを「避ける」設計はしない。**

背景に被写体・図形・グラフィックがある場合でも、それを避けて空白を作るのではなく、空間全体を無駄なく使う。ただし、文字で埋め尽くすことが目的ではない。講座名が最初に目に入り、サブタイトルが次に入る情報順序を守ることが最優先。

視認性が確保できない場合の対処は、以下の優先順位で検討する:

1. **構図の見直し**: 被写体とテキストが干渉しない配置に変える
2. **テキストのコントラスト強化**: 文字色・サイズ・太さで視認性を上げる
3. **ブラー・オーバーレイ**: ブラーをかける場合は背景全体に適用する。部分的なブラーは境界線が不自然になるため使わない。強度はテキストの可読性を確保できる最低限にとどめ、背景の雰囲気を残す。または半透明オーバーレイに切り替える（最終手段）

### 3層構造で奥行きを出す

```
奥: 背景（色・テクスチャ・写真）
中: 空気層（図形・エフェクト・パーティクル）
手前: テキスト + 主役ビジュアル
```

この3層を意識するだけで「普通のバナー」が「プロの広告」に変わる。

## 5. 配色の思考法

### 色は「感情」を設計するツール

| 色相 | 心理効果 | 向いている場面 |
|------|---------|---------------|
| 赤系 | 緊急性、情熱、エネルギー | セール、期間限定、アクション系 |
| 青系 | 信頼、知性、冷静 | ビジネス、テック、教育系 |
| 緑系 | 成長、安心、自然 | 入門系、ヘルスケア、サステナ |
| 黄/オレンジ系 | 活力、楽しさ、注目 | イベント、エンタメ、初心者向け |
| 紫系 | 創造性、高級感、神秘 | AI/テック、プレミアム、ブランド系 |
| ピンク系 | 親しみ、柔らかさ、フレンドリー | 女性向け、カジュアル、コミュニティ |
| 黒/ダーク系 | 高級感、権威、ドラマ | プレミアム、シリアス、プロ向け |
| 白/ライト系 | 清潔感、余白、洗練 | ミニマル、ビジネス、教育 |

### 配色選定の検証ステップ

配色を決める前に、以下の順で必ず考える。ネイビー・紺系はビジネス・AI・セミナー系で多用されており、指定しなければAIが自動でネイビーに寄せやすい。意図せずネイビーに流れないよう、必ず他の選択肢を先に検討する。

```
1. ターゲットは何を感じたいか？（安心・興奮・信頼・楽しさ・urgency…）
      ↓
2. その感情に最も合う色相はどれか？（上の表から選ぶ）
      ↓
3. ブランドカラーがある場合、それと整合するか？
      ↓
4. 競合サムネイルと差別化できるか？（同じ色相ばかりなら別を選ぶ）
      ↓
5. ネイビーを選ぶ場合は「なぜネイビーでなければならないか」を言語化する
```

**ネイビー以外で使いやすい代替パターン:**

| ターゲット層 | 代替配色の方向 |
|------------|-------------|
| 初心者・入門層 | 黄/オレンジ × 白（親しみやすさ・低障壁感） |
| 女性・コミュニティ系 | ピンク × ホワイト × ゴールド |
| プレミアム・高単価 | 黒 × ゴールド × 白 |
| AI・テック（差別化） | 紫 × 黒 × シアン |
| 成長・学び系 | 緑 × 白 × オレンジアクセント |

### 3色ルール

1. **背景色（60%）**: 全体の印象を決める
2. **テキスト色（30%）**: 読みやすさを確保する
3. **アクセントカラー（10%）**: 視線を集める。最も伝えたい要素に使う

**検証**: アクセントカラーの文字だけ読んで、メッセージが通るか？

## 6. タイポグラフィの思考法

### サムネのテキストは「読む」ではなく「見る」

- 文字サイズの強弱は極端に。中途半端は効かない
- ヒーロー語 = 画面の30%以上を占めるくらい大きく
- サブ情報は思い切り小さく。読ませなくていい、「ある」ことが伝わればいい

### サイズ比率の目安

```
ヒーロー語: ████████████████ (100%)
フック行:   █████████ (50-60%)
テーマ名:   ███████ (40-50%)
サブタイトル: ████ (25-30%)
日程・詳細: ██ (15-20%)
```

### 人物が登場する場合の名前・肩書きの比率

人物名は講座名の2/3程度のサイズにする。肩書きはさらに小さく、人物名の50〜60%程度を目安にする。名前・肩書きはあくまで補足情報。講座名の存在感を損なわないサイズに抑える。

### 単位系の処理

数字は大きく、単位（月/日/円/名/回等）は小さく。
例: **22**日 **SUN** ← 数字が主役、単位は添え物

### フォント選びの考え方

| 印象 | フォントタイプ |
|------|---------------|
| 信頼・権威 | セリフ体（明朝系） |
| モダン・クリーン | サンセリフ（ゴシック系） |
| 手書き・温かみ | スクリプト/手書き風 |
| インパクト | 極太ウェイト（Black/Heavy） |
| こなれ感 | 和文 + 小さめの欧文の組み合わせ |

フォント3種以内。見出し用 + 本文用 + アクセント用。

## 7. レタリングエリア（テキスト余白）の設計

### 画像生成AIの最大の課題

画像生成AIはテキストの正確な描画が苦手。だからこそ:

- **プロンプト段階でテキストを入れるスペース（レタリングエリア）を構図に組み込む**
- 「右半分をネガティブスペースにしてテキスト配置用に確保」等、明示的に指示する
- 生成後に Canva / Figma 等でテキストを追い込み仕上げする前提で設計する

### レタリングエリアの確保方法

| 方法 | 説明 |
|------|------|
| 左右分割 | 片側に被写体、もう片側をテキストエリアに |
| 上下分割 | 上部にビジュアル、下部に帯を敷いてテキスト |
| 背景ぼかし | 被写体の周囲をぼかして文字が読める領域を作る。**視認性確保の最終手段として使う** |
| オーバーレイ | 半透明の色面をテキストの背景に敷く。**視認性確保の最終手段として使う** |
| 被写体内の余白 | 構図自体に自然な余白ができるように被写体を配置 |

## 8. 情報密度のバランス

### 余白を恐れない。でも余白だらけもNG

- **余白が多すぎ**: 情報が薄く、プロっぽさに欠ける
- **余白が少なすぎ**: ゴチャゴチャして何も伝わらない
- **ちょうどいい**: 「広告らしい密度感」がある。プロのバナーを参考にする

### 判断基準

```
スマホの画面で見たとき:
- メインメッセージが1秒以内に読めるか？ → YES なら適切
- 3つ以上の情報が同時に目に入るか？ → YES なら詰めすぎ
- 何もない空間が画面の50%以上か？ → YES なら寂しすぎ
```

## 設計プロセスのまとめ

毎回このフローで思考する:

```
1. 誰に何を伝える？（ターゲット × メッセージ）
     ↓
2. どう視線を誘導する？（構図の選択と理由）
     ↓
3. 何を主役にする？（被写体の選択と理由）
     ↓
4. どんな空気感？（背景・配色の選択と理由）
     ↓
5. テキストはどこに？（レタリングエリアの確保）
     ↓
6. 文字の強弱は？（タイポグラフィの設計）
```

全てのステップで「なぜそうするか」を言語化できること。
言語化できない設計判断は、ただの当てずっぽう。
