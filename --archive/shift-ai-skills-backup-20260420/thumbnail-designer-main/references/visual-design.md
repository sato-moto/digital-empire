# ビジュアルコンセプト設計ガイド

Phase 2 専用。企画情報 → ビジュアルコンセプト設計 → 配色確認。
設計原則の詳細 → **references/design-thinking.md**

---

## 大原則

テンプレートは使わない。広告設計のプロの思考で、毎回ゼロベースで最適解を導く。

---

## STEP 1: ビジュアルコンセプト設計

以下のフォーマットで出力する。

```
【ビジュアルコンセプト】

■ 構図・レイアウト
[何をどこに、なぜそう配置するか。黄金比・三分割法・視線誘導の根拠]

■ 主役・被写体
[メインビジュアルの具体的描写。人物なら表情・服装・ポーズ。オブジェクトなら形状・質感]
※参照画像ありの場合: 人物が向いている方向（左向き／右向き）を確認し、視線の先にテキストエリアを配置する。配置の左右はこの視線方向から決定する。
※人物を小さく丸型で配置するパターン: 宣材写真を円形にトリミングしてサムネイル内に小さく配置する。登壇者の顔を認識させつつ、講座名の存在感を損なわないサイズに抑える。名前・肩書きはこの丸型写真の隣に添える。

■ 背景・グラフィック
[背景処理、図形、テクスチャ、エフェクト。「普通の写真」で終わらない合成・加工の提案]

■ 配色（3色以内）
[色名レベルで選定理由（心理効果・ブランド整合・視認性）を明記]
※ロゴ視認性ルールを必ず確認すること（下記参照）

■ タイポグラフィ計画
[テキスト要素の配置・サイズ比率・フォントイメージ。文字の余白エリア（レタリングエリア）も指定]

■ この設計で狙う効果
[1-2文で。なぜこの設計がターゲットに刺さるか]
```

---

## STEP 2: 配色確認（STEP 1完了後に必ず実施）

ビジュアルコンセプトを出力したら、Phase 3に進む前にユーザーに配色を確認する。

```
【配色確認】

■ 推奨配色
[選定した配色 + 選定理由を1文で]

■ 他の候補
案A: [配色名] → [ターゲットへの効果・印象]
案B: [配色名] → [ターゲットへの効果・印象]
案C: [配色名] → [ターゲットへの効果・印象]

この配色で進めますか？変更する場合は候補から選ぶか、ご希望の配色をお知らせください。
```

- ユーザーが配色を確定したら、その配色をPhase 4のトーンセクションに必ず反映する
- ユーザーが回答せずに次の話題に進んだ場合は、Phase 3に進む前に必ず確認を取る

## ロゴ視認性ルール

SHIFT AI ロゴ（赤〜マゼンタ〜紫〜ネイビーのグラデーション）が左上に自動合成される。
**左上エリアの背景色**が以下に該当する配色は避けること。

### NG背景色（ロゴが埋もれる）

- 赤・マゼンタ・ピンク系
- 紫・バイオレット系
- ワインレッド・ダークレッド系

### 推奨背景色（ロゴが映える）

| 背景色 | 理由 |
|--------|------|
| ダークネイビー（濃） | 実績あり。赤〜紫が鮮やかに浮く |
| チャコール / ダークグレー | 無彩色でロゴ全色が際立つ |
| ブラック | 最も安全 |
| ダークグリーン / フォレストグリーン | 赤の補色でロゴが目立つ |
| ダークティール / シアン系 | 色相が離れていて共存しやすい |
| ホワイト / ライトグレー | 明度差が十分 |
| ベージュ / クリーム | 温かみあり、赤紫が自然に映える |
