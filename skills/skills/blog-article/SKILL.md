# SKILL: Blog Article

熊取つーしん専属ライタースキル。
キーワードを受け取り、SEO最適化されたWEB記事HTMLを生成し、WordPressへ下書き投稿するまでを自動化する。

---

## ファイル保存ルール

**確認なしで即保存する。**

- **保存先**: `~/blog/articles/`
- **ファイル名**: `XX_[カテゴリ英字]_[スラッグ].html`
  - XX = 既存記事の最大番号 + 1（現在は51が最大 → 次は52）
- **既存ファイルは絶対に上書きしない**

---

## Phase 0: Context Loading

以下を順番に読み込む。存在しない場合はスキップしてPhase 1へ進む。

1. `~/blog/CLAUDE.md` — ペルソナ・業務範囲・品質ルール
2. `~/blog/writing-research-rules.md` — Phase 1〜7の詳細ルール
3. `~/blog/.claude/rules/article-rules.md` — フォーマット・SEOルール・Yoast緑化チェック

## Phase 1: ヒアリング

以下をユーザーに確認する。まとめて1回で聞く。

```
記事制作を開始します。以下を教えてください。

1. ターゲットKW（例: 熊取 ランチ おすすめ）
2. 記事のネタ・背景情報（知っていること・取材メモ・SNS情報など）
3. カテゴリ（下記から選択）
   - グルメ・飲食店
   - 観光・おでかけ
   - 暮らし・住まい
   - 子育て・教育
   - イベント・祭り
   - 歴史・文化
   - 自然・アウトドア
4. 既存の内部リンク候補（任意。なければこちらで選定）
```

**ユーザーが「任せる」と言った場合：** カテゴリ・内部リンクはこちらで判断して進める。

## Phase 2: 構成案の作成

writing-research-rules.md の Phase 1〜5 に従い、以下を出力する。

### 2-1. KW設計メモ（簡略版）

```markdown
## KW設計
- メインKW: [KW]
- 共起語候補: [3〜5語]
- 検索意図: [Know / Do / Go / Buy]
- ペルソナ: [1文で]
```

### 2-2. 構成案

```markdown
## 構成案

**タイトル**: [32文字以内・KW前方配置・数字or感情ワード入り]
**SEOタイトル**: [30文字前後]
**メタディスクリプション**: [80文字以内]
**スラッグ**: [英小文字・ハイフン区切り]
**フォーカスKW**: [メインKW]

---

■ リード文（150〜200文字の方針）

■ h2: [セクション1]
  └ h3: [サブ]
  └ h3: [サブ]

■ h2: [セクション2]
  └ h3: [サブ]
  └ h3: [サブ]

■ h2: [セクション3]
  └ h3: [サブ]

■ h2: まとめ

■ 内部リンク候補: [2〜3本]
■ 外部リンク候補: [1本以上]
```

構成案を出力したあと、**ユーザーの確認を待たずに即Phase 3へ進む**。
（修正が必要な場合はユーザーが指摘する）

## Phase 3: 本文執筆（HTML生成）

writing-research-rules.md の Phase 6 に従い、以下のフォーマットで完全なHTMLファイルを生成する。

### 出力フォーマット

```html
<!-- META:
{
  "title": "記事タイトル",
  "seo_title": "SEOタイトル（30文字前後）",
  "meta_description": "メタディスクリプション（80文字以内）",
  "slug": "url-slug-here",
  "category": "カテゴリ名",
  "tags": ["タグ1", "タグ2", "タグ3"],
  "focus_keyword": "フォーカスKW",
  "article_number": XX,
  "last_updated": "YYYY-MM"
}
-->

<h1>記事タイトル</h1>

<p>リード文...</p>

<!-- 目次 -->
<nav>
<h2>目次</h2>
<ul>
  <li><a href="#section1">セクション1</a></li>
  ...
</ul>
</nav>

<!-- 本文 -->
...

<!-- まとめ -->
<h2>まとめ</h2>
...

<!-- 関連記事 -->
<div class="related-articles">
<h3>関連記事</h3>
<ul>
  <li><a href="/内部リンク先/">記事タイトル</a></li>
</ul>
</div>
```

### 執筆ルール（必須）

- 文字数：2,500〜3,500文字（目標3,000文字）
- 冒頭1〜2段落にフォーカスKWの全語を自然に含める
- フォーカスKWを本文中に7回以上登場させる（見出し含む）
- 全h2にフォーカスKWまたは類義語を含める
- 一文は60文字以内
- 一段落は3〜4行まで
- AI感ワード禁止：「〜することが重要です」「〜が求められます」「〜を活用しましょう」
- 内部リンク2〜3本・外部リンク1本以上
- 店舗・人物のSNS公式アカウントがあれば必ずリンク化

## Phase 4: 品質チェック（自動）

HTML生成後、以下を自己チェックして問題があれば即修正する。ユーザーへの確認は不要。

### Yoast緑化チェック

- [ ] SEOタイトルが30文字前後
- [ ] メタディスクリプションが80文字以内
- [ ] 冒頭1〜2段落にフォーカスKWの全語が含まれている
- [ ] フォーカスKWが本文中に7回以上登場している
- [ ] 全h2にフォーカスKWまたは類義語が含まれている
- [ ] 内部リンク1本以上・外部リンク1本以上存在する

### コンテンツ品質チェック

- [ ] 文字数が2,500〜3,500文字の範囲内
- [ ] AI感ワードが含まれていない
- [ ] 不確かな情報（憶測・裏取り未済）が含まれていない
- [ ] 飲食店情報は3ヶ月以内の鮮度要件を満たしているか（ユーザー提供情報に依存）
- [ ] h1 → h2 → h3 の階層が正しい

チェック完了後、問題なければPhase 5へ進む。
問題があれば修正してから進む。

## Phase 5: ファイル保存 → アイキャッチ生成 → WP投稿

### Step 1: ファイル保存

```bash
# ファイルを articles/ に保存（wp-auto-post.js が読み込む形式）
# ファイル名: XX_[カテゴリ英字]_[スラッグ].html
```

カテゴリ英字の対応表：

| カテゴリ | 英字 |
|---------|------|
| グルメ・飲食店 | gourmet |
| 観光・おでかけ | tourism |
| 暮らし・住まい | living |
| 子育て・教育 | parenting |
| イベント・祭り | event |
| 歴史・文化 | history |
| 自然・アウトドア | nature |

### Step 2: アイキャッチ画像生成

```bash
node generate-eyecatch.js articles/XX_xxx.html
```

- 生成完了まで待つ（数分かかる場合あり）
- 出力先: `images/XX_xxx.png`

### Step 3: WordPress下書き投稿

```bash
node wp-auto-post.js articles/XX_xxx.html
```

- 投稿ステータスは常に **draft（下書き）**
- 投稿完了後、WP管理画面URLを案内する

### Step 4: 完了報告

```
✅ 記事制作完了

- ファイル: articles/XX_xxx.html
- 文字数: 約X,XXX文字
- WP下書き: 投稿済み（ID: XXXX）
- アイキャッチ: images/XX_xxx.png

⚠️ 確認事項（WP管理画面で実施）
- [ ] ファクトチェック（営業時間・価格・住所など）
- [ ] アイキャッチ画像の確認・差し替え
- [ ] Yoast SEOの緑ランプ確認
- [ ] 公開前の最終レビュー
```

---

## 重要原則

1. **ファクトチェックは人間がやる** — AIは構造と文章を担当。事実の最終確認はマスターが行う
2. **確認より前進** — Phase 2の構成案はユーザー確認を待たず即Phase 3へ進む
3. **下書き投稿のみ** — 公開（publish）は絶対に行わない
4. **既存ファイルは上書きしない** — 記事番号を必ず確認してからファイル名を決定する
5. **AI感を出さない** — 定型表現・曖昧表現は徹底排除
