# 記事評価AI（ダメ出し専用）
# ※ 褒めることより改善点の指摘を最優先とする。
# ※ このファイルはSKILL.md Phase 4.2で必ず参照すること。

---

## 使い方

1. 評価対象のHTMLを読み込む
2. 以下の参照ファイルを必ず読んでから評価する：
   - `article-base-structure.md`（8構造骨格）
   - `article-content-rules.md`（執筆基準・型別ルール）
   - `article-template-catalog.md`（型テンプレート・理想構成） ← **記事の型を特定して該当テンプレと照合すること**
3. 記事の型を判定する（A〜Jのどれか）
4. 採点シート（10項目×10点）で採点する
5. 下記フォーマットで評価レポートを出力する
6. **合格ライン：80点以上**。79点以下は差し戻し → Phase 3 で修正 → 再評価

## 採点シート（10項目 × 10点 = 100点満点）

---

### ① 構造の遵守（10点）

**配点基準**

| 状態 | 点数 |
|------|------|
| 8構造すべて存在する | 10点 |
| ⑤または⑥のいずれかが欠落 | 3点 |
| ⑤と⑥の両方が欠落 | 0点 |
| ①〜④の一部が欠落 | 5点 |

**チェック項目**
- [ ] ① リード文が冒頭にある（背景説明から始まっていない）
- [ ] ② 要点まとめが箇条書きor表で3〜5項目ある
- [ ] ③ 詳細セクションが②の繰り返しになっていない
- [ ] ④ 現地・体験・一次情報セクションがある（または未確認注記がある）
- [ ] ⑤ 「誰におすすめか」が2〜4属性で具体的に書かれている ← **欠落で即3点減**
- [ ] ⑥ 「注意点・デメリット」が実用的な内容で書かれている ← **欠落で即3点減**
- [ ] ⑦ CTA（外部リンク＋内部リンク）が設置されている
- [ ] ⑧ まとめが100〜150文字で書かれている

**即0点パターン**
- ⑤⑥の両方が欠落
- h1が存在しない
- まとめセクションが存在しない

---

### ② 各セクションの質（10点）

**配点基準**

| 状態 | 点数 |
|------|------|
| 全セクションが基準を満たしている | 10点 |
| 1〜2セクションが基準を下回る | 6〜8点 |
| 3〜4セクションが基準を下回る | 3〜5点 |
| 5セクション以上が基準を下回る | 0〜2点 |

**チェック項目（各2点）**
- [ ] ① リード文が150〜200文字で「誰向けか」を明示している
- [ ] ② 要点まとめに日時/場所/料金/注意点が含まれている（記事性質による）
- [ ] ④ 現地情報が「公式サイトの転記」ではなく独自情報を含んでいる
- [ ] ⑥ 注意点が「〜には注意が必要です」で終わっておらず、具体的な内容が書かれている
- [ ] ⑧ まとめが①のリード文と別の言葉で書かれており、関西弁感想が1箇所以上ある

---

### ③ 情報の具体性（10点）

**配点基準**

| 状態 | 点数 |
|------|------|
| 全ての主張に具体的な数値・名称・描写がある | 10点 |
| 一部に抽象表現があるが致命的でない | 6〜8点 |
| 抽象表現が目立ち、読んでも状況が浮かばない | 2〜5点 |
| ほぼすべての文が抽象的 | 0〜1点 |

**即減点トリガー（各−2点）**
- 「とても〜」「非常に〜」「素晴らしい〜」が根拠なく使われている → -2点
- 営業時間・価格・住所等の基本情報が欠落している → -2点
- 「〜が人気です」「〜で有名です」と書いてあるが根拠がない → -2点
- 「雰囲気が良い」「居心地が良い」のような形容詞のみで終わっている → -2点

**OK例 vs NG例**

OK：「平日12時に訪問。8割方埋まっており、カウンター4席・テーブル3卓（約16席）の店内は10分で満席になった」
NG：「人気のお店で、いつも多くのお客さんで賑わっています」

---

### ④ 一次情報の有無（10点）

**配点基準**

| 状態 | 点数 |
|------|------|
| 取材・実体験に基づく独自情報がある | 10点 |
| SNS・口コミ等2次情報だが出典明記あり | 6点 |
| 公式サイトの内容のみ（転記レベル） | 2点 |
| 未確認注記もなく出典不明 | 0点 |

**チェック項目**
- [ ] 「実際に行った・食べた・使った」という主語のある記述がある
- [ ] 公式サイトに書かれていない情報が1つ以上ある
- [ ] 未取材の場合「※編集部未確認・情報は〇〇年〇月時点」と明記されている

**即0点パターン**
- 本文が公式サイト・Googleマップ情報の転記のみで構成されている

---

### ⑤ 読者目線（10点）

**配点基準**

| 状態 | 点数 |
|------|------|
| ターゲット読者が明確で全編を通じてブレない | 10点 |
| ターゲットがリード文にはあるが途中から消える | 6点 |
| ターゲットが曖昧or「みんなにおすすめ」 | 2点 |
| ターゲット情報がどこにもない | 0点 |

**チェック項目**
- [ ] リード文で「誰向けか」が1文で明示されている
- [ ] ⑤のセクションに2〜4属性の具体的なおすすめ対象がある
- [ ] 「みんなにおすすめ」「あらゆる方に」等の全方向表現がない
- [ ] 属性ごとに「なぜ合っているか」の理由が書かれている

**NG検出ワード**（出現で-3点）
- 「どなたにも」「幅広い方に」「老若男女問わず」「どんな方でも」

---

### ⑥ 注意点・デメリットの質（10点）

**配点基準**

| 状態 | 点数 |
|------|------|
| 実用的な注意点が2〜4項目、具体的に書かれている | 10点 |
| 注意点はあるが「〜には注意が必要です」で終わっている | 4点 |
| 注意点セクションがない | 0点 |
| 「デメリットはありません」等の褒め記事化 | 0点 |

**チェック項目**
- [ ] 注意点が「誰が・どんな状況で・具体的に何に困るか」まで書かれている
- [ ] アクセス・駐車場・混雑・営業時間のうち少なくとも1項目が含まれている
- [ ] 「〜には注意が必要です」だけで終わっておらず、具体的な情報がある

**OK例**: 「駐車場は10台のみ。休日11〜13時は満車になることが多い。近隣にコインパーキングなし」
**NG例**: 「混雑することもあるので気をつけてください」

---

### ⑦ CTAの強さ（10点）

**配点基準**

| 状態 | 点数 |
|------|------|
| 外部リンク（アンカーテキスト）＋内部リンク2〜3本が設置されている | 10点 |
| 外部リンクはあるが内部リンクが1本以下 | 6点 |
| 内部リンクのみ、または外部リンクのみ | 4点 |
| CTAがない or URLの生貼りのみ | 0点 |

**チェック項目**
- [ ] 外部リンクがアンカーテキストでリンク化されている（URLそのまま貼り禁止）
- [ ] 内部リンクが2〜3本、関連性の高い記事にリンクされている
- [ ] 「次の行動」が1つ明確に提示されている（複数CTA乱立は-2点）
- [ ] 来店・参加・予約等の行動転換に直結するリンクがある

**即0点パターン**
- CTAセクションが存在しない
- `https://〜` をテキストとして貼り付けただけ（リンクなし）

---

### ⑧ SEO観点（10点）

**配点基準**

| 状態 | 点数 |
|------|------|
| タイトル・KW出現・h2構成すべて基準内 | 10点 |
| 1〜2項目が基準を外れている | 6〜8点 |
| 3項目以上が基準を外れている | 2〜5点 |

**チェック項目（各2点）**
- [ ] SEOタイトルが30文字前後（32文字超は減点）
- [ ] メタディスクリプションが40〜80文字（短すぎ・長すぎ両方オレンジ）
- [ ] フォーカスKWが冒頭1〜2段落に含まれている
- [ ] フォーカスKWが本文中に7回以上登場している
- [ ] 全h2にフォーカスKWまたは類義語が含まれている

---

### ⑨ 読みやすさ（10点）

**配点基準**

| 状態 | 点数 |
|------|------|
| 一文60字以内・一段落4行以内・AI感なし | 10点 |
| 一部長文・冗長があるが概ね読める | 6〜8点 |
| 長文多発・AI感が漏れている | 2〜5点 |
| 全体的にAI感が強く地域メディアの文体になっていない | 0〜1点 |

**即減点トリガー（各−2点）**
- 一文が80文字を超えている → -2点/1件
- 同じ内容を異なる表現で2回書いている（①②の繰り返し等） → -2点
- HTML目次を本文に手書きしている（WPプラグインが自動生成するため禁止） → -3点

---

### ⑩ NG表現チェック（10点）

**10点満点からの減点式。各NG検出で-2点。最低0点。**

**自動検出すべきNGワード・パターン**

| NGワード/パターン | 減点 | 代替表現の例 |
|----------------|------|------------|
| 「〜することが重要です」 | -2点 | 「〜しておくと失敗しない」 |
| 「〜が求められます」 | -2点 | 「〜が必要になる場面がある」 |
| 「〜を活用しましょう」 | -2点 | 「〜を使うのがおすすめ」 |
| 「〜してみましょう」 | -2点 | 「〜してみて」 |
| 「〜でしょう」（憶測） | -2点 | 断言するか情報源を明記 |
| 「〜と言われています」（根拠なし） | -2点 | 情報源を明記か削除 |
| 「〜のようです」（伝聞のみ） | -2点 | 確認済み情報に差し替え |
| 「老若男女」「幅広い方」 | -2点 | 具体的な属性に書き直し |
| 「ぜひ最後までお読みください」 | -2点 | 削除 |
| 「以上です」「いかがでしたか」 | -2点 | 削除してまとめを書き直し |

## 差し戻しループの設計

```
Phase 3（HTML生成）
    ↓
Phase 3.5（固有名詞確認）
    ↓
Phase 4（Yoast緑化チェック）
    ↓
Phase 4.2（評価AI ← このファイル）
    ↓ 79点以下
   差し戻し ────────────────→ Phase 3に戻る（修正指示を渡す）
    ↓ 80点以上
Phase 4.5（制作ログ生成）
    ↓
Phase 5（ファイル保存・WP投稿・Notion登録）
```

**差し戻し時の注意**
- 同じ記事の最大差し戻し回数：2回
- 3回目の評価で79点以下の場合：マスターに報告して指示を仰ぐ（自律修正を打ち切る）
- 差し戻し理由は修正指示と一緒にPhase 3に渡す

## 評価の厳格度基準

| 項目 | 厳格度 | 理由 |
|------|--------|------|
| ⑤ 誰におすすめか | **最高** | 欠落でも世に出てしまう。必須項目 |
| ⑥ 注意点 | **最高** | 欠落は読者への不誠実。信頼構築の根幹 |
| ④ 一次情報 | **高** | ないと大手サイトと同列。存在意義がない |
| ③ 情報の具体性 | **高** | 抽象的な記事は検索で評価されない |
| ⑩ NG表現 | **高** | AI感はブランドダメージ |
| ① 構造遵守 | **中** | 一部崩れても致命的でない場合あり |
| ⑦ CTA | **中** | 内部リンク1本欠けても記事として機能する |
| ⑧ SEO | **中** | Yoast緑化でPhase 4に担保されている |
