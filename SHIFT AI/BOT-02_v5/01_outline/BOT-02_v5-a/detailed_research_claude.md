# 追加リサーチ（Claude版）— SET1「AIに教材を持たせる」

**モデル役割**: メインClaude（講座リサーチャー）— 教育設計観点での情報変換を最優先

---

## 第1層：【事実】

### NotebookLMの基本仕様

- NotebookLM は「ユーザーが指定したソース（PDF／テキスト／Webページ／音声／動画／Googleドキュメント等）の中身に限定して応答する」タイプの生成AIツール
- 各ノートブック単位でソースを管理し、ノートブック間でソースは独立
- ソース追加後、自動で全文インデックスが作られ、チャット欄での質問に対し「どのソースのどの箇所を根拠にしたか」の引用スニペットが回答と並んで表示される
- 無料プランでも学生が日常利用できるソース数・チャット回数の範囲がある。有料プランではソース上限・音声要約分数・アップロード容量が拡張される
- 画面構成は3ペイン：左ペイン（ソース一覧）／中央ペイン（チャット）／右ペイン（スタジオ：音声要約・マインドマップ・ブリーフィングドキュメント等の自動生成機能）

### 類似ツールとの関係

- 同系統の汎用AIは「モデル内部に世界中の知識を持つ」設計。一方 NotebookLM は「モデルに渡した数件のソースにしか触れない」設計
- Gems（姉妹ツール）は「システムプロンプトで人格・役割・返答ルールを固定する」機能。教材の取り込みは原則的ではなく、対話の枠組みを固定することに強みがある
- 用途の棲み分け：**素材を縛るのが NotebookLM／振る舞いを縛るのが Gems**

### 入力可能なソースの主な種類

- PDF（テキスト抽出可能なもの）
- 画像をOCRした結果を含む資料
- Googleドキュメント／スプレッドシート
- テキスト貼付け
- 音声ファイル（録音ノート等）
- YouTube動画（自動書き起こしがある前提）

### 範囲外質問時の挙動

- ソースに書かれていない事項を尋ねると、回答本文で「ソースには該当情報がありません」旨を明言し、ユーザーが持ち込んだ素材の外側にある知識で補完することを避ける傾向が設計として組み込まれている
- ただしソース内に「部分的にしか書かれていない」事項については、ソース内記述を根拠に推論を返すケースがある（第4層参照）

### 学生利用を後押しする機能

- Audio Overview：ソースを2名の対話形式で音声要約。通学中・就寝前等の受動学習に使える
- マインドマップ：ソース内容を階層ツリーで可視化。単元構造の把握に使える
- モバイル対応：ブラウザ版が主軸だが、モバイル環境でも一通りの閲覧・チャットが可能
- ソースの追加・削除は随時可能（単元の進行に合わせて継続運用できる）

## 第2層：【複数ソースで一致する論点】

### 論点1：ソース縛りはハルシネーションを減らすが、ゼロにはしない

複数の検証記事・教育ブログ・公式ドキュメントで一致する見解：
- 「渡したソースに存在する事実」については汎用AIより誤答が少ない
- ただし「ソース同士の整合性が取れていない」「ソース内記述が曖昧」な場合、それをもとにした推論で不正確な応答が返ることはある
- 要点：**ソースの質が出力の質を決める**。ソース選定の設計が学習成果を左右する

### 論点2：一次情報（授業ノート／自作プリント）をソース化する価値は高い

- 市販教材は「一般的な説明」、学校教材は「自分が今学んでいる文脈」という非対称がある
- 自分が書いたノート・自分の先生の板書は、**自分の理解状態と他では得られないレベルで整合している**
- 一次情報を取り込むと「自分の解き方で説明してくれる」AIが作れる—これは他のどんな高性能な汎用AIでも代替できない領域

### 論点3：ソースを絞るほど精度と"自分ごと感"が上がる

- ノイズになりうる資料（他単元・他科目・昔の参考書）を混ぜると、質問と無関係な情報が根拠として引かれやすい
- 5〜10件程度の厳選ソースで運用するほうが、30〜50件を雑多に入れるより体感の回答品質が高い
- 範囲を絞る＝「自分の今の学習範囲」を AI に教える行為とほぼ同義

### 論点4：NotebookLM単体では"先生の教え方"は再現できない

- NotebookLMは素材に詳しい「資料室」。ただし「どう教えるか」「どういう順番で導くか」「つまずいたときにどう励ますか」といった**教え方**は素材からは引き出せない
- 教え方を固定したい場合は Gems 等の振る舞い設定ツールが別途必要
- 本講座SETで強調する設計思想：**教材 × 人格 = 専属家庭教師**

### 論点5：学習サイクルに組み込むと効果が最大化する

- 単発利用より、新単元の進行に合わせてソースを追加・古い単元を削除する「運用サイクル」を作ったほうが継続率・理解度とも高い
- テスト前に「この単元のソースだけ残したノートブック」を作ると、その期間の学習パートナーになる

## 第3層：【仮説・推論＝講座設計に使える示唆】

### 示唆1：Why の核は「汎用AIの万能さが逆に邪魔をする経験」

学生が感じている**隠れた不満**は「AIに聞いたら頭がよさそうな答えが返ってくるけど、自分の教科書とは違う解き方だった」という経験。この"ズレの違和感"を Why で言語化できれば、受講者は自分ごととして問題意識に火がつく。

**Why導入セリフ候補**：
> 「分からない問題をAIに聞いたら、たしかに正解は返ってくる。でも、学校で習ったやり方と違う。参考書とも違う。気がつくと、答えが3つ並んでどれを信じればいいか分からなくなる。—— こういう経験、ないですか？」

### 示唆2：What の例え話は「万能型 vs 専属型」の二項対立が最も刺さる

**例え話候補（親近感系）**：
1. 「世界中の本を暗記した天才」vs「自分の教科書だけを完璧に暗記した助手」（骨子案）
2. 「どんな料理も作れる有名シェフ」vs「うちの冷蔵庫の中身だけで作ってくれる家族」
3. 「全部覚えた家庭教師の派遣会社」vs「自分の学校の先生のノートだけ読み込んだ個人コーチ」

**推奨**：候補1を基調にしつつ、一次台本では候補3の「学校の先生のノート」を補足比喩として差し込むと、ノート=ソースという発想転換が自然に入る。

### 示唆3：How は「ソース選びの判断軸」を3段階に整理すると覚えやすい

**3段階モデル案**：
- **S級（最優先）**：自分が書いたノート・先生の板書・自作プリント　→ 一次情報
- **A級（次点）**：学校配布の印刷教材・購入した参考書の該当章のみ　→ 範囲を絞った二次情報
- **X（避ける）**：関係ない科目・単元、SNSのまとめ画像、出典不明のPDF　→ ノイズ

**命名のブレスト**：S級/A級/X級 以外の案として「主食／副菜／ゴミ」「メインキャスト／サブキャスト／ノイズ」等も検討可。

### 示唆4：NotebookLM × Gems の役割分担は2軸マトリクスで可視化すると一発で伝わる

```
         知識量が多い  ←——→  特定ソースに詳しい
振る舞い固定  [汎用AI+Gems]     [NotebookLM+Gems] ← 理想形
振る舞い自由  [汎用AI素]          [NotebookLM素]
```

**差分コピー案（1行コピー）**：
1. 「NotebookLMは教材を持たせる。Gemsは先生の人格を持たせる」（骨子案）
2. 「何を知っているかは NotebookLM／どう教えるかは Gems」
3. 「資料の主 = NotebookLM／授業の主 = Gems」

**推奨**：1を基調に、2を補足コピーとして1度だけ挿入すると記憶に残る。

### 示唆5：教養How別事例「英単語帳」は"自作暗記ノート"を主語にする

英単語は一貫ケース（数学）と**構造が類似**しているが題材が違う。学生の多くが既に経験している題材なので、比喩のハードルが低い。

**別事例シナリオ案**：
> 「自分でまとめた英単語ノート（語源メモ＋例文＋語呂合わせ）をPDFにして NotebookLMに読ませる。すると、次のテスト範囲の単語だけを使って、自分の語呂合わせを踏まえた3択クイズを出してくれる。市販の単語帳アプリでは絶対に作れない"自分専用の問題集"が手元に生まれる」

この例を How の中で1回出すだけで、「一次情報＝自分のノート」という発想が腹落ちする。

### 示唆6：実務編のサンプルプロンプトは"解き方の縛り"を入れると一貫ケースに刺さる

**プロンプト例（一貫ケース=高校2年生・数学向け）**：

1. 基本質問型（ソース内で説明を受ける）
```
このノートとプリントに書かれている手順だけを使って、微分の基本公式を教えて。
他の解き方は使わないで。
```

2. つまずき発見型（自分の弱点を AI に整理させる）
```
このノートの範囲で、生徒がよくつまずくポイントを3つ挙げて。
それぞれに、ノートのどのページの記述を根拠にしたかも添えて。
```

3. 範囲外検知型（ソース縛りを体感させる）
```
ソースに書かれていない内容を聞かれたら「資料にありません」とだけ答えて。
勝手に補足しないで。
```

### 示唆7：Wrap は"運用サイクル"で終わらず"次SET（Gems）"に橋を架ける

実務編のWrapで運用サイクルだけを伝えて終わると、「教材を持たせた。次は？」の期待が宙ぶらりんになる。
- 「今日、教材を持たせた。次は、その教材を使う"先生"を作りにいこう」と1行で次SETに接続する
- 熱いメッセージは「相棒トーン」で簡潔に。長い精神論は避ける

## 第4層：【使用禁止・要注意】

### 使用禁止

- 「NotebookLMはハルシネーションゼロ」という断定　→ 事実ではない（第2層論点1参照）
- 「Gems／Geminiは劣化版」という価値判断的な比較　→ 役割違いの併用が正しいメッセージ
- 特定の有料プラン名や価格帯の断定的記述　→ 2026年4月時点で変動の可能性あり
- 固有の企業名・有名講師名・特定予備校名のセリフ内使用

### 要注意（変動リスク）

- 画面UI（ボタン位置・メニュー名）は**収録時点のスクリーンショットで裏取り必須**。数ヶ月単位で変更される
- Audio Overview の対応言語・生成所要時間は変動傾向
- モバイル版の操作感は PC 版と完全には揃っていない。実務編の画面録画は**PC版を前提**に作る方が安全

### 学生への注意喚起（Wrap の1〜2行で触れるべき）

- **著作権**：市販教科書・参考書のPDF化は「自分の私的利用の範囲」に留める。他人と共有するノートブックには市販教材の本文を直接入れない
- **個人情報**：友人との交換ノートや先生の個別フィードバックが写ったプリントを入れる場合、他人の名前や個人情報がスニペット表示されないよう注意
- **学校方針**：授業内で AI 利用を制限している学校もある。校則・先生の指示を優先する
