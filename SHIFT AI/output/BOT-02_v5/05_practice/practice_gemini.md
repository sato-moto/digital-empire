# 実践動画 指示書：BOT-02_v5-b - 自分の学習Botを作ってみる（実演）

## 0. メタ情報
- **講座**: SHIFT AI講座 / BOT-02_v5-b / 自分の学習Botを作ってみる（実演）
- **対応章**: SET1 第2講座（実務編）
- **想定尺**: 8〜12分（想定10分）
- **使用ツール**: NotebookLM（ブラウザ版）、Googleアカウント（撮影用サブアカウント推奨）
- **アウトプット**: 自分の学習範囲に限定した専属AI（ノートブック）と、運用サイクルの理解

## 1. 収録の目的
受講者が「自分の教材（ノート・プリント・教科書PDF）を NotebookLM に読み込ませ、自分の学習範囲に限定した専属AIを起動し、質問して回答を得るところまでを一人で実行できる」状態になること。

## 2. 事前準備（撮影前にすべて完了させること）

### 2-1. 環境
- **画面解像度**: 1920×1080（フルHD）
- **デスクトップ**: タスクバー / Dock / 通知センター / デスクトップアイコンが**映らない**ように設定する
- **通知**: 集中モード / おやすみモード ON（すべての通知を完全停止）
- **ブラウザ**: シークレットウィンドウ推奨（ブックマークバー・拡張機能アイコン非表示）
- **ウィンドウサイズ**: 録画範囲いっぱいに最大化

### 2-2. 開いておくタブ・アプリ
- NotebookLMのトップ画面（未ログイン状態、またはログイン直後のホーム画面）
- 不要なタブはすべて閉じておくこと

### 2-3. ダミーデータ
以下の3点のPDFファイルをローカル（デスクトップ等すぐ開ける場所）に用意しておくこと。
- **授業ノート2冊のPDF**（数学Ⅱ・微分の単元）
- **先生配布のプリントのPDF**（微分・1単元分）
- **参考書の該当章PDF**（微分）

### 2-4. 映してはいけないもの（NG事項）
- デスクトップバー / タスクバー / Dock
- ブックマークバー、拡張機能アイコン
- **教材PDF内の個人名・学校名・友人名・成績情報・個人連絡先**（必ず事前に黒塗り等でマスク処理しておくこと）
- 他のアプリの通知ポップアップ
- 撮影者のプライベートなGoogleアカウント情報（撮影用サブアカウントの使用を推奨）

## 3. Phase構成（本編）

> 詳細は同梱の CSV 指示書を参照。
> Markdown側では操作のフェーズごとに区切り、補足が必要な部分のみ説明を加える。

### Phase 0: Opening
- **狙い**: 実践への切り出しと、前講座の思想的な接続、ゴール宣言。
- **補足**: 操作は行わず、NotebookLMの画面を開いた状態で語りかける。

### Phase 1: ログイン＆新規ノートブック作成
- **狙い**: サービスへのアクセスから自分の作業領域を作るまでの動線を見せる。
- **補足**: Googleの2段階認証が発生する可能性に触れ、受講者のつまずきを先回りしてケアする。

### Phase 2: ソース追加
- **狙い**: 「一次情報を入れる」という思想を実際の操作で体感させる。
- **補足**: PDFをアップロードする際、読み込みの待ち時間が発生するため、その間にソース選びの重要性（AIを迷わせないこと）を語る。

### Phase 3: 自分の解き方で質問
- **狙い**: プロンプトを打ち込み、自分のノートを根拠にした回答が返ってくる安心感を見せる。
- **補足**: 回答が出たら、必ず引用スニペット（番号）をクリックし、元ソースがハイライトされる挙動を見せて「根拠がわかる強み」を強調する。

### Phase 4: 範囲外検知の体感
- **狙い**: あえて学習範囲外（積分）を聞き、「資料にありません」と返されることで、ノイズが混ざらない強みを実感させる。
- **補足**: ハルシネーションを抑制する効果を視覚的に証明する重要なシーン。

### Phase 5: 運用サイクル
- **狙い**: 単発利用ではなく、新単元の追加や古いソースの削除といった継続運用のイメージを持たせる。

### Phase 6: Ending
- **狙い**: 完成画面を映したまま、できるようになったことを振り返り、次SET（Gems編）への期待感を高める。

## 4. 収録時の話し方ルール
- 「です・ます」調を全文で徹底する。
- 一人称は「私」、二人称は「みなさん」で統一。
- 体言止めの連発を避ける。
- 歓迎演出（カメラ目線での挨拶、「ようこそ」など）は行わない。
- 操作の説明だけでなく「ここは迷いやすいので一緒に考えましょう」「はい、出ましたね」など、感情の言葉や共感を適度に混ぜる。
