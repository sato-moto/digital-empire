## 台本の語り口（厳守）

`practice_video_guideline.md §3-5` 準拠：

- **「です・ます」調を全文で徹底**（1文でも常体が混ざると NG）
- 一人称「私」、二人称「みなさん」（「あなた」「視聴者の方」NG）
- 体言止めの連発禁止（3行以上連続禁止）
- 「だから→なので／だからこそ」「でも→ただ／ですが」「〜けど→〜ですが」等の置換
- **カメラ目線／会釈／「ようこそ」等の歓迎演出禁止**。画面は常に操作対象が映っている想定
- 必須構成要素（§3-5-2）：
  1. 実践への切り出し（CSV1行目）
  2. 前講座との接続（ただし「教養編で」「1本目で」の講座番号は使わない。「前回までに〜を学んでいただきましたね」のニュートラル接続）
  3. 今回のゴール宣言
  4. Phase間のブリッジ
  5. 受講者への声かけ（「ここは迷いやすいので一緒に考えましょう」「手元で同じようにやってみてください」等）
  6. 間・相づち
  7. 振り返り（エンディング前）
  8. ねぎらい（エンディング）
  9. 次への接続（末尾）

## Phase構成（6フェーズ＋Opening/Ending・推奨）

| # | Phase | 概要 | 目安尺 |
|---|---|---|---|
| 0 | Opening | 画面に開いた状態の NotebookLM 新規画面を見せたまま、実践への切り出し＋前講座接続（設計思想の復習）＋ゴール宣言 | 1分 |
| 1 | ログイン＆新規ノートブック作成 | ブラウザでNotebookLMにアクセス、Googleアカウントログイン、新しいノートブック作成 | 1〜1.5分 |
| 2 | ソース追加 | 「+追加」ボタンから授業ノートPDF→配布プリントPDF→参考書該当章PDF の3点をアップロード。ソース名・ページ数を確認 | 2〜2.5分 |
| 3 | 自分の解き方で質問 | 3-A：ノート＋プリントの解き方に絞ったプロンプト入力→回答確認→引用スニペット確認（ノートのどこが根拠か） | 2〜3分 |
| 4 | 範囲外検知の体感 | あえて範囲外の質問（積分）→「資料にありません」挙動を見る→ノイズが混ざらない強みを解説 | 1〜1.5分 |
| 5 | 運用サイクル | 新単元ノート追加の手順（ソース追加のみを見せる）＋ 不要ソース削除手順（テスト後の整理） | 1〜1.5分 |
| 6 | Ending | 完成画面を映しながら、できるようになったこと3〜4点の振り返り＋ねぎらい＋次SET（Gems編）への橋渡し | 1分 |

Phase分けは上記の推奨に準拠するが、尺配分は制作者判断で微調整可。

## サンプルプロンプト（Phase 3〜4で使用）

### Phase 3-A：概念理解（解法統一）
```
アップロードした授業ノート2冊と配布プリントの情報だけを使って、
微分係数の定義を教えてください。
教科書以外の難しい定理（ロピタルの定理など）は絶対に使わないでください。
ノートに書かれている先生の板書の解説手順に沿って説明してください。
```

### Phase 3-B（任意・尺に余裕があれば）：つまずき抽出
```
この参考書の該当章PDFと私のノートを比較して、
私が授業ノートで書き漏らしている重要な公式や、
この単元で一番つまずきやすいポイントを3つ挙げてください。
それぞれ、どの資料のどの内容が根拠かも示してください。
```

### Phase 4：範囲外検知
```
このノートの解き方を使って、積分を用いた体積の求め方を教えてください。
ノートに記載がない場合は「資料にありません」と明示してください。
一般知識で補わないでください。
```

## 出力形式（**厳守**）

以下の形式で、**1つの出力内にMarkdown指示書パートとCSVパートの両方を含めて**ください。パート間は `<<<CSV_SEPARATOR>>>` で明確に区切ってください。

```

（practice_video_guideline.md §2 テンプレートに完全準拠）
# 実践動画 指示書：BOT-02_v5-b - 自分の学習Botを作ってみる
## 0. メタ情報
...
## 1. 収録の目的
...
## 2. 事前準備
### 2-1. 環境
### 2-2. 開いておくタブ・アプリ
### 2-3. ダミーデータ
### 2-4. 映してはいけないもの（NG事項）
## 3. Phase構成（本編）
### Phase 0: Opening
### Phase 1: ...
（省略）
## 4. 収録時の話し方ルール
## 5. 撮影後チェックリスト

<<<CSV_SEPARATOR>>>

操作内容,台本
"（1行目：画面を見せたままの切り出し）","（ここからは実践に入っていきます。前回までに〜）"
...
```

### CSV要件
- **ヘッダ行**：`操作内容,台本`（2カラム固定）
- **エンコード**：UTF-8 with BOM 前提（出力はそのまま書けばよい）
- **区切り**：カンマ、二重引用符でクォート、セル内改行は `\n`
- **操作内容**：画面位置・クリック箇所・入力文字列を**一言一句**記述（プロンプト本文はまるごとコピペ可能な形で）
- **台本**：1セル1ショット感覚、40字以内目安、敬語統一
- **行数**：25〜40行程度を目安（尺10分、1行10〜20秒換算）
