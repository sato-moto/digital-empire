## 0. メタ情報
- **講座**: SHIFT AI講座 / BOT-02_v5-b / 自分の学習Botを作ってみる（実演）
- **対応章**: SET1「AIに教材を持たせる（NotebookLM編）」の実演パート
- **想定尺**: 8〜12分（想定10分）
- **使用ツール**:
  - NotebookLM（PCブラウザ版）
  - Googleアカウント（撮影用サブアカウント推奨）
  - PDF教材3点
- **アウトプット**:
  1. 数学Ⅱ「微分」専用のNotebookLMノートブック
  2. 授業ノート2冊・配布プリント・参考書該当章PDFを読み込んだ学習環境
  3. 自分のノートの解き方で質問できる状態
  4. 新単元追加・不要ソース削除の基本運用イメージ

## 1. 収録の目的
- みなさんが、NotebookLMに自分の教材を読み込ませて、範囲を絞った学習Botを立ち上げる流れを一人で再現できるようにします。
- 特に、**ログイン → 新規ノートブック作成 → ソース追加 → 質問 → 引用確認 → 範囲外検知 → 運用サイクル**までを通しで見せます。
- 汎用AIに丸投げするのではなく、**先に教材を渡してから質問する**という発想を、手の動きとして定着させます。

## 2. 事前準備

### 2-1. 環境
- **画面解像度**: 1920×1080（フルHD）に設定
- **デスクトップ**: タスクバー / Dock / 通知 / デスクトップアイコンが映らないように設定
  - Windows: タスクバー自動非表示 ON
  - Mac: Dock自動非表示 ON、必要に応じてメニューバーも隠す
- **通知**: 集中モード / おやすみモード ON
- **ブラウザ**: シークレットウィンドウ推奨（ブックマークバー・拡張機能アイコン非表示）
- **ウィンドウサイズ**: ブラウザを最大化して録画範囲いっぱいに表示
- **推奨端末**: PC版で収録すること。資料一覧と対話欄を同時に見せやすいためです
- **UI確認**: 収録直前にNotebookLMの最新UIを確認し、「新規作成」「追加」「削除」などの表記差分を確認すること

### 2-2. 開いておくタブ・アプリ
- シークレットウィンドウでNotebookLMのトップ画面、または新規ノートブック作成直前の画面
- ファイル選択ですぐ選べる場所に、以下のPDFを保存しておく
  1. `数学II_授業ノート1_微分.pdf`
  2. `数学II_授業ノート2_微分.pdf`
  3. `数学II_配布プリント_微分.pdf`
  4. `数学II_参考書_微分_該当章.pdf`
  5. `数学II_授業ノート_積分_予告.pdf`（Phase 5 の追加実演用。任意で短いダミーPDF）
- 不要なタブはすべて閉じる
- ファイルエクスプローラー / Finder は開かず、ファイル選択ダイアログ内で完結できるよう配置しておく

### 2-3. ダミーデータ
- 個人情報・学校情報を除去した教材PDFを事前に用意する
- 今回の一貫ケースは以下で固定する
  - **主体**: 高校2年生・進学志望・自学自習型
  - **科目**: 数学（高校数学Ⅱ）
  - **単元**: 微分（微分係数の定義・導関数）
  - **教材**:
    - 授業ノート2冊（数学Ⅱ）
    - 先生配布のプリント（微分・1単元分）
    - 参考書の該当章PDF
- PDFは文字が読める解像度で作成する
- ページ数が分かるようにしておくと確認シーンが撮りやすい
- 推奨ファイル名
  - `数学II_授業ノート1_微分.pdf`
  - `数学II_授業ノート2_微分.pdf`
  - `数学II_配布プリント_微分.pdf`
  - `数学II_参考書_微分_該当章.pdf`
  - `数学II_授業ノート_積分_予告.pdf`

### 2-4. 映してはいけないもの（NG事項）
- タスクバー / Dock / 通知バナー / デスクトップアイコン
- ブックマークバー、拡張機能アイコン
- Googleアカウントの個人情報が分かる画面
- 教材PDF内の**個人名・学校名・友人名・成績情報・個人連絡先**

## 3. Phase構成（本編）

> 詳細な操作と台本は同梱のCSVを参照してください。  
> ここでは、フェーズごとの狙い・操作の流れ・補足をまとめます。

### Phase 0：Opening
- **狙い**: 実践への切り出し、前回までの思想との接続、今回のゴール共有
- **操作の流れ**:
  1. NotebookLMの新規画面を開いたまま操作せずに導入する
  2. 前回までに学んだ「AIに何を持たせるかが大事」という考え方を短く想起させる
  3. 今日のゴールを「数学Ⅱの微分専用Botを作って質問できる状態」と明示する
- **補足**:
  - 歓迎演出は入れません
  - 画面は常にNotebookLMを映したまま進めます
  - 尺は約1分を目安にします

### Phase 1：ログイン＆新規ノートブック作成
- **狙い**: 入口でつまずかず、最初の器を作れるようにする
- **操作の流れ**:
  1. NotebookLMにアクセスする
  2. 必要ならGoogleアカウントでログインする
  3. 新規ノートブックを作成する
  4. ノートブック名を数学Ⅱの微分用として分かりやすく設定する
- **補足**:
  - 2段階認証が出る場合は、編集でカットしやすいよう短く言及しておくと親切です
  - UI差分があるため、位置ではなく機能名で説明してください
  - ノートブック名は運用しやすさを意識して、単元が分かる形にします

### Phase 2：ソース追加
- **狙い**: 「何を入れるか」を画面で具体化し、3点セットの優先順位を定着させる
- **操作の流れ**:
  1. ソース追加メニューを開く
  2. 授業ノートPDF2冊をアップロードする
  3. 配布プリントPDFをアップロードする
  4. 参考書の該当章PDFをアップロードする
  5. ソース一覧で名称とページ数を確認する
- **補足**:
  - ノイズを増やさず、今の単元に絞る意図を台本で必ず言語化します
  - 文字が読みにくいPDFは精度に影響するので、可読性の話を短く補足してください
  - 収録では「追加待ち」の空白が長い場合、編集で短縮しやすいよう適度に相づちを入れます

### Phase 3：自分の解き方で質問
- **狙い**: 単に答えを返すだけではなく、**自分のノートの解き方で答える**感覚を見せる
- **操作の流れ**:
  1. チャット欄に以下のプロンプトを入力する  
     ```
     アップロードした授業ノート2冊と配布プリントの情報だけを使って、
     微分係数の定義を教えてください。
     教科書以外の難しい定理（ロピタルの定理など）は絶対に使わないでください。
     ノートに書かれている先生の板書の解説手順に沿って説明してください。
     ```
  2. 回答を確認する
  3. 引用スニペットを開き、どのノートやプリントが根拠かを確認する
  4. 必要に応じて「この答えの根拠」を視覚的に示す
- **補足**:
  - 根拠追跡性がこのツールの強みなので、引用確認は必ず見せます
  - 参考書も読み込んでいても、プロンプトでノートとプリントに絞れることを見せるのがポイントです
  - 尺に余裕があれば、比較・つまずき抽出の任意プロンプトを追加しても構いません

### Phase 4：範囲外検知の体感
- **狙い**: 「資料にありません」と返す価値を、実際の挙動で実感させる
- **操作の流れ**:
  1. 範囲外の質問として積分の内容を入力する  
     ```
     このノートの解き方を使って、積分を用いた体積の求め方を教えてください。
     ノートに記載がない場合は「資料にありません」と明示してください。
     一般知識で補わないでください。
     ```
  2. 回答を確認する
  3. 資料にない内容を無理に埋めないことの価値を説明する
- **補足**:
  - 実際の挙動は完全固定ではありません
  - そのため、プロンプトで「一般知識で補わないでください」と明示する運用の大切さを伝えてください
  - 「答えないこと」が弱みではなく、学習ではむしろ強みであると整理します

### Phase 5：運用サイクル
- **狙い**: 単発の実演で終わらせず、継続運用の基本を見せる
- **操作の流れ**:
  1. 新単元用のノートPDFを追加する動きを見せる
  2. テスト後に不要になった古いソースを削除する動きを見せる
  3. 単元ごとに追加・整理する考え方を共有する
- **補足**:
  - ここでは詳細な質問実演までは不要です
  - 「新単元を足す」「古い資料を外す」の2つを短く見せれば十分です
  - ノートブックを単元ごとに分ける運用も口頭で補足可能です

### Phase 6：Ending
- **狙い**: 今日できるようになったことを整理し、次の学習導線につなぐ
- **操作の流れ**:
  1. 完成したノートブック画面を見せる
  2. 今日の振り返りを3〜4点でまとめる
  3. ねぎらいを入れる
  4. 次のSETへの接続を入れる
- **補足**:
  - 次への接続は、「教材を持たせた次は、使う先生の話し方や人格を整える」という橋渡しにします
  - 断定的な効果表現は避け、「復習しやすくなる」「説明のブレを減らしやすい」と表現してください

## 4. 収録時の話し方ルール
- **です・ます調を全文で徹底**してください
- 一人称は**私**、二人称は**みなさん**で統一してください
- 1文は短めにし、40字以内を目安にしてください
- 各フェーズ冒頭で、**次に何をするか予告**してください
- 操作後は、**何ができたか確認**してください
- 入力中の沈黙は問題ありません。入力後に1拍置いてから説明してください
- 操作説明だけでなく、感情の言葉を混ぜてください  
  例: 「では入力します。……はい、出ましたね。ここがポイントです」
- 受講者への声かけを適度に入れてください  
  例: 「ここは迷いやすいので、一緒に見ていきましょう」
- 初出の専門用語は1回だけ言い換えてください  
  例: 「引用スニペット、つまり根拠の参照箇所です」
- くだけた接続詞は避けてください
  - 「だから」→「なので / だからこそ」
  - 「でも」→「ただ / ですが」
  - 「〜けど」→「〜ですが」
- 常体を混ぜないでください
- カメラ目線、会釈、歓迎演出は入れないでください
