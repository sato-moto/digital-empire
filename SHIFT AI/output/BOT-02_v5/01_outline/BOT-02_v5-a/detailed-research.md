## 第1層：【事実】

### 1-1. NotebookLMの基本仕様

- ユーザーが指定したソース（PDF／テキスト／Webページ／Googleドキュメント／音声／動画書き起こし等）に**限定して**回答を生成するRAG（検索拡張生成）型AI
- 各ノートブックはソース群を独立管理する。ノートブック間でソースは共有されない
- 回答には**引用スニペット**（番号付きリンク）が表示され、クリックすると元ソースのハイライト箇所に飛ぶ
- 画面は3領域構成：**資料一覧（左）／対話（中央）／生成物・スタジオ系機能（右）**。モバイル環境ではペインが切替式になる
- ソース追加後に自動インデックス化される。追加・削除は随時可能

### 1-2. 主な機能（2025-2026時点）

- 資料からの要約・FAQ生成・学習ガイド自動生成
- 音声対話形式の自動要約（通学中等の受動学習に使える）
- マインドマップによる単元構造の可視化
- モバイル対応（PC版と比べ同時表示情報量は少ない）
- 無料プランで学生が日常利用できる範囲あり。有料プランでソース上限・生成分数等が拡張

### 1-3. 学習文脈での汎用AIの困りごと

- 汎用AI（対話型AI全般）は**学習指導要領外の定理**（例：高校数学範囲外の解法）を混ぜて説明することがある
- 汎用AIは「受講者が今どの単元まで履修しているか」「どの公式を優先すべきか」等の**文脈（コンテキスト）を保持しない**。質問のたびに長文プロンプトで条件指定が必要
- 数学学習では「最短解法／別解／大学以降の見方」が混在し、「正しいが今は使えない」回答になりやすい

### 1-4. NotebookLMの範囲外応答挙動

- ソースに該当記述がない質問には「提供されたソースには該当情報がありません」と明示する挙動がある
- ただし**完全ではない**：プロンプト設計や「部分的な記述」のケースでは一般知識で補う応答が起こりうる
- 対策：「資料にない場合はそう明示して、一般知識で補わないで」と**プロンプトで明示**する運用が有効

### 1-5. 画面フロー（収録の基本動線）

ログイン → 新規ノートブック作成 → ソース追加 → 資料読込完了 → チャットで質問 → 引用スニペット確認 → （必要に応じて）スタジオ系機能で要約・音声化

- 新規作成導線は、ホーム画面の**新規作成ボタン**に集約されている傾向
- ソース追加は、ノートブック内の**資料領域の追加ボタン**
- 実務収録では**PC版推奨**（資料確認とチャットの往復がしやすい）

## 第2層：【複数ソースで一致する論点】

### 2-1. 学生の典型的な失敗パターン

- 学生がAIで挫折する主因は「AIが嘘をつくから」ではなく、**「自分の文脈と合わないから」**
- 学校の先生独自の教え方や、自分の理解レベルにAIが寄り添えないため「使えない／余計混乱する」と判断して利用をやめる
- 共通する失敗運用：**答えだけ回収する／理解確認をしない／自分の教材と照合しない**

### 2-2. NotebookLMの本質的な強みと限界

- ハルシネーションを**強力に抑制するが、ゼロではない**
- 代償として**「Garbage In, Garbage Out」**の原則が汎用AIより顕著：ソースの質がそのまま回答品質に直結
- 価値は「正解率の高さ」より**「根拠追跡性」×「範囲限定性」**。学習での復習・間違い探し・自力再現に効く

### 2-3. ソース選びの優先順位

- **情報量が多い＝良い、ではない**。網羅的な分厚い参考書を丸ごと入れるより、**「教科書のテスト範囲の章／配布プリント／自分の授業ノート」の3点セット**のほうが精度が高い
- 一次情報（先生のプリント・自分のノート）は**学習者の現在地に最も近い**
- ノイズ資料を増やすと、回答は「情報量不足」ではなく**優先順位の混線**で質が下がる

### 2-4. ノートブック型AI × 人格設計型AIの役割分担

- ノートブック型：**教材理解・範囲内説明・根拠提示に強い**
- 人格設計型：**励まし・習慣化・雑談・長期伴走・リマインドに強い**
- 「教材を読ませる役」と「毎日声をかける役」を**分ける設計**が学習支援として相性がよい

### 2-5. 運用サイクルの価値

- 単発利用より「新単元のノートを追加、古い単元を外す」という**継続運用**が効果を最大化する
- テスト前に「その単元だけのノートブック」を作ると、試験期間の専属パートナーになる

## 第3層：【仮説・推論＝講座設計に使える示唆】

### 3-1. Whyの核：発想の転換を迫る

**核メッセージ案**：「AI選びで差がつくのではなく、**AIに何を持たせるかで差がつく**」

**Before/After の対比**：
- Before：その場でAIに聞く → 返ってきた答えに自分の教材が合わせる
- After：**先に教材を渡してからAIに聞く** → 自分の教材にAIが合わせる

**Why導入のミニ台本候補**：
> 「分からない問題をAIに聞いたら、たしかに正解は返ってくる。でも、学校で習ったやり方と違う。参考書とも違う。気がつくと答えが3つ並んで、どれを信じればいいか分からなくなる。——こんな経験、ないですか？ AIが賢すぎるから、あなたは混乱している。今欲しいのは、世界中を知ってる天才じゃない。**あなたの教科書とノートだけを見て、あなたの歩幅で教えてくれる相棒**のはずです。」

### 3-2. Whatの例え話候補（カテゴリ別・採用候補8案）

#### 仕組み説明系
1. **「世界中の図書館を暗記した天才」 vs 「渡した数冊の本だけを暗記した机上の助手」**（骨子の基調）
2. **「教科書持ち込み可のテストで隣に座ってくれる親友」**——汎用AIが「スマホ持ち込み」なら、NotebookLMは「自分のノートだけを見て一緒に答えを探す親友」
3. **「渡したカンペしか読まない超優秀なアナウンサー」**——原稿（ソース）以外はアドリブしない。だから授業から逸脱しない

#### 親近感系
4. **「学年1位の友達 vs 同じクラスで同じ授業を受けた友達」**——学年1位より、同じプリントを持ってる友達のほうが今は説明が通じる
5. **「巨大辞書 vs 付箋だらけの自分専用参考書」**

#### 差別化系
6. **「何でも屋の先生 vs 教材持ち込み制のコーチ」**
7. **「外海の波 vs 自分のいけす」**——広すぎる外海は飲まれる。自分が釣りたい魚だけ放流したいけすが安全
8. **「街全体の地図 vs 校舎の教室配置」**——汎用AIは街全体、NotebookLMは今いる校舎を完璧に知ってる

**推奨採用**：1を基調（骨子準拠）、2または4を補足で1回だけ差し込むと記憶に残りやすい。

### 3-3. 1行コピー候補（採用候補9案）

**汎用AIとの差分**：
- **「汎用AIは世界を知っている。NotebookLMは、今のあなたの教材を知っている。」**
- **「何でも答えるAIより、今の単元だけ外さないAI。」**
- **「広く賢いAIを使う前に、狭く深いAIを作る。」**

**NotebookLM × Gems の差分**：
- **「NotebookLMは教材を持たせる。Gemsは先生の人格を持たせる。」**（骨子の基調）
- **「何を知っているかはNotebookLM／どう教えるかはGems。」**
- **「資料に沿って教えるのがNotebookLM、会話のキャラを整えるのがGems。」**

**役割の対比**：
- **「教材を読む役はノートブック型、背中を押す役は人格設計型。」**
- **「教科書に強いAIと、習慣化に強いAIは別チームで使う。」**
- **「"何を知っているか"を決めるのがNotebookLM、"どう話すか"を決めるのがGems。」**

**推奨採用**：骨子基調の「教材を持たせる／人格を持たせる」をメインコピーに。Whatで「世界を知る vs 教材を知る」を1度、Howで「役割分担」を1度、計3回ほど繰り返すと定着する。

### 3-4. Howの設計思想：ソース選びの判断軸

#### 二段階モデル（受講者に伝える版：シンプル）

- **入れる（一次情報）**：授業ノート・配布プリント・先生の板書メモ・今やってる教科書の該当章だけ
- **入れない（ノイズ）**：関係ない単元／解き方が違う別参考書／去年の古いノート

#### 5軸モデル（設計者・内省用：精緻）

| 軸 | チェック項目 |
|---|---|
| 一致性 | 今の授業方針と一致しているか。記法・解き方・用語が先生とずれていないか |
| 関連性 | 今回の単元に直接関係あるか。別単元が混ざっていないか |
| 鮮度 | 最新ノートか。修正版があれば新しい方か |
| 可読性 | 文字が読めるか。PDF化・撮影が鮮明か |
| 運用性 | 後から追加・削除しやすいか。単元単位で入れ替えられるか |

**推奨運用**：教養編の How では「入れる／入れない」の2段階モデルで直感的に伝え、5軸モデルは補足スライドまたは台本ナレーションで1回だけ触れる程度に留める。

#### 避けるべき資料の具体例（Howで使える）

- 今回の単元と無関係な他単元の大量資料
- 昔の模試解き直しで、今の先生の方針と違う解法が混在するもの
- 写真をそのまま束ねた暗い画像PDF（OCRが通らない）
- 友達のノートで、自分が理解していない略記が多いもの
- 1冊まるごと入れた参考書で、範囲外が大量に含まれるもの

**台本示唆**：「AIを迷わせないことがあなたの仕事。優秀なアシスタントの机に、関係ない書類を山積みにしちゃいけない。」

### 3-5. NotebookLM × Gems マトリクス（採用候補3軸）

#### 軸案1：知識の出どころ × 関わり方（推奨）
- 横軸：知識源が**限定** ↔ **広域**
- 縦軸：**説明中心** ↔ **伴走中心**

#### 軸案2：学習範囲の固定度 × 会話自由度
- 横軸：範囲固定 ↔ 範囲可変
- 縦軸：定型支援 ↔ 自由対話

#### 軸案3：授業適合度 × モチベーション設計
- 横軸：学校授業に合わせやすい ↔ 合わせにくい
- 縦軸：やる気・習慣に効く ↔ 効かない

**推奨採用**：軸案1を基調に、スライド上の2×2マトリクスで「NotebookLM（限定×説明）」「Gems（広域×伴走）」の対比を可視化する。

### 3-6. 教養How別事例「英単語帳」の設計

**構造対応**（一貫ケース=数学との対称性）：
- 数学：教科書・参考書＝公式教材／授業ノート・プリント＝自分の現場ルール
- 英単語：単語帳＝公式教材／**自分の暗記ノート（語呂合わせ・語源メモ・例文）＝自分の現場ルール**

**汎用AIの困りごと**（英単語版）：
> 「"deduce"の覚え方を聞いたら、一般的な語源（de+duce）を長々と解説された。自分がノートに書いた『で、ジュースを推測する』という独自の語呂合わせは無視された。」

**NotebookLMでの解決プロンプト例**：
> 「このノートにある私独自の語呂合わせやイラストのイメージを使って、今日覚えるべき20単語の穴埋めテストを作って。間違えたら、私のノートの言葉を使ってヒントを出して。」

**採用文言案**：「単語の意味はどのAIでも出せる。でも、自分がどう覚えているかは、自分のノートを渡さないと分からない。」

### 3-7. 実務編サンプルプロンプト（一貫ケース=高校2年生・数学・そのまま撮影で使える）

#### ① 概念理解プロンプト（解法の統一）
```
アップロードした授業ノート2冊と配布プリントの情報だけを使って、
微分係数の定義を教えてください。
教科書以外の難しい定理（ロピタルの定理等）は絶対に使わないでください。
ノートに書かれている先生の板書の解説手順に沿って説明してください。
```

#### ② 弱点分析プロンプト（つまずき抽出）
```
この参考書の該当章PDFと私のノートを比較して、
私が授業ノートで書き漏らしている重要な公式や、
この単元で一番つまずきやすいポイントを3つ挙げてください。
それぞれ、どの資料のどの内容が根拠かも示してください。
```

#### ③ 範囲外検知プロンプト（ソース縛りを体感）
```
このノートの解き方を使って、積分を用いた体積の求め方を教えてください。
※ノートに記載がない場合は「資料にありません」と明示してください。
一般知識で補わないでください。
```
**演出意図**：「資料にありません」と返る画面を録画し、ハルシネーション抑制の証拠として見せる。

#### ④ 類題生成プロンプト（発展）
```
このプリントの記法に合わせて、例題1と同じ流れで類題を1問作ってください。
途中式を省略せずに書いてください。
```

#### ⑤ 復習カード化プロンプト（Wrap近辺で使える）
```
今回の範囲を「定義・公式・解き方・よくあるミス」の4カテゴリに分けて、
復習カード形式で整理してください。
資料の外の知識は入れないでください。
```

### 3-8. Why→What→How→Wrap の配置示唆

| パート | 配置する核メッセージ |
|---|---|
| Why | 汎用AIで"合ってるのに困る"体験を言語化 |
| What | ノートブック型AIは「知識を増やす」のではなく「参照範囲を絞る」 |
| How | 良いAIはプロンプトではなく、**ソース選びで作る** |
| Wrap | AIに勉強させるのではなく、**自分の勉強環境を設計できる人になる** |

### 3-9. 実務編Wrapの接続設計

- 運用サイクル（ソース追加・削除）の実演で終わらせず、**次SET（Gems編）**に1行で橋を架ける
- 推奨接続：「今日、教材を持たせた。次は、その教材を使う"先生"そのものを作りにいこう。」
- 熱いメッセージは「相棒トーン」で簡潔に。精神論は不要

## 第4層：【使用禁止・要注意】

### 4-1. 裏取り不十分で断定禁止

- 「NotebookLMはハルシネーションゼロ」の断定　→ 誤り（一致論点2-2）
- 「ソース外には絶対に答えない」の断定　→ 挙動上の例外あり
- 「入れた教材を100%正確に理解する」の断定
- 「すべてのPDFで同じ精度が出る」の断定
- 「モバイル版でもPC版と完全同等に操作できる」の断定
- 「Gems／Geminiは劣化版」「汎用AIは悪」の価値判断　→ 役割違いの併用が正しい語り方

### 4-2. 画面UI・機能の変動リスク

- ボタン位置・ラベル名・ペイン名称・メニュー階層は**アップデートで変わりうる**
- 台本では「左上のボタン」ではなく**「新規作成ボタン」「ソース追加メニュー」**など機能名ベースで記述
- スタジオ系機能（音声要約・マインドマップ等）の名称・対応言語・生成速度・利用回数上限は変動しうる
- 収録直前に**最新UIで再確認必須**

### 4-3. 著作権・プライバシー（Wrapまたは実務編内で触れる）

- 市販教科書・参考書のPDF化は**私的利用の範囲**に留める。他人との共有ノートブックには本文を直接入れない
- **ノートブックの共有機能**：著作物を含むノートブックを他人に共有すると著作権法違反（公衆送信権侵害等）の可能性。「共有は自作オリジナルノートのときだけ」と注意喚起
- 学校配布プリントには**個人情報・学校名・先生名・クラス名・友人の記名**が写りこみやすい。マスク推奨
- ノート画像には成績・出席情報等の学習外の個人情報が入りやすい
- 友人のノートや第三者作成資料を使う場合は、**共有・アップロードの可否**に配慮
- 学校が授業内AI利用を制限している場合は、校則・先生の指示を優先

### 4-4. 講座表現の要注意

- 「このAIを使えば成績が上がる」の断定 → **NG**
- 正しい表現：「教材に沿って復習しやすくなる」「説明のブレを減らせる」「自分専用の学習環境を作りやすくなる」
- 学校の評価への不正利用（課題の丸写し等）を助長しない。「あくまで自分が理解するための壁打ち・解説コーチ」というスタンスを前置き
