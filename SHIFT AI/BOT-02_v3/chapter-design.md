# Step 4-T3: 章ロール設計 — BOT-02_v3

## 章構成

### Ch0：冒頭あいさつ（フック）
章タイトル：AIに聞いたのに、教科書と違う答えが返ってきた経験はないか？
章ロール：認識転換（問題提起）
章ゴール：「汎用AIで勉強すると範囲外の情報が混ざって困る」という課題を自分ごとにさせ、「自分の教材だけで答えてくれるAI」という解決策の存在を認知する

### Ch1：NotebookLMとは何か（概念理解）
章タイトル：「なんでも知っている天才」と「自分の教科書だけ読んでくれる助手」の違い
章ロール：概念理解
章ゴール：NotebookLMが「渡した資料からしか答えを生成しない」ソース制御型AIであることを直感的に理解し、汎用AIとの本質的な違いを説明できる

### Ch2：何を読み込ませるか（素材の選定）
章タイトル：何を渡すかで、先生の実力が決まる
章ロール：判断基準の獲得
章ゴール：学習効果を最大化するためのソース選定基準を理解し、「何を・どう整理して」ノートブックに入れるべきか自分で判断できる

### Ch3：質問の仕方の設計
章タイトル：「教えて」で終わらせない聞き方を手に入れる
章ロール：実行手順の習得
章ゴール：目的別の質問の型（理解確認／テスト生成／比較整理）を理解し、ソースに基づいた対話の設計ができる

### Ch4：ツール使い分けと最高の連携
章タイトル：NotebookLMとGemsを組み合わせて、最強の学習環境を設計する
章ロール：応用展開
章ゴール：NotebookLM（教材理解）とGems（先生の人格設定）の役割分担を理解し、両者を組み合わせた学習環境の設計思想を持てる

### Ch5：まとめと環境のアップデート
章タイトル：自分の学習記録がそのままAIの知識になる
章ロール：統合定着
章ゴール：「新単元ではソース追加、テスト後は整理」という運用サイクルの概念を理解し、継続的に学習環境をアップデートするマインドセットを持てる

## 心理順序の検証

```
Ch0: 興味（問題提起→「自分も困ってた」→解決策の提示で好奇心喚起）
Ch1: 納得（「なぜNotebookLMなのか」の概念理解→仕組みへの納得）
Ch2: 納得→実行準備（判断基準を獲得し、実行の前提条件を整える）
Ch3: 実行（具体的な質問の型を習得→実際に使える手順の獲得）
Ch4: 応用（単体利用→複合利用への展開→より高い可能性の認識）
Ch5: 自信（全体統合→運用サイクルの確立→「自分にもできる」という確信）
```

→ 興味→納得→実行→自信の心理順序に沿っている。
