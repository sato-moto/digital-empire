## 本講座における重要概念の定義

講座内で中高生が混乱しないよう、以下の用語を定義して使用する。

| 用語 | 定義（中高生向け） |
|------|-------------------|
| 汎用AI | 世界中の膨大なデータを学習し何でも答えられるが、時に「知ったかぶり」をするAI（例：ChatGPT, 素のGemini） |
| ソース制御型AI | ユーザーが渡した資料だけを読んで答えるよう制限がかけられたAI（例：NotebookLM） |
| ハルシネーション | AIがもっともらしい嘘や事実と異なる情報を生成してしまう現象 |
| グラウンディング | AIの回答が「どの資料の・どの部分に」基づいているかを明示する仕組み |
| RAG | AIが回答を作る前に、まず指定された資料を「検索」し、その情報をもとに文章を「生成」する技術 |
| Gems | Gemini内で、特定の役割（キャラクター・指導方針）をプロンプトで事前設定できるカスタマイズ機能 |

## 1. NotebookLMの最新仕様・機能・制約（2026年4月時点）

### 主要機能
- **ソースアップロード**: PDF、Google Docs/Slides、Webページ URL、テキスト、画像（OCR対応）、CSV、EPUB、YouTube動画URL、音声ファイルに対応
- **質問応答**: アップロードしたソースのみを情報源として回答を生成（ハルシネーション抑制）。回答に引用マーク付き、元ソース該当箇所へジャンプ可能
- **Studio機能（2026年追加）**: ワンクリックでAudio/Video Overview、マインドマップ、スライドデッキ、インフォグラフィック、データテーブル、クイズ、フラッシュカードを生成
- **学習ツール**: フラッシュカード・クイズの進捗がセッション間で保存。「Got it / Missed it」マーク、シャッフル、ミスした問題の再挑戦が可能
- **Interactive Audio**: 「Join」ボタンでAudio Overviewに参加し、マイクで質問・中断が可能

### ソース制限
| プラン | ソース数上限 | 1ソースあたり上限 |
|--------|-------------|------------------|
| 無料（Standard） | 50 | 50万語 or 200MB |
| Plus | 100〜300 | 同上 |
| Pro | 300 | 同上 |
| Ultra | 600 | 同上 |

- コンテキストウィンドウ: Geminiの100万トークンフルコンテキストが全プランで利用可能
- チャット入力: 過去プロンプトの文字制限が500→10,000文字に拡張
- 言語: 180以上の地域、35以上の言語に対応（Audio Overviewは英語のみ）

### Gemini連携（2025年12月〜）
- GemにNotebookLMのノートブックを直接接続可能
- Geminiから「私の『○○』ノートブックに基づいて回答して」と指示可能
- **無料プランでは連携機能が制限される場合がある**
- 代替案: それぞれのツールを別々のタブで開いて並行利用

## 2. NotebookLMの教育活用事例・ベストプラクティス

### 中高生向け活用ガイド
- テラゴヤが「中高生と指導者向けNotebookLM学習活用ガイド」を無料配布
- 中1〜高3、主要教科（国語・社会・数学・理科・英語・地理歴史・公民）対象

### 学習での活用パターン
| パターン | 具体例 |
|----------|--------|
| 情報整理 | 授業ノート・プリント・参考書をアップロードし、要約・整理 |
| 苦手分野の特定 | AIに質問→自分の理解が曖昧な箇所を発見 |
| クイズ・フラッシュカード | ゲーミフィケーションを活かした復習（進捗保存機能あり） |
| 探究学習 | Web資料やプリントを1つのノートブックにまとめ、発表用スライド構成を班ごとに作成 |
| テスト前の振り返り | 重要ポイントだけを復習用にまとめ |
| 予想問題集の生成 | 授業ノート＋先生のプリントから「自分専用の予想問題集」を自動生成 |

### ソース選定・整理のベストプラクティス
- **最優先**: 自分の授業ノート・板書メモ・つまずきポイントのメモ（一次情報）
- **次点**: 教科書・参考書のPDF/画像（範囲限定のため）
- **避けるべき**: 関係ない資料の混入（ノイズ→回答精度低下）
- **設計思想**: 目的別にノートブックを分割（「期末テスト用」「数学第3章用」等）
- **注意**: 答えのない問題集だけでは解説不可。「問題集」＋「解説プリント」をセットで投入

## 3. NotebookLMとGemsの比較・使い分け

### 機能比較
| 機能 | NotebookLM | Gems |
|------|-----------|------|
| 教材の理解・記憶 | ◎（ソース制御で正確） | △（全知識が混ざる） |
| 先生の人格設定 | △（カスタム制約は苦手） | ◎（プロンプトで自在） |
| 教え方のルール | △（「ヒントだけ出す」等の複雑な制約は困難） | ◎（制約設定が柔軟） |
| ハルシネーション抑制 | ◎（ソース限定） | △（汎用知識が混入しうる） |
| 大量教材の横断読み込み | ◎（RAG特化） | △（安定性で劣る） |

### 設計思想：ハイブリッド運用
- **NotebookLM＝「情報整理と教材のインデックス」**: 教材の要約、重要語句の抽出、テスト問題の自動生成など資料ベースの作業
- **Gems＝「対話型のチューター」**: NotebookLMが作った要約や問題をベースに、対話を通じて教える役割
- 比喩: NotebookLMが「膨大な資料を収めた図書館」、Gemが「その本を完璧に理解して教えてくれる家庭教師」

### 連携の実態（2025年12月〜）
- GemにNotebookLMのノートブックを直接接続可能に
- NotebookLMの「正確な情報に基づいた回答」＋Gemの「柔軟なカスタマイズ」のハイブリッド
- Gem経由でGoogle Workspaceアプリとの連携も可能
- **無料プランでの制限あり**。代替案: 別々のタブで並行利用

## 4. 汎用AIの学習利用における課題

### 中高生が直面する3つの落とし穴
1. **範囲外知識の混入**: 中学生の数学の質問に高校範囲の公式（判別式、微積分等）で解説される。テストでバツになる
2. **ハルシネーションの悪影響**: 歴史の年号や英単語のニュアンスで嘘をつかれ、間違った知識を暗記してしまう
3. **思考力の低下**: すぐに完璧な「答え」を出してしまい、プロセスを学ぶ機会が奪われる

### 調査データ
- 教育委員会の81%がAI活用に「課題あり」と回答
- うち37%がハルシネーションを課題として指摘
- ハルシネーション認識率: 教育委員会全体で68.4%

### 典型的な課題パターン
| パターン | 具体例 |
|----------|--------|
| 答えだけを聞く | AIを検索エンジン代わりに使い、考えるプロセスが省略される |
| 出典不明の情報 | 汎用AIの回答には出典がなく、正誤判断ができない |
| 範囲外の説明 | 学年・単元を超えた知識で説明され、かえって混乱する |
| 学習意欲の低下 | 「AIに聞けばいい」で思考停止する |

## 5. ソース制御型AIの概念・原理（中高生向け翻訳）

### RAGの直感的理解
- **通常のAI（Gemini等）**＝「世界中の図書館を暗記している天才」→ 何でも答えられるが、聞いてもいない情報まで混ざる。時々記憶違いをする
- **NotebookLM**＝「あなたが渡した数冊の本だけを完璧に暗記して机に置いている助手」→ 渡した資料からしか答えない

### RAGの2ステップ
1. **Retrieval（検索）**: 質問されたら、まずアップロードされた教材の中から関連するページを探し出す
2. **Generation（生成）**: 見つけた情報「だけ」を使って、分かりやすい言葉で回答を作る

### グラウンディングの重要性
- 回答に引用マークが付き、元のソース該当箇所にジャンプ可能
- 「AIが言ったから正しい」ではなく「この教科書のこの部分に基づいている」と検証できる
- ファクトチェックの習慣化につながる

## 6. 中高生の学習行動・AIリテラシーの現状

### 利用率
- 中高生の約8割が生成AIの利用経験あり（2026年1月調査、1,200人対象）
- 中学生の生成AI利用は1年で倍増（2025年2月調査）
- 学習目的だけでなく「趣味」「相談相手」としての利用も拡大

### 利用方法の実態
- 「答えを聞く」辞書的な利用が主流
- プライベートでの使用（趣味・相談相手）も増加傾向
- AIを「検索エンジンの代わり」として使う層が多数

### 心理的障壁
| 障壁 | 詳細 |
|------|------|
| 思考停止への不安 | 「AIに頼ると自分で考えなくなるのでは」 |
| 信頼性の懸念 | 「AIの答えが正しいかどうか判断できない」 |
| 後ろめたさ | 「先生にAIを使ったことがバレたくない」 |
| ズルをしている罪悪感 | AIを使うこと自体に罪悪感を感じる |

### 保護者・教員の認識
- 「学習効率が上がる」という期待 vs 「考える力が育たない」という懸念の二分
- ハルシネーションによる誤学習への不安
- 個人情報の取り扱いに対する懸念
- NotebookLMの「教材限定」環境は、答えをショートカットするものではなく「教材を深く読み解くための補助線」であるという認識が重要
