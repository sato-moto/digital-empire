# ワークフロー分岐・Phase遷移ルール

## 起動時の分岐

スキル起動時に、ユーザーに以下を選択させる:

```
どちらから始めますか？

① カリキュラム設計から始める（Phase 1 → Phase 2）
   カリキュラム全体の構成（セット・講座一覧）をリサーチ→設計→壁打ちで作り、
   その後に個別の講座を制作します。

② 講座制作から始める（Phase 2 から）
   既にカリキュラム構成が決まっている、または単体の講座を作りたい場合。
   必要な情報をヒアリングしてから制作に入ります。
```

### ① を選んだ場合
→ Phase 1（Step 0）から開始。通常フロー。

### ② を選んだ場合
→ Phase 2 から開始。必要な情報をユーザーからヒアリングする。

**まず制作単位を選択させる:**

① おすすめ: Lv.1 の全講座を上から順に作る（全SET単位）
② 範囲を指定して制作する（SET単位）
③【従来】タイトル→ゴール→章→アウトラインを段階設計（outline_v1.md準拠、段階パイプライン）

**①②を選んだ場合:**
- 「既存の course-structure.md のパスを教えてください」とユーザーに聞く
- course-structure.md を読み込み → SET一覧を表示 → ユーザーが範囲選択
- `guidelines/実務_講座設計ガイドライン_2.md` を Read してSET内設計方針を決定
- Step 3（リサーチ）へ

**③を選んだ場合:** 従来パイプライン（変更なし）

### ①②（SET単位）を選んだ場合のフロー

1. course-structure.md を読み込み、SET一覧とステータスを表示
2. ①の場合: Lv.1 の全SETを上から順に選択
   ②の場合: ユーザーがSETを指定
3. `guidelines/実務_講座設計ガイドライン_2.md` を Read
4. SET内設計方針を確認・補完:
   - 一貫ケースが定義済みか → 未定義なら設計
   - 問題解決ストーリーのこのSETの位置づけ
   - 教養How用別事例の割当
   - 使用ツール
5. ユーザーに確認後、SET内の各講座をcourse-structure.mdの順で制作（Step 3 へ）

### 【従来】のヒアリング項目

従来パイプラインでは T1/T2（タイトル+ゴール一括設計）で設計するため、ヒアリングでは `outline_v1.md` の①②の出力を生成するのに必要な**素材情報**を収集する。タイトル・ゴールの完成品はヒアリングでは聞かない。

**①タイトル設計（outline_v1.md ①）に必要な情報:**

| # | 項目 | 必須 | 例 |
|---|---|---|---|
| 1 | 講座テーマ | 必須 | 「営業メールをAIで効率化する方法」 |
| 2 | 想定受講者 | 必須 | 「中小企業の営業担当」 |

**②ゴール設計（outline_v1.md ②）に必要なペルソナ情報:**

| # | 項目 | 必須 | 例 |
|---|---|---|---|
| 3 | 属性 | 必須 | 職種: 営業 / 役職: 一般社員 |
| 4 | 経験値 | 必須 | AI未経験 |
| 5 | 現在の課題 | 必須 | 「メール作成に毎回15分かかっている」 |
| 6 | 活用目的 | 必須 | 「メール業務の時短」 |
| 7 | 想定活用シーン | 必須 | 「取引先へのお礼メール作成」 |
| 8 | 学習動機 | 必須 | 「AIを使えるようになりたいが何から始めればいいか分からない」 |
| 9 | 制約 | 必須 | 「業務時間内で学びたい、専門用語は避けてほしい」 |

**コンテキスト情報:**

| # | 項目 | 必須 | 例 |
|---|---|---|---|
| 10 | 前後の講座 | 任意 | 前: 【教養】○○ / 次: 【実務】△△ |
| 11 | コース文脈 | 任意 | 副業コース or 業務効率化コース |
| 12 | 使用ツール | 任意 | ChatGPT |

ヒアリング後、メインClaude が入力情報を `{out}/lecture-brief.md` に整理し、**Step 4-T1/T2（タイトル+ゴール一括設計）から段階パイプライン**で進行する。

## Phase 2 の分岐構造

```
SET単位（①②選択時）:
  → guidelines/実務_講座設計ガイドライン_2.md でSET内設計
  → SET内の各講座を種別で自動判定:
    ├─ 教養 → guidelines/教養_講座設計ガイドライン.md → Step 3〜6（一括パイプライン）
    └─ 実務 → guidelines/実務_講座設計ガイドライン.md → Step 3〜6（一括パイプライン）

従来（③選択時）:
  └─ guidelines/outline_v1.md → 4-T1/T2 + Step 3 + 4-T3 + 4-T4 + 5-T + 6（段階パイプライン）
```

## Phase 間遷移ルール

**各 Phase が完了するたびに、必ずユーザーに確認を取ってから次の Phase に進むこと。**
**確認なしに次の Phase へ自動進行してはならない。**

| タイミング | 報告内容 | 次のアクション |
|---|---|---|
| Phase 1 完了時 | `research-summary.md` と `course-structure.md` の概要を提示 | ユーザーの承認後、推奨（Lv.1から順）or 講座指定で Phase 2 の対象範囲を決定 |
| Phase 2（1SET完了時） | SET内の全成果物を提示 | ユーザーに次アクションを確認（次SET / Phase 3 / 修正 / サムネ / 終了） |
| Phase 3（全講座完了時） | 完成した全スライド構成の講座名一覧を提示 | 講座名入り選択肢を提示（①この順でPhase 4 / ②順番変更 / ③修正 / ④終了） |
| Phase 4（全講座完了時） | 完成した全講座の一覧を提示 | 次の未着手講座名を含む選択肢を提示（①次の講座群 / ②指定 / ③新規 / ④サムネ / ⑤終了） |

## Phase 2 の進行フロー

**SET単位で進行し、1SET完了ごとにユーザーに次のアクションを確認する。**

```
Phase 2 開始
    │
    ▼
┌──────────────────────────────┐
│ SET N の制作                   │
│ （教養 → 実務 の順で Step 3〜6）│
└─────────┬────────────────────┘
          │
          ▼
    SET N の全講座が完了
    ステータス更新
          │
          ▼
    ★ ユーザーに確認:
    「SET {N} が完成しました！次にどうしますか？」
    ① 次のSETに進む（SET {N+1}）
    ② このSETの教養講座を先に Phase 3（スライド構成）に進める
    ③ 修正する
    ④ サムネイルを生成する
    ⑤ ここで終了する
          │
    ①─────┼──► 次のSETへ（Phase 2 続行）
    ②─────┼──► 教養講座のみ Phase 3 へ（実務は完了済み）
    ③─────┼──► 修正 → SET N に戻る
    ④─────┼──► /thumbnail-designer 起動 → 完了後、再度次アクション選択
    ⑤─────┴──► メモリ保存 → 終了
```

**Phase 3 への遷移ルール:**
- 教養講座 → Phase 3（スライド構成）→ Phase 4（台本+スライド生成）へ進む
- 実務講座 → 収録指示書（md + csv）で完了。Phase 3/4 は対象外
- Phase 3 への遷移は1SET完了時に選択可能（②を選んだ場合）
- 複数SETの教養講座をまとめて Phase 3 に進めることも可能

### 1SET完了時の確認メッセージ

```
SET {N}「{SETテーマ}」の全講座が完成しました！

📁 教養講座:
  1. 【教養】{講座名A} → outline-final.md ✅

📁 実務講座（収録指示書）:
  2. 【実務】{講座名B} → practice-{N}-final.md + .csv ✅

次にどうしますか？

① 次のSET（SET {N+1}「{次のSETテーマ}」）に進む
② このSETの教養講座を Phase 3（スライド構成）に進める
③ 修正する → 講座名を指定してください
④ サムネイルを生成する → 講座を指定して /thumbnail-designer を起動
⑤ ここで終了する → プロジェクト進捗を保存
```

- ④ の場合: `/thumbnail-designer` スキルを起動。対象講座をユーザーに選択させ、入力として教養講座は `output/{project}/01_outline/{講座名}/outline-final.md`、実務講座は `output/{project}/05_practice/practice-{N}-final.md` を渡す。完了後、再度次アクション選択（①〜⑤）を表示

### Phase 3 完了時の確認メッセージ

```
全{N}講座のスライド構成が完成しました！

📁 完成したスライド構成:
  1. {講座名A} → slide-design.csv ✅
  2. {講座名B} → slide-design.csv ✅
  3. {講座名C} → slide-design.csv ✅

次にどうしますか？

① この順番で Phase 4（台本+スライド生成）に進める（おすすめ）
  1. {講座名A}
  2. {講座名B}
  3. {講座名C}

② 順番や対象を変えて Phase 4 に進める
  → 講座を指定してください

③ スライド構成を修正してから進める
  → 修正したい講座名を教えてください

④ ここで終了する
  → プロジェクトの進捗を保存して終了
```

## Phase 4 完了後の次アクション選択

Phase 4（台本 + スライド生成）が **対象範囲の全講座で** 完了したら:

1. **完了した講座の結果をメモリに保存する**
2. **course-structure.md のステータスを更新する**（該当講座を「完了」に変更）
3. **ユーザーに次のアクションを選択させる:**

```
対象範囲の全{N}講座が完了しました！

📁 完成した講座:
  1. {講座名A} → スライド ✅ / 台本 ✅
  2. {講座名B} → スライド ✅ / 台本 ✅
  3. {講座名C} → スライド ✅ / 台本 ✅

次にどうしますか？

① 次の講座群を制作する（おすすめ）
   → 次の未着手講座:
     1. 【教養】{次の講座名D}
     2. 【実務】{次の講座名E}
     ...

② 講座を指定して制作する
   → course-structure.md から選択

③ 新しい講座をヒアリングから制作する
   → Phase 2 直接開始のヒアリングフローへ

④ サムネイルを生成する
   → /thumbnail-designer を起動し、講座のアウトラインからサムネイルを作成

⑤ 中高生向けにテイスト変換する（サブスキちゃんを呼ぶ）
   → /subsuki を起動し、完成した講座一式を中高生向けに着せ替える
   → 構成・台本の骨格は触らず、図解PNG・サムネ・Body/台本のビジネス語彙のみ変換
   → 出力先は output_teen/ 配下（元ファイルは保持）

⑥ ここで終了する
   → プロジェクトの進捗を保存して終了
```

- ① の場合: course-structure.md の上から順（Lv.1→Lv.2→...）で次の未着手講座群を提示し、ユーザー承認後 Phase 2 から開始
- ② の場合: course-structure.md の全講座一覧（ステータス付き）を表示し、講座を選択させる
- ③ の場合: Phase 2 直接開始のヒアリングフローへ
- ④ の場合: `/thumbnail-designer` スキルを起動。対象講座をユーザーに選択させ、入力として教養講座は `output/{project}/01_outline/{講座名}/outline-final.md`、実務講座は `output/{project}/05_practice/practice-{N}-final.md` を渡す。完了後、再度次アクション選択（①〜⑥）を表示
- ⑤ の場合: `/subsuki` スキルを起動。カレントを講座フォルダ（`SHIFT AI/{講座ID}/`）として、サブスキちゃんが一気通貫で中高生向けテイスト変換を実行。完了後、再度次アクション選択（①〜⑥）を表示
- ⑥ の場合: 進捗をメモリに保存し、次回会話で再開できるようにする
