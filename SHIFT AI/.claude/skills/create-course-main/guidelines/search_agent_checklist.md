# リサーチエージェント チェックリスト

> **目的**: 講座テーマについてWeb検索で情報を収集し、「講座判断材料」としてのリサーチレポートを作成するためのチェックリスト。
> 目的は情報収集ではなく、収集した情報を講座設計に使える材料へ変換すること。
> （参照: `0_search/report_v1.txt` ガイドライン）

---

## 📌 使い方

1. 講座テーマ（仮タイトル）を入力として受け取る
2. STEP①で調査の目的・論点を明確にする
3. STEP②で6観点の情報を収集する
4. STEP③で収集した情報を講座設計観点へ変換する
5. STEP④でレポートの品質を最終確認する

---

## STEP① 調査準備（テーマ理解と論点設定）

### 1-A. 調査目的の明確化

- [ ] この調査が何を判断するためのものか明文化した（新規企画/改善/競合比較 等）
- [ ] 「講座として作るべきか」「誰向けに作るべきか」「どの形式で作るべきか」のうち、今回確認すべき問いを特定した
- [ ] 調査テーマを「講座設計の何を判断するための調査か」の形で記述した

### 1-B. 対象者・分類の仮説設定

- [ ] 想定受講者（初心者/中級者/実務者）の仮説を設定した
- [ ] 講座分類の仮説を設定した（教養・知識系/実務スキル系/ツール活用系）
- [ ] 仮説に基づいて検索方針（見るべき情報の優先順位）を決めた

### 1-D. ターゲットクラスターの特定

- [ ] クラスター分析サマリー（`ref/cluster-summary.md`）を読み込んだ
- [ ] この講座の主対象クラスター（1〜3つ）を特定した
- [ ] 各ターゲットクラスターの入会動機・課題・目標金額を把握した
- [ ] ターゲットクラスターの特徴に基づき、検索方針を調整した
- [ ] ターゲットクラスターが不安・未定層（C3/C4）を含む場合、方向性提示の必要性を認識した

### 1-C. 主要論点の設定

- [ ] 市場性（需要はあるか、一時的か継続的か）を論点に含めた
- [ ] 企業ニーズ（現場で求められているか）を論点に含めた
- [ ] 学習者ニーズ（何に困っているか）を論点に含めた
- [ ] 競合状況（既存講座と差別化できるか）を論点に含めた
- [ ] 教育化しやすさ（体系的に教えられるか）を論点に含めた
- [ ] 継続性（長く使える内容か、頻繁な更新が必要か）を論点に含めた

---

## STEP② 情報収集（6観点の調査）

### 2-A. 市場動向

- [ ] テーマが社会的・業界的に拡大しているかを確認した
- [ ] 流行の有無だけでなく、どの層で重要性が増しているかを確認した
- [ ] テーマの検索ボリューム傾向（上昇/下降/安定）を確認した
- [ ] 関連する業界レポート・統計データを1件以上取得した
- [ ] 一時的な話題性か、継続的な教育価値があるかを見極めた

### 2-B. 企業ニーズ

- [ ] 求人情報からテーマに関連するスキル要件を3件以上確認した
- [ ] 企業の導入事例・活用事例を3件以上収集した
- [ ] 業界記事から現場で求められているスキル・知識を特定した
- [ ] 企業がテーマに関連してどのような成果を出しているかを確認した

### 2-C. 学習者ニーズ

- [ ] 想定受講者が使う検索キーワードを10個以上抽出した
- [ ] Q&Aサイトでの質問傾向を5件以上収集した
- [ ] 受講者の「悩み・課題」を受講者自身の言葉で5つ以上集めた
- [ ] 受講者のよくある誤解・思い込みを3つ以上収集した
- [ ] 受講者の学習動機パターンを3つ以上整理した
- [ ] 受講者がよく訪れるメディア・コミュニティを3つ以上特定した
- [ ] 受講者の「過去に試したが失敗した方法」を3つ以上収集した
- [ ] ターゲットクラスターの入会動機と講座テーマの関連性を分析した
- [ ] ターゲットクラスターの「目標金額」と講座の成果目標の整合性を確認した
- [ ] クラスター固有の不安・課題（例: C3/C4の将来不安、C5の時間制約）を調査に反映した

### 2-D. 競合講座・類似コンテンツ

- [ ] 同テーマの既存講座・セミナーを5件以上リストアップした
- [ ] 各競合講座のタイトル表現パターンを記録した
- [ ] 各競合講座の対象者（誰向けか）を記録した
- [ ] 各競合講座の価格帯・受講形態を記録した
- [ ] 競合講座のレビュー・評判を3件以上確認した
- [ ] 同テーマの書籍の目次構成を3冊以上確認した
- [ ] 競合講座が不足している領域（差別化余地）を特定した

### 2-E. 技術・ツール・概念の変化

- [ ] テーマに関連する最新ニュース・トレンドを3件以上収集した
- [ ] 概念講座で長く使える内容か、頻繁な更新が必要かを見極めた
- [ ] テーマの知識体系における「必須概念」をリストアップした
- [ ] 前提知識は何が必要かを調査した

### 2-F. 実務への接続情報

- [ ] テーマが実際にどの業務で使われているかを5つ以上確認した
- [ ] どのような成果につながるかの事例を3件以上収集した
- [ ] テーマに関連するテンプレート・フレームワークを3つ以上収集した
- [ ] 学んだことの実務活用の再現性を確認した

### 2-G. 実務テーマ・ロール・プロンプト調査（実務講座の場合のみ）

- [ ] ロール候補を5パターン以上特定し、日常業務・課題・AI経験を調査した
- [ ] 各ロールのミッション・成果物を調査した
- [ ] 実務テーマ別のBefore/After数値（時間・コスト等）を収集した
- [ ] AIで効率化できる具体的な業務を10以上リストアップした
- [ ] サンプルプロンプト素材（実際の業務プロンプト事例）を5件以上収集した
- [ ] コース文脈（副業/業務効率化）の判断材料を整理した
- [ ] 1動画=1テーマの分割候補を洗い出した
- [ ] 各テーマで使用するツール・機能（1-2個）を特定した
- [ ] 「やりたいこと→ツール活用」の目的起点ユースケースを10以上整理した

---

## STEP③ 講座設計観点への変換（情報の翻訳）

> 収集した情報をそのまま並べず、講座設計の観点で再整理・変換する。
> この変換がないと、レポートは「調査メモ」の域を出ない。

### 3-A. 6つの変換を実施

- [ ] 社会トレンド → 教育テーマへ変換した（「話題」ではなく「学ぶ価値」として再定義）
- [ ] 企業ニーズ → 学習項目へ変換した（「現場で必要」→「何を教えるべきか」へ翻訳）
- [ ] 受講者の悩み → 講座の学習目的へ変換した（「困っている」→「講座で何を解決するか」）
- [ ] 競合講座の特徴 → 自社講座の差別化論点へ変換した
- [ ] 技術情報 → 教えるべき範囲と教えなくてよい範囲の判断へ変換した
- [ ] 事例情報 → 講座内で扱うユースケース候補へ変換した

### 3-B. 講座設計の主要観点で整理

- [ ] 誰向けか（初心者/経験者/実務者）を判定した
- [ ] 何を学ばせるか（概念理解/業務理解/操作スキル/成果物作成）を判定した
- [ ] どのカテゴリか（教養・知識系/実務系/ツール系）を判定した
- [ ] どの順番で学ばせるべきか（前提知識の要否、基礎先行/ユースケース先行）を検討した
- [ ] どこに差別化余地があるかを特定した
- [ ] 更新頻度（概念中心で安定/ツール依存で要更新）を判定した

### 3-D. クラスターニーズとの整合性確認

- [ ] 収集した情報がターゲットクラスターの動機に合致しているか確認した
- [ ] 講座の訴求ポイントがクラスターの課題解決に直結しているか確認した
- [ ] 「一般的な学習者ニーズ」ではなく「自社会員の具体的ニーズ」で語れているか確認した

### 3-E. 実務講座設計観点への変換（実務講座の場合のみ）

- [ ] ロール候補をターゲットクラスターと照合し、最適ロールを推定した
- [ ] コース文脈（副業/業務効率化）の推奨を判断した
- [ ] 1講座=1テーマで講座ゴールが一文で言えるか確認した
- [ ] 動画分割候補を「やりたいこと」形式の章タイトル候補に変換した
- [ ] サンプルプロンプト素材をHowパートで使える形式に整理した
- [ ] Before/After数値を定量的な受講効果として整理した

### 3-C. 事実と解釈の分離

- [ ] 調査結果の各項目で、事実（出典あり）と解釈（分析者の見解）が明確に分かれている
- [ ] 「注目されている」「重要である」等の断定に根拠を付けた
- [ ] 数値データには出典と時点を明記した

---

## STEP④ レポート品質チェック（最終確認）

- [ ] このレポートが「何を判断するためのもの」か明確である
- [ ] 調査結果が講座設計に関係する情報に絞られている
- [ ] 市場性・企業ニーズ・学習者ニーズ・競合・教育化しやすさの観点が揃っている
- [ ] 事実と分析が明確に分かれている
- [ ] 情報を並べるだけで終わっていない（講座設計観点への変換がされている）
- [ ] 講座カテゴリ・対象者・内容方向性への示唆がある
- [ ] 「結局どうすべきか」（提案）が明確である
- [ ] 読んだ人が講座判断しやすい状態になっている
- [ ] 収集情報の出典・信頼性を確認した
- [ ] 情報の鮮度（最新性）を確認した
- [ ] ターゲットクラスターが明記されている
- [ ] クラスターの動機・課題に基づいた示唆がある
- [ ] 講座の訴求ポイントが自社会員のニーズに基づいている（一般論ではない）
- [ ] 実務講座の場合: ロール候補・Before/After数値・サンプルプロンプト素材が含まれている
- [ ] 実務講座の場合: 目的起点（機能起点ではない）で整理されている

---

## 💡 調査のコツ

- **「何のために集めるか」を常に意識する**: 情報収集自体が目的ではなく、講座判断材料を作ることが目的
- **検索クエリの工夫**: 「〇〇 初心者 悩み」「〇〇 講座 口コミ」「〇〇 導入事例」「〇〇 求人 スキル」など観点別にクエリを変える
- **情報ソースの多様化**: Google検索、X（Twitter）、YouTube、Udemy、note、Qiita、Amazon書籍レビューなど複数ソースを活用する
- **生の声を重視**: 統計データだけでなく、Q&Aサイトやレビューの「生の言葉」を収集する
- **量より質**: 件数ノルマを満たすことより、講座判断に直結する質の高い情報を優先する
- **記事要約で終わらせない**: 集めた情報は「講座を作る観点」で必ず再構成する
