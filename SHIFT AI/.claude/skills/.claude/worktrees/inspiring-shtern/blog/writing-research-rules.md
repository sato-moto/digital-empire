# 📋 プロのライティング・リサーチ ルールブック

熊取つーしんの記事制作にあたり、プロのSEOライター・地域ブログ運営者の手法をリサーチし体系化したもの。  
全記事制作時にこのルールに従うこと。

---

## 🔑 大原則

> **リサーチは記事作成時間の60〜70%を占めるべき。**  
> 丁寧なリサーチが記事の質・執筆速度・SEO評価のすべてを決める。

---

## Phase 1: キーワード設計（所要時間目安: 10分/記事）

### 1-1. メインKWの確認
- [ ] 固定KW「熊取」「熊取町」がタイトル・h1に含まれているか
- [ ] ターゲットKWの組み合わせが明確か（例: 「熊取 ランチ おすすめ」）

### 1-2. 関連KW・サジェストの調査
- [ ] Googleサジェスト（検索窓に入力して出る候補）を記録
- [ ] 「再検索ワード」（検索結果下部の関連検索）を記録
- [ ] ラッコキーワードで関連語を網羅的に収集
- [ ] 共起語（本文中に自然と出現すべき語）をリストアップ

### 1-3. 検索ボリューム確認
- [ ] Googleキーワードプランナー or Ubersuggestで月間検索数を確認
- [ ] ビッグ（競合強）・ミドル・ロングテール（競合弱・CV高）の分類
- [ ] ドメインパワーが弱いうちはロングテールから優先

---

## Phase 2: 検索意図の深掘り（所要時間目安: 15分/記事）

### 2-1. 検索意図の4分類で整理
| 分類 | ユーザーの目的 | 例（熊取系） |
|------|-------------|-------------|
| **Know（知りたい）** | 情報を知りたい | 「熊取町 歴史」「熊取 人口」 |
| **Do（やりたい）** | 行動を起こしたい | 「熊取 ハイキング コース」 |
| **Go（行きたい）** | 特定の場所に行きたい | 「永楽ダム アクセス」 |
| **Buy（買いたい）** | 購入・予約したい | 「熊取 不動産 相場」 |

### 2-2. ペルソナ設計（1記事1ペルソナ）
各記事で以下を明確にする：
- [ ] **誰？** → 年齢層・性別・家族構成（例: 30代・子育てママ）
- [ ] **どんな状況？** → 引越し検討中？観光計画中？日常の情報探し？
- [ ] **何を知りたい？** → 顕在ニーズ（直接の質問）
- [ ] **本当に欲しいものは？** → 潜在ニーズ（根本的な欲求）
- [ ] **知識レベルは？** → 初心者？地元民？専門家？

### 2-3. 読者のリアルな声を調査
- [ ] **Yahoo!知恵袋** で関連質問を3〜5件チェック
- [ ] **X（Twitter）** で「熊取 ○○」を検索しリアルな声を拾う
- [ ] **Googleの「他の人はこちらも質問」** をすべて記録
- [ ] **Instagram** で「#熊取」の投稿傾向を確認

---

## Phase 3: 競合分析（所要時間目安: 20分/記事）

### 3-1. 上位記事の構造分析
- [ ] ターゲットKWでGoogle検索し、**上位5〜10本**を開く
- [ ] 各記事の以下を記録：

| 分析項目 | チェック内容 |
|---------|-----------|
| タイトル | 文字数・KWの位置・訴求ポイント |
| 見出し構成 | h2/h3の数と内容をリストアップ |
| 文字数 | 大まかな文字数を把握 |
| 独自情報 | 写真・体験談・独自データの有無 |
| 内部リンク | どこにリンクしているか |
| CTA | 読者に何を促しているか |
| 情報の鮮度 | 最終更新日はいつか |

### 3-2. 差別化ポイントの発見
- [ ] 上位記事に**共通して含まれている情報**（=必須要素）を整理
- [ ] 上位記事に**欠けている情報**（=差別化チャンス）を特定
- [ ] 「熊取つーしん」ならではの**独自の強み**を1つ以上設定
  - 例: 地元住民の実体験、現地取材写真、運営者の個人的おすすめ

### 3-3. 勝ち筋の判断
- [ ] 上位サイトのドメインパワーを確認（強すぎる場合はKW変更を検討）
- [ ] 自サイトが勝てる角度を決定（網羅性 / 専門性 / 鮮度 / 独自性）

---

## Phase 4: 情報収集（所要時間目安: 20分/記事）

### 4-1. 信頼できる情報源の選定ルール

| 優先度 | 情報源 | 用途 |
|-------|-------|-----|
| ★★★ | 公的機関（熊取町公式サイト、大阪府、統計データ） | 数値・制度・公式情報 |
| ★★★ | 現地取材・実体験 | 独自性・E-E-A-T強化 |
| ★★☆ | 公式プレスリリース・ニュース記事 | 最新情報・トレンド |
| ★★☆ | 専門書籍・学術論文 | 歴史・文化の深掘り |
| ★☆☆ | 個人ブログ・SNS投稿 | 口コミ・体験談の参考 |
| ☆☆☆ | Wikipedia（裏取り必須） | 概要把握のみ |

### 4-2. 地域ブログ特有の情報収集
- [ ] **熊取町公式サイト**（kumatori.lg.jp）で最新情報を確認
- [ ] **くまとりにぎわい観光協会**のサイトで観光・イベント情報
- [ ] **広報くまとり**（町の広報誌）で行政情報
- [ ] **Googleマップ**でスポットの営業時間・口コミ・写真を確認
- [ ] **食べログ / ホットペッパー** でグルメ系の口コミ・メニュー情報
- [ ] **求人情報誌** で新規オープン店舗の手がかり
- [ ] **Googleアラート** で「熊取」を登録し継続的に情報収集

### 4-3. 情報収集の時間制限ルール
> ⚠️ **1記事あたりの情報収集は最大30分。**  
> 完璧を求めず、80%集まったら次のPhaseへ進む。  
> 不足分は執筆中に補足リサーチで補う。

---

## Phase 5: 構成案の作成（所要時間目安: 15分/記事）

### 5-1. 仮説ファースト
- [ ] **記事のゴール（結論）を最初に決める**
  - 「この記事を読んだ人に何をしてもらいたいか」を1文で書く
- [ ] 結論を支える根拠や情報の仮説を3〜5個用意

### 5-2. 構成案テンプレート

```
【タイトル】32文字前後 / KW前方配置 / 数字 or 感情ワード入り
【メタディスクリプション】120文字前後 / 検索意図に合致する要約

---

■ リード文（150〜200文字）
  - 読者の悩みに共感 → 記事で解決できることを提示 → 読む動機を作る

■ h2: [セクション1のテーマ]
  └ h3: [サブトピック]
  └ h3: [サブトピック]

■ h2: [セクション2のテーマ]
  └ h3: [サブトピック]
  └ h3: [サブトピック]

■ h2: [セクション3のテーマ]
  └ h3: [サブトピック]

■ h2: まとめ
  - 記事の要点を3行で整理
  - 読者の次の行動を提案

■ 関連記事リンク（2〜3本）
```

### 5-3. 構成チェック
- [ ] **MECE**（漏れなくダブりなく）になっているか
- [ ] 上位記事の**必須要素**をすべてカバーしているか
- [ ] **差別化要素**が1つ以上含まれているか
- [ ] 読者が**上から順に読んで自然な流れ**になっているか

---

## Phase 6: 本文執筆ルール（所要時間目安: 40分/記事）

### 6-1. ライティングの基本原則
- [ ] **結論ファースト**（PREP法: 結論→理由→具体例→再結論）
- [ ] **一文は60文字以内** を目安に短く
- [ ] **一段落は3〜4行まで** で改行
- [ ] **専門用語は必ず解説** を添える
- [ ] **具体的な数字・データ** を積極的に使う

### 6-2. SEOライティングルール
- [ ] タイトルに「熊取」を含める（前方配置推奨）
- [ ] h2に「熊取」を2〜3回自然に含める
- [ ] 本文中にメインKWを**1,000文字あたり2〜3回**（自然な頻度で）
- [ ] 共起語を本文全体に散りばめる
- [ ] 画像にはKWを含むALTテキストを設定
- [ ] 内部リンクを2〜3本設置

### 6-3. E-E-A-T強化ルール（Google評価基準）
| 要素 | 具体的な施策 |
|------|-----------|
| **Experience（経験）** | 「実際に行ってみた」「食べてみた」等の体験ベース表現 |
| **Expertise（専門性）** | 正確なデータ引用、専門的な切り口 |
| **Authoritativeness（権威性）** | 運営者プロフィール、取材実績の明示 |
| **Trustworthiness（信頼性）** | 情報源の明記、最終更新日の記載 |

### 6-4. 地域ブログ特有の執筆ルール
- [ ] **住所・電話番号・営業時間** は最新情報を確認して記載
- [ ] **アクセス方法** は車・電車・バスの3パターンで記載
- [ ] **地図の埋め込み** またはGoogleマップへのリンクを設置
- [ ] **季節限定情報** には時期を明記（例:「桜の見頃は3月下旬〜4月上旬」）
- [ ] **口コミ・評判** を引用する場合は出典を明記

## Phase 7: 品質チェック（所要時間目安: 10分/記事）

### 7-1. SEOチェックリスト
- [ ] タイトルタグに「熊取」が含まれ、32文字前後
- [ ] メタディスクリプションが120文字前後
- [ ] 見出し階層（h1→h2→h3）が正しい
- [ ] 本文が2,500〜3,500文字の範囲
- [ ] 内部リンクが2〜3本含まれている
- [ ] 画像ALTにKWが含まれている
- [ ] FAQ形式の記事にはFAQスキーマを付与

### 7-2. コンテンツ品質チェックリスト
- [ ] 読者の検索意図に的確に答えているか
- [ ] 競合上位記事にない独自の情報・視点があるか
- [ ] 事実情報の裏取りが完了しているか
- [ ] 誤字脱字・不自然な表現がないか
- [ ] 読者が次の行動を取れるようCTAがあるか

### 7-3. ファクトチェック（情報の正確性・鮮度）

#### 📅 参照元の鮮度チェック基準

| 情報カテゴリ | 許容期限 | 理由 |
|------------|---------|------|
| 飲食店の営業情報（営業時間・定休日・メニュー・価格） | **3ヶ月以内** | 頻繁に変更される |
| 開店・閉店情報 | **1ヶ月以内** | 速報性が命 |
| イベント・祭りの日程 | **当年の公式発表** | 毎年変わる可能性あり |
| 行政制度・支援金・助成制度 | **1年以内 + 公式サイトで現行版か確認** | 年度で改廃される |
| 不動産相場・物件情報 | **6ヶ月以内** | 市場変動が大きい |
| 交通情報（時刻表・路線） | **ダイヤ改正後の最新版か確認** | 改正で無効になる |
| 歴史・文化・地理情報 | **3年以内でもOK**（普遍的情報） | 変化が少ない |
| 人口・統計データ | **最新の公式統計**を優先 | 古い数値は誤解を招く |

- [ ] 記事で引用した全ての情報源の**更新日・公開日**を確認したか
- [ ] **1年以上前**の情報には「※○○年時点の情報です」と注記したか
- [ ] 公式サイトのURLが**リンク切れ**になっていないか確認したか

#### 📱 SNS投稿の信頼性判定基準

SNSから情報を引用する際は、以下の基準で信頼性を判定する：

| チェック項目 | 判定基準 |
|------------|---------|
| **投稿者は誰か？** | 公式アカウント ＞ 実名の地元住民 ＞ 匿名アカウント |
| **投稿日はいつ？** | 6ヶ月以内の投稿のみ採用。古い投稿は「当時の情報」として扱う |
| **複数ソースで裏取りできるか？** | 1つのSNS投稿だけでは採用しない。最低2ソースで確認 |
| **写真・動画の添付があるか？** | 実際の体験に基づく投稿かの判断材料 |
| **誇張・炎上目的ではないか？** | 極端な表現はバイアスの可能性あり。冷静な投稿を優先 |

**SNS引用ルール:**
- [ ] 個人のSNS投稿をそのまま転載しない（著作権・プライバシー）
- [ ] 「SNS上では○○という声もある」のように**傾向として言及**する
- [ ] 飲食店のSNS公式アカウントの情報は**店舗への電話確認**で裏取り推奨
- [ ] 口コミの引用は**ポジティブ・ネガティブ両方**をバランスよく

#### 🔍 情報の裏取り（クロスチェック）ルール

| 情報の重要度 | 必要な裏取り数 | 例 |
|-------------|-------------|-----|
| **高（お金・安全に関わる）** | 3ソース以上 + 公式確認 | 医療費助成、料金、アクセス |
| **中（生活情報）** | 2ソース以上 | 営業時間、イベント日程 |
| **低（一般的な解説）** | 1ソース（信頼性が高ければ） | 歴史、地理、一般知識 |

- [ ] **数値データ**は必ず一次ソース（公式統計・公式サイト）から取得したか
- [ ] **「〜と言われている」「〜らしい」** のような曖昧表現を使っていないか
  - → 使う場合は情報源を明記するか、断定を避ける理由を文中で説明
- [ ] **AIが生成した情報**をそのまま事実として記載していないか
  - → AIの出力は必ず人間が事実確認を行う

### 7-4. WordPress投稿前チェック
- [ ] HTML構造が正しく動作するか
- [ ] カテゴリ・タグが正しく設定されているか
- [ ] アイキャッチ画像が設定されているか
- [ ] パーマリンク（URL）が適切か
