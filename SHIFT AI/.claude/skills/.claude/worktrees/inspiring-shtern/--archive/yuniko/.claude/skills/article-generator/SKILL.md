# 記事生成スキル

## 概要

このスキルは、参考記事（`参考記事/` フォルダ）のスタイルを参照し、指定テーマの記事を生成する。
出力はnote・stand.fm両方に対応した `.md` ファイルとして保存する。

## 参考スタイル（ユニコ記事スタイル）

| 要素 | 内容 |
|------|------|
| 書き出し | 体験談・問いかけから始める（読者の悩みを代弁） |
| 構成 | 体験談 → 現状解説 → ツール・手法紹介 → 地域活用 → FAQ → まとめ |
| 見出し | 番号付き（例：`## 1. タイトル` / `### 1-1. サブタイトル`） |
| トーン | 親しみ・プロフェッショナル・エネルギッシュの3要素バランス |
| 禁止表現 | 「〜することが重要です」「〜が求められます」などのAI感のある定型句 |
| 熊取町要素 | 基本的に含める。ユーザーから「減らして」と指示があった場合のみ省略 |

## 生成手順

### Step 1: 参考記事の確認
- `参考記事/` フォルダ内のファイルをRead
- スタイル・構成・トーンを確認する

### Step 2: 記事本文の生成
以下の構成で記事を作成する：

```
# [タイトル（SEO・キャッチー）]

[書き出し：読者の悩みを問いかけ形式で代弁]
[自己紹介・熊取町メディア運営者としての文脈]

---

## 1. [大テーマ]
### [サブ見出し]
[体験談・現状・問題提起]

## 2. [解説・ツール紹介]
### [具体的な内容]
[箇条書き・表を活用]

## 3. [地域・実践との接続]（熊取町要素）
[熊取町メディアでの活用・地方創生との繋がり]

## 4. [マネタイズ・活用法]

## 5. よくある質問（FAQ）
**Q. [質問]**
A. [回答]

## まとめ：[行動を促すタイトル]
[締め：読者へのアクション喚起]
```

### Step 3: ハッシュタグの生成

**note用**（最大15個）：
- 記事テーマに関連するキーワード
- `#AI活用` `#熊取町` `#地方創生` は基本的に含める
- `#note` を末尾に追加

**stand.fm用**（最大8個）：
- noteより絞り込んで厳選
- `#standfm` `#ラジオ` を含める

### Step 4: stand.fm放送説明文の生成

```
今回は「[テーマ]」をテーマにお話しします。[熊取町・地域メディア運営者としての文脈]。[記事の要点を2〜3文で要約]。[聴いてほしい読者層へのメッセージ]！
```

### Step 5: .mdファイルとして保存

- 保存先：`C:\Users\motok\OneDrive\Desktop\Claude code\yuniko\[テーマ名]記事.md`
- 構成：記事本文 → ハッシュタグ（note用/stand.fm用） → stand.fm説明文

## 品質チェック

生成後に以下を確認する：
- [ ] AI感のある表現が含まれていないか
- [ ] 事実確認できない情報が含まれていないか
- [ ] 文字数が3000文字以下か
- [ ] 熊取町要素が自然に組み込まれているか（省略指示がない場合）
- [ ] ハッシュタグがnote用・stand.fm用で分けられているか
- [ ] stand.fm説明文が150文字前後か
