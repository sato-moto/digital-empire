# OS 変換ルール — Windows CSV → Mac PDF

**役割**: `practice-{N}-final.csv` / `.md`（Windows 表記固定）から Mac 受講者向け PDF（`practice-{N}-guide-mac.pdf`）を生成する際に適用する変換マッピングの Single Source of Truth。

**参照元**: `references/phase2-bulk-pipeline.md` の Step 6.5-P（Mac 版 PDF 生成フェーズ）でメインClaude が Read する。

---

## 1. 変換ポリシー（全体方針）

### 1-1. 基本原則

- **CSV / MD は Windows 表記で固定**されている。講師の収録環境が Windows であるため、台本カンペは Windows 表記が唯一の正。
- **Mac 対応は PDF 生成時のみ**行う。本ルール表を参照して、Windows 表記を Mac 表記に変換する。
- Windows 版 PDF（`-guide-win.pdf`）は CSV をそのまま受講者目線に変換するだけ。本ルール表は **Mac 版 PDF 生成専用**。

### 1-2. 変換失敗時の原則

**サイレントに誤変換しない。** 本ルール表に該当しないパターン（未知のショートカット・アプリ名・ファイルパス表現）を発見したら、メインClaude は以下を実施する:

1. 変換せずに該当行をログに出力
2. ユーザーに警告:
   ```
   ⚠️ Mac 変換ルールにないパターンを検出しました:
     - 該当行: {CSV 行番号・内容}
     - 検出された Windows 固有表現: {ショートカット等}
     ユーザー判断をお願いします:
       ① このまま Windows 表記で出力（Mac 受講者には不親切）
       ② Mac 版の対訳を指示する
       ③ os-conversion-rules.md への追加を検討する
   ```
3. ユーザー回答を得てから PDF 生成を継続

### 1-3. 文脈判断が必要なケース

以下は機械的置換では誤変換する可能性があるため、メインClaude は文脈を読んで判断する。不確実な場合はユーザー確認。

- `Ctrl + クリック`（Windows の複数選択 vs Mac の右クリック）
- `Enter`（テキスト確定 vs ファイル名変更確定）
- 画面分割（macOS バージョンによってサポートが違う）

| `Ctrl + S` | `Cmd + S` | 保存 |
| `Ctrl + A` | `Cmd + A` | 全選択 |
| `Ctrl + F` | `Cmd + F` | 検索 |
| `Ctrl + P` | `Cmd + P` | 印刷 |
| `Ctrl + T` | `Cmd + T` | 新規タブ |
| `Ctrl + W` | `Cmd + W` | タブ/ウィンドウを閉じる |
| `Ctrl + N` | `Cmd + N` | 新規ウィンドウ/文書 |

| `Win + Shift + S`（範囲スクショ） | `Cmd + Shift + 4` | |
| `Ctrl + Shift + Esc`（タスクマネージャー） | `Cmd + Option + Esc`（強制終了ダイアログ） | |

### 2-2. ウィンドウ操作（画面分割）

| Windows（CSV 原文） | Mac PDF 変換後 | 備考 |
|---|---|---|
| `Windowsキー + 左矢印` | **3段階フォールバック併記**（下記参照） | 画面左半分に配置 |
| `Windowsキー + 右矢印` | **3段階フォールバック併記**（下記参照） | 画面右半分に配置 |
| `Windowsキー + 上矢印` | 緑信号ボタンを `Option + クリック`（ズーム）または ダブルクリック | 最大化 |
| `Windowsキー + 下矢印` | `Cmd + M` | 最小化（Dock に収納） |
| `Windowsキー + D`（デスクトップ表示） | `F11`（Mission Control 設定依存）または トラックパッド3本指スワイプ | |
| `Alt + Space + X`（最大化） | 緑信号ボタンをクリック | |

#### 画面分割の3段階フォールバック（Mac PDF での記載テンプレ）

Windows キー + 矢印は Mac には直接対応がないため、Mac PDF では以下のように **操作の目的 + 3段階の手段** を併記する:

```
【操作の目的】ウィンドウを画面左半分に配置する

【手段1】macOS 14 Sonoma 以降の場合:
  ウィンドウメニュー（画面上部） → 「左にタイル表示」をクリック

【手段2】Rectangle.app（無料）をインストール済みの場合:
  Ctrl + Option + ←（左半分）/ Ctrl + Option + →（右半分）
  ※ Rectangle.app は https://rectangleapp.com/ から無料でダウンロード可能

【手段3】macOS 13 以前、またはツール未インストールの場合（手動）:
  1. ウィンドウ上部のタイトルバーをドラッグ
  2. 画面左端までドラッグして離す（ウィンドウを画面左半分に配置）
  または:
  1. ウィンドウ左上の緑信号ボタンにマウスを重ねる（クリックではなくホバー）
  2. 表示されるメニューから「ウィンドウを画面左側にタイル表示」を選択
```

### 2-3. マウス操作

| Windows（CSV 原文） | Mac PDF 変換後 | 備考 |
|---|---|---|
| `右クリック` | `右クリック` | 両 OS で通じる。Mac でも「右クリック」で統一 |
| `ダブルクリック` | `ダブルクリック` | 共通 |
| `Shift + クリック`（範囲選択） | `Shift + クリック` | 共通 |
| `Ctrl + クリック`（複数選択） | `Cmd + クリック` | **§3-1 参照** |

脚注として Mac PDF の冒頭または初出時に1度だけ補足:
> Mac で右クリックできない場合は、トラックパッドを2本指でタップ、または Control キーを押しながらクリックしてください。

### 2-7. ブラウザ・Web アプリ（共通扱い・変換不要）

以下は OS 非依存。変換せず共通記載のまま Mac PDF に出力する:

| 項目 | 扱い |
|---|---|
| Chrome / Firefox / Edge / Safari 内の操作 | OS 非依存。変換不要 |
| ChatGPT / Claude / Notion / Figma 等 Web アプリ | OS 非依存。変換不要 |
| ブラウザ内ショートカット `Ctrl + Tab`（タブ切替） | Mac も `Ctrl + Tab`（変換しない） |
| ブラウザ内ショートカット `Ctrl + Shift + T`（閉じたタブ復元） | Mac は `Cmd + Shift + T` |
| URL の入力、ボタンクリック、フォーム入力 | OS 非依存 |

### 2-8. 変換不要の項目一覧

以下は両 OS で同じ記法なので変換しない:

- マウス操作（左クリック・ダブルクリック・スクロール・ドラッグ）
- Web アプリの UI 名・URL・ボタン名
- 一般的な概念（「入力欄に貼り付け」「Enter を押す」「ボタンをクリック」等）
- スクロール・ズーム操作
- ブラウザのアドレスバー URL

## 3. 変換判定の難しいケース

### 3-1. `Ctrl + クリック` の両義性

Windows の `Ctrl + クリック` は「複数選択」を意味するが、Mac では `Ctrl + クリック` は「右クリック」と同義になる。文脈で判断:

| 文脈 | Mac 変換後 |
|---|---|
| 「ファイルを複数選択する」「項目を複数選ぶ」 | `Cmd + クリック`（Mac の複数選択） |
| 「右クリックメニューを表示する」 | `右クリック` を使う（Ctrl + クリックと書かない） |
| 不明確な場合 | **ユーザー確認**（サイレント変換禁止） |

### 3-2. `Windows キー + 矢印` の macOS バージョン分岐

画面分割はバージョンによってネイティブサポート状況が異なる:

- **macOS 14 Sonoma 以降**: ウィンドウメニューから「左にタイル表示」等が利用可能
- **macOS 13 以前**: ネイティブサポートなし。Rectangle.app 等のサードパーティツール推奨

Mac PDF では **必ず §2-2 の3段階フォールバックを全部記載**する。1つの手段だけに絞らない（受講者の macOS バージョンが不明なため）。

### 3-3. アプリ固有ショートカット（Cmd 単純置換でよいケース）

draw.io、Notion、Figma、VSCode 等の**クロスプラットフォームアプリ**では、基本的に Ctrl → Cmd の単純置換でよい。例:

- draw.io: `Ctrl + S`（保存）→ `Cmd + S`
- Notion: `Ctrl + N`（新規ページ）→ `Cmd + N`
- VSCode: `Ctrl + P`（ファイル検索）→ `Cmd + P`

ただしアプリによっては Mac 版で別キーに割り当てられている場合もあるので、広く使われている主要アプリ以外では要確認。

### 3-4. 変換不能パターン発見時のユーザー確認フロー

本ルール表の §2 にも §3 にも該当しないパターンを発見した場合:

1. メインClaude は変換を試みない
2. Mac PDF 生成を一時中断
3. ユーザーに警告（§1-2 のフォーマット）
4. ユーザー回答を得て:
   - 「Windows 表記のまま出力」なら該当箇所にその旨注記
   - 「こう変換して」と指示されたら適用
   - 「ルール表に追加すべき」なら `os-conversion-rules.md` への追記を提案
5. 解決後に PDF 生成を継続

## 4. 事前準備セクションの OS 分岐

`practice-{N}-final.md` の §2（事前準備）は Windows 表記で書かれている。PDF 生成時に OS 別で再構成する。

### 4-1. Windows 版 PDF の事前準備（無変換）

- CSV の §2 をそのまま採用
- タスクバー自動非表示、集中モード ON、通知 OFF 等の Windows 項目をそのまま記載

### 4-2. Mac 版 PDF の事前準備（§2-4 適用 + Mac 独自項目追加）

以下の追加項目を Mac 版の事前準備セクションに挿入する:

```
【Mac 版受講者の追加事前準備】

□ Dock を自動的に表示/非表示にする
  システム設定 → デスクトップと Dock → 「Dock を自動的に表示/非表示」を ON

□ おやすみモード ON
  コントロールセンター → 集中モード → おやすみモード

□ （推奨）Rectangle.app のインストール
  本講座では画面を左右に分割する操作があります。
  macOS 14 Sonoma 以降の方は標準機能で実施可能ですが、
  macOS 13 以前の方は Rectangle.app（無料）のインストールを推奨します:
  https://rectangleapp.com/

□ 画面解像度を 1920×1080 相当に設定
  システム設定 → ディスプレイ → 解像度スケーリング
  ※ Retina ディスプレイの場合、「スペースを拡大」を選択すると近い見え方になります
```

## 5. 警告とフォールバック

### 5-1. 変換表にないショートカット検出時

メインClaude は以下のロジックで検出:

1. CSV / MD 内から以下のパターンを正規表現で抽出:
   - `Ctrl\s*\+\s*\w+`
   - `Alt\s*\+\s*\w+`
   - `Win(dows)?\s*(キー)?\s*\+\s*\w+`
   - `F\d+`（ファンクションキー）
2. 本ルール表 §2-1 / §2-2 / §2-7 に掲載されていないものを列挙
3. 該当があれば §1-2 のフォーマットで警告

### 5-2. macOS バージョン差に関する注意書きテンプレ

画面分割等、macOS バージョンで挙動が違う項目は以下の注記を Mac PDF に1度だけ挿入:

```
💡 macOS バージョンによる注意:
本手順書の一部操作（画面分割等）は macOS 14 Sonoma 以降の機能を
前提としています。macOS 13 Ventura 以前をお使いの方は、
Rectangle.app（無料）のインストールまたは手動リサイズをご利用ください。
```

### 5-3. パス変換時の注意

- `C:\Users\{name}\...` → `/Users/{name}/...` は機械的置換で問題ない
- ただし CSV 内に `C:\` 以外のドライブ（`D:\`、`E:\` 等）が出てきたら、Mac には直接対応がないため **ユーザー確認**
- 外付けドライブは `/Volumes/{ドライブ名}/` にマウントされる旨を注記
