# ワークフロー分岐・Phase遷移ルール

## 起動時の最優先処理: 未完了セッションの検出

**`/create-course` 起動時、起動分岐（①②）を提示する前に、必ず以下を実行する:**

1. メモリの `MEMORY.md` を確認し、`project_*.md` が存在するかチェック
2. 各プロジェクトの `output/{project_name}/progress.json` を Read（存在する場合）
3. `progress.json` の `next_action` が未完了の状態（=最終 Step に到達していない）であれば、**再開提案を最初に表示**

### 再開提案フォーマット

```
🔄 前回の続きから再開できるプロジェクトがあります。

📋 プロジェクト: {curriculum_name}（{project_name}）
🏁 前回完了: {current_phase} / {current_step}
⏭️ 次の作業: {next_action}
📝 引き継ぎメモ: {handoff_notes}

続きから再開しますか？

① はい、続きから再開する
② 別のプロジェクトの続きから再開する（複数ある場合）
③ 新しい作業を始める（下記の通常分岐へ）
```

**再開選択時の動作:**
- `progress.json` の `completed_artifacts` から既存成果物を把握
- `next_action` に指定された Step から **同じレギュレーション** で実行（3エージェント並列等を省略しない）
- 完了済み Step は絶対に再実行しない

**詳細は `session-handoff.md` を参照。**

## 起動時の分岐（新規開始時）

未完了セッションがない、またはユーザーが「③新しい作業を始める」を選んだ場合、以下を提示する:

```
どちらから始めますか？

① カリキュラム設計から始める（Phase 1 → Phase 2）
   カリキュラム全体の構成（セット・講座一覧）をリサーチ→設計→壁打ちで作り、
   その後に個別の講座を制作します。

② 講座制作から始める（Phase 2 から）
   既にカリキュラム構成が決まっている、または単体の講座を作りたい場合。
   必要な情報をヒアリングしてから制作に入ります。
```

### ① を選んだ場合
→ Phase 1（Step 0）から開始。通常フロー。

### ② を選んだ場合
→ Phase 2 から開始。必要な情報をユーザーからヒアリングする。

**まず制作単位を選択させる:**

① おすすめ: Lv.1 の全講座を上から順に作る（全SET単位）
② 範囲を指定して制作する（SET単位）
③【従来】タイトル→ゴール→章→アウトラインを段階設計（outline_v1.md準拠、段階パイプライン）

**①②を選んだ場合:**
- 「既存の course-structure.md のパスを教えてください」とユーザーに聞く
- course-structure.md を読み込み → SET一覧を表示 → ユーザーが範囲選択
- `guidelines/実務_講座設計ガイドライン_2.md` を Read してSET内設計方針を決定
- Step 3（リサーチ）へ

**③を選んだ場合:** 従来パイプライン（変更なし）

### ①②（SET単位）を選んだ場合のフロー

1. course-structure.md を読み込み、SET一覧とステータスを表示
2. ①の場合: Lv.1 の全SETを上から順に選択
   ②の場合: ユーザーがSETを指定
3. `guidelines/実務_講座設計ガイドライン_2.md` を Read
4. SET内設計方針を確認・補完:
   - 一貫ケースが定義済みか → 未定義なら設計
   - 問題解決ストーリーのこのSETの位置づけ
   - 教養How用別事例の割当
   - 使用ツール
5. ユーザーに確認後、SET内の各講座をcourse-structure.mdの順で制作（Step 3 へ）

### 【従来】のヒアリング項目

従来パイプラインでは T1/T2（タイトル+ゴール一括設計）で設計するため、ヒアリングでは `outline_v1.md` の①②の出力を生成するのに必要な**素材情報**を収集する。タイトル・ゴールの完成品はヒアリングでは聞かない。

**①タイトル設計（outline_v1.md ①）に必要な情報:**

| # | 項目 | 必須 | 例 |
|---|---|---|---|
| 1 | 講座テーマ | 必須 | 「営業メールをAIで効率化する方法」 |
| 2 | 想定受講者 | 必須 | 「中小企業の営業担当」 |

**②ゴール設計（outline_v1.md ②）に必要なペルソナ情報:**

| # | 項目 | 必須 | 例 |
|---|---|---|---|
| 3 | 属性 | 必須 | 職種: 営業 / 役職: 一般社員 |
| 4 | 経験値 | 必須 | AI未経験 |
| 5 | 現在の課題 | 必須 | 「メール作成に毎回15分かかっている」 |
| 6 | 活用目的 | 必須 | 「メール業務の時短」 |
| 7 | 想定活用シーン | 必須 | 「取引先へのお礼メール作成」 |
| 8 | 学習動機 | 必須 | 「AIを使えるようになりたいが何から始めればいいか分からない」 |
| 9 | 制約 | 必須 | 「業務時間内で学びたい、専門用語は避けてほしい」 |

**コンテキスト情報:**

| # | 項目 | 必須 | 例 |
|---|---|---|---|
| 10 | 前後の講座 | 任意 | 前: 【教養】○○ / 次: 【実務】△△ |
| 11 | コース文脈 | 任意 | 副業コース or 業務効率化コース |
| 12 | 使用ツール | 任意 | ChatGPT |

ヒアリング後、メインClaude が入力情報を `{out}/lecture-brief.md` に整理し、**Step 4-T1/T2（タイトル+ゴール一括設計）から段階パイプライン**で進行する。

## Phase 2 の分岐構造

```
SET単位（①②選択時）:
  → guidelines/実務_講座設計ガイドライン_2.md でSET内設計
  → SET内の各講座を種別で自動判定:
    ├─ 教養 → guidelines/教養_講座設計ガイドライン.md → Step 3〜6（一括パイプライン）
    └─ 実務 → guidelines/実務_講座設計ガイドライン.md → Step 3〜6（一括パイプライン）

従来（③選択時）:
  └─ guidelines/outline_v1.md → 4-T1/T2 + Step 3 + 4-T3 + 4-T4 + 5-T + 6（段階パイプライン）
```

## Phase 間遷移ルール

**各 Phase が完了するたびに、必ずユーザーに確認を取ってから次の Phase に進むこと。**
**確認なしに次の Phase へ自動進行してはならない。**

| タイミング | 報告内容 | 次のアクション |
|---|---|---|
| Phase 1 完了時 | `research-summary.md` と `course-structure.md` の概要を提示 | ユーザーの承認後、推奨（Lv.1から順）or 講座指定で Phase 2 の対象範囲を決定 |
| Phase 2（1SET完了時） | SET内の全成果物を提示 | ユーザーに次アクションを確認（次SET / Phase 3 / 修正 / 終了） |
| Phase 3（全講座完了時） | 完成した全スライド構成の講座名一覧を提示 | 講座名入り選択肢を提示（①この順でPhase 4 / ②順番変更 / ③修正 / ④終了） |
| Phase 4（全講座完了時） | 完成した全講座の一覧を提示 | サムネイル一括生成を提案 → 次アクション選択（①次の講座群 / ②指定 / ③新規 / ④終了） |

## Phase 2 の進行フロー

**SET単位で進行し、1SET完了ごとにユーザーに次のアクションを確認する。**

```
Phase 2 開始
    │
    ▼
┌──────────────────────────────┐
│ SET N の制作                   │
│ （教養 → 実務 の順で Step 3〜6）│
└─────────┬────────────────────┘
          │
          ▼
    SET N の全講座が完了
    ステータス更新
          │
          ▼
    ★ ユーザーに確認:
    「SET {N} が完成しました！次にどうしますか？」
    ① 次のSETに進む（SET {N+1}）
    ② このSETの教養講座を先に Phase 3（スライド構成）に進める
    ③ 修正する
    ④ ここで終了する
          │
    ①─────┼──► 次のSETへ（Phase 2 続行）
    ②─────┼──► 教養講座のみ Phase 3 へ（実務は完了済み）
    ③─────┼──► 修正 → SET N に戻る
    ④─────┴──► メモリ保存 → 終了
```

**Phase 3 への遷移ルール:**
- 教養講座 → Phase 3（スライド構成）→ Phase 4（台本+スライド生成）へ進む
- 実務講座 → 収録指示書（md + csv）+ PDF手順書で完了。Phase 3/4 は対象外
- Phase 3 への遷移は1SET完了時に選択可能（②を選んだ場合）
- 複数SETの教養講座をまとめて Phase 3 に進めることも可能

### 1SET完了時の確認メッセージ

```
SET {N}「{SETテーマ}」の全講座が完成しました！

📁 教養講座:
  1. 【教養】{講座名A} → outline-final.md ✅

📁 実務講座（収録指示書）:
  2. 【実務】{講座名B} → practice-{N}-final.md + .csv
     └─ PDF 手順書: practice-{N}-guide-win.pdf / practice-{N}-guide-mac.pdf ✅

次にどうしますか？

① 次のSET（SET {N+1}「{次のSETテーマ}」）に進む
② このSETの教養講座を Phase 3（スライド構成）に進める
③ 修正する → 講座名を指定してください
④ ここで終了する → プロジェクト進捗を保存
```

※ サムネイル生成は Phase 4（台本+スライド生成）完了後にカリキュラム単位で一括生成する。SET完了時点では提示しない。

### Phase 3 完了時の確認メッセージ

```
全{N}講座のスライド構成が完成しました！

📁 完成したスライド構成:
  1. {講座名A} → slide-design.csv ✅
  2. {講座名B} → slide-design.csv ✅
  3. {講座名C} → slide-design.csv ✅

次にどうしますか？

① この順番で Phase 4（台本+スライド生成）に進める（おすすめ）
  1. {講座名A}
  2. {講座名B}
  3. {講座名C}

② 順番や対象を変えて Phase 4 に進める
  → 講座を指定してください

③ スライド構成を修正してから進める
  → 修正したい講座名を教えてください

④ ここで終了する
  → プロジェクトの進捗を保存して終了
```

## Phase 4 完了後の次アクション選択

Phase 4（台本 + スライド生成）が **対象範囲の全講座で** 完了したら:

1. **完了した講座の結果をメモリに保存する**
2. **course-structure.md のステータスを更新する**（該当講座を「完了」に変更）
3. **サムネイル一括生成を提案する**（プロアクティブ）
4. **ユーザーに次のアクションを選択させる**

### サムネイル一括生成の提案（Phase 4 完了直後）

Phase 4 完了直後に、**必ず** 以下を提示してサムネイル一括生成を提案する:

```
対象範囲の全{N}講座が完了しました！（台本 ✅ / スライド ✅）

📁 完成した講座:
  1. {講座名A} → スライド ✅ / 台本 ✅
  2. {講座名B} → スライド ✅ / 台本 ✅
  3. {講座名C} → スライド ✅ / 台本 ✅

🎨 カリキュラム全体のサムネイルを一括生成しますか？
  → 完了済みの全講座（教養+実務）のサムネイルをまとめて生成します
  → 共通設定（登壇者・ブランド等）を1回確認した後、各講座分を順次生成します

① はい、サムネイルを一括生成する
② いいえ、次の作業に進む
```

- ① の場合: 下記「カリキュラム単位サムネイル一括生成フロー」を実行 → 完了後に次アクション選択へ
- ② の場合: 下記「次アクション選択」へ

### カリキュラム単位サムネイル一括生成フロー

`/create-course` から `/thumbnail-designer` を起動する場合は、常にこのカリキュラム単位パターンを使用する。
（`/thumbnail-designer` を直接起動した場合は従来の講座単位パターンで動作する。）

```
Step S-1: 対象講座一覧の収集
    │
    ▼
Step S-2: カリキュラム共通設定の確認（1回だけ）
    │
    ▼
Step S-3: 各講座のサムネイル生成ループ
    │
    ▼
Step S-4: 一括生成完了報告
    → 次アクション選択へ
```

**Step S-1: 対象講座一覧の収集**

1. `course-structure.md` + `progress.json` から完了済み講座を収集
2. 各講座の入力ファイルパスを一覧化:
   - 教養: `output/{project}/01_outline/{講座名}/outline-final.md`
   - 実務: `output/{project}/05_practice/practice-{N}-final.md`
3. ユーザーに一覧を提示し、対象を確認:

```
📋 サムネイル生成対象の講座一覧:

  1. 【教養】{講座名A} → outline-final.md
  2. 【実務】{講座名B} → practice-{N}-final.md
  3. 【教養】{講座名C} → outline-final.md
  ...

全{N}講座のサムネイルを生成します。対象を変更しますか？
  A. この一覧でOK
  B. 除外する講座がある（講座番号を指定）
```

**Step S-2: カリキュラム共通設定の確認（1回だけ）**

全講座に共通する設定をまとめて確認する（`/thumbnail-designer` の確認0〜3に相当）:

| 確認項目 | 内容 | 備考 |
|---|---|---|
| 登壇者/人物 | 全講座共通の人物情報（名前・肩書き） or 講座ごと個別指定 | 共通の場合は1回の入力で全講座に適用 |
| 人物配置パターン | A. 丸型で小さく配置 / B. 大きく配置 / C. 人物なし | 全講座統一 |
| 参照画像 | 人物画像のパス / AI生成 | 共通の場合は1回の指定で全講座に適用 |
| デザイン参照画像 | ブランドスタイルの参照画像（あれば） | 全講座統一 |
| サイズ | デフォルト: 1000x300（コース講座用） | 全講座統一 |
| ブランド/ロゴ | ロゴ画像のパス・ブランドカラー（あれば） | 全講座統一 |

※ 講座名は各 outline/practice ファイルから自動抽出するため、ここでは確認しない。

**Step S-3: 各講座のサムネイル生成ループ**

Step S-1 の対象一覧を順に処理:

1. 各講座について `/thumbnail-designer` を起動
2. Step S-2 の共通設定を引き継ぎ（`/thumbnail-designer` の確認フローをスキップ）
3. 講座固有情報（講座名・狙い・強調ポイント等）は各 outline/practice ファイルから自動抽出
4. 1講座完了ごとに結果を表示:
   ```
   ✅ {講座名} → output/{project}/06_thumbnails/{講座名}.png
   ```

**Step S-4: 一括生成完了報告**

```
🎨 全{N}講座のサムネイルが完成しました！

📁 出力先: output/{project}/06_thumbnails/
  1. {講座名A}.png ✅
  2. {講座名B}.png ✅
  3. {講座名C}.png ✅

修正したいサムネイルはありますか？
  A. 全てOK → 次アクション選択へ
  B. 修正する → 講座番号を指定してください（/thumbnail-designer で再生成）
```

### 次アクション選択（サムネイル一括生成後 or スキップ後）

```
次にどうしますか？

① 次の講座群を制作する（おすすめ）
   → 次の未着手講座:
     1. 【教養】{次の講座名D}
     2. 【実務】{次の講座名E}
     ...

② 講座を指定して制作する
   → course-structure.md から選択

③ 新しい講座をヒアリングから制作する
   → Phase 2 直接開始のヒアリングフローへ

④ ここで終了する
   → プロジェクトの進捗を保存して終了
```

- ① の場合: course-structure.md の上から順（Lv.1→Lv.2→...）で次の未着手講座群を提示し、ユーザー承認後 Phase 2 から開始
- ② の場合: course-structure.md の全講座一覧（ステータス付き）を表示し、講座を選択させる
- ③ の場合: Phase 2 直接開始のヒアリングフローへ
- ④ の場合: 進捗をメモリに保存し、次回会話で再開できるようにする

### 全体フロー図

```
Phase 1 完了
    │
    ▼
Phase 2: SET単位で講座制作
    │  SET N: 教養 → 実務（Step 3〜6）
    │
    ▼
★ 1SET完了ごとにユーザーに確認:
    ① 次のSETに進む → Phase 2 続行
    ② 教養講座を Phase 3 へ → Phase 3 開始
    ③ 修正する → SET N に戻る
    ④ ここで終了する → メモリ保存
    │
    ▼（②を選んだ場合）
Phase 3: スライド構成設計（教養講座のみ）
    │
    ▼
確認: 講座名一覧を提示 →「この順で Phase 4 に進めますか？」
    │
    ▼
Phase 4: 台本 + スライド生成（教養講座のみ）
    │
    ▼
🎨 サムネイル一括生成の提案（カリキュラム単位）
    ① はい → Step S-1〜S-4（/thumbnail-designer をカリキュラム単位で起動）
    ② いいえ → 次アクション選択へ
    │
    ▼
┌─────────────────────────────┐
│ 次アクション選択             │
│ （次の未着手講座名を提示）   │
└─────────┬───────────────────┘
          │
    ①─────┼──► 次の未着手講座群（名前提示）→ Phase 2
          │
    ②─────┼──► 講座を指定選択 ──► Phase 2
          │
    ③─────┼──► 新規ヒアリング ──► Phase 2
          │
    ④─────┴──► メモリ保存 ──► 終了
```
