# カリキュラム設計ガイドライン（教養 × 実務）

このSkillは、複数SET（単元）から構成されるカリキュラムにおいて、**教養（概念・思考法）講座**と**実務（ハンズオン）講座**をどう噛み合わせるかの設計原則をまとめたものです。特定のコーステーマに依存しない汎用ルールとして使ってください。

---

## 1. 設計の3原則

### 原則1: 一問題解決ストーリーで貫く（ただし固定ではない）

ハンズオン系の一連のSETは、可能な限り受講者が **1つの問題解決を通して体験する** 構成を基本とする。SETごとに毎回別の題材に切り替えると、前提の再説明にコストがかかり、プロセスの連続性が失われる。前のSETの成果物が次のSETの入力になる設計を基本とする。

ただし、コースのテーマや学習目標によっては、途中のSETから別の題材・別の問題に切り替わる場合もある。その場合は以下を守る:

- 切り替わるタイミングと理由をカリキュラム設計ドキュメントに明記する
- 切り替わり後のSETでは、新しい題材の前提を改めて提示する
- 切り替わりが学習導線として自然か（前のSETで得たスキルが次の題材に転用される構造か）を確認する

「1つの問題で貫く」か「途中で切り替える」かは、コースごとの設計判断として明示的に決める。

### 原則2: スキルが先、AI活用が後

各講座では「**まず人間のスキル（思考法・フレームワーク）を理解する → それをAIで効率化する**」の順を徹底する。AIの操作から入ると、受講者はフレームワークをブラックボックス化したまま使う癖がつく。

### 原則3: ツールの使い分けを明示する

各SETで使うツール（対話型AI、エージェント型AI、特定ツールなど）をカリキュラム表に明記する。同じ「AIを使う」でも、単発の壁打ちと全プロセスの仕組化では必要なスキルが違う。

---

## 2. 標準一貫ケースを用意する

SETをまたいで使う **共通ケース** を1つ用意する。最低限、以下の要素を固定しておく。

| 項目 | 内容 |
|---|---|
| 組織／主体 | 業種・規模感 |
| 課題 | 定量的に測定できる指標の変化 |
| 規模 | 対象となる件数・金額などのスケール |
| 受講者のロール | どの立場から関わるか |

**使用ルール:**
- 実務講座の動画デモ・テキスト教材は、このケースのデータをプロンプトや演習に組み込む
- **教養講座のHow具体例には、この一貫ケースを使わない**（→ Section 4 参照）

---

## 3. 教養 / 実務の役割分担

| 区分 | 役割 | 典型構成 |
|---|---|---|
| 教養 | 概念・思考法・フレームワークを理解する | Why → What → How → Wrap |
| 実務 | 一貫ケースに対してAIを使って実際に手を動かす | 動画デモ + テキスト教材（ハンズオン） |

教養で「何をする技術か」を学び、実務で「その技術を一貫ケースに当てはめて実行する」流れ。教養が先、実務が後。

---

## 4. 教養の How 例外ルール（答え漏洩の防止）

### 問題

教養のHowセクションで一貫ケースを題材に具体例を示すと、**実務ハンズオンの答えを先に見せてしまう**。受講者は「考える」前に「思い出す」だけになる。

### ルール

**教養のHow具体例は、一貫ケースとは別の業界・別の問題パターンを使う。**

SETごとに、そのSETで扱う思考法と構造が似ているが題材が異なる事例を用意する。カリキュラム設計時点で、SETごとの「教養How用事例リスト」を決めておくとよい。

| SET | 実務で扱う一貫ケース | 教養Howで使う別事例 |
|---|---|---|
| SET N | 固定 | 別業界・別問題A |
| SET N+1 | 固定 | 別業界・別問題B |
| ... | ... | ... |

**例外:** カリキュラム最終SET（仕組化・統合）では、受講者がすでに一貫ケースを解き終わっているため、教養Howで一貫ケースを参照してもよい。

---

## 5. カリキュラム設計時のチェックリスト

新しいコースを設計する際、以下を順に決める。

- [ ] コース全体のゴール（受講後に何ができるか）は定義されているか
- [ ] SETは問題解決ストーリーの流れに沿っているか
- [ ] 標準一貫ケースは定義されているか（組織／課題／規模／ロール）
- [ ] 各SETの教養・実務の対応関係は明確か
- [ ] 教養Howの「別事例」はSETごとに割り当てられているか
- [ ] 前SETの成果物 → 次SETの入力、という連鎖は成り立つか
- [ ] 使用ツールはSETごとに明記されているか
- [ ] 最終SET（仕組化）は、それ以前のSETの全工程を統合する設計か

---

## 6. 設計決定を記録する

カリキュラムを設計するたびに発生する **固有の決定事項**（一貫ケースの具体値、SET数、教養How事例の割当、ツール選定など）は、コース直下の `CONVENTIONS.md` など別ファイルに記録する。このSkillは汎用ルールのみを提供し、コース固有の値は保持しない。

固有決定事項の記録項目（最低限）:

- 標準一貫ケースの具体値
- SET一覧と各SETのテーマ
- 各SETで使うツール
- 教養How用事例の割当表
- 変更履歴（日付・変更内容・理由）
