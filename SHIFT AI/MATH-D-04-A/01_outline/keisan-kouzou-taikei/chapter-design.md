# Step 4-T3: 章ロール設計 — MATH-D-04-A

## 章構成

### Ch0：冒頭あいさつ
- 章タイトル：暗記から「構造ハック」へ
- 章ロール：興味喚起（フック）
- 章ゴール：受講者が「この動画を最後まで見たい」と感じ、中学版との接続（「教えてもらう」→「自分の思考を検証する」への段階引き上げ）を理解している状態になる
- 尺目安：60秒以内

### Ch1：計算の構造を見抜くためのAIへの聞き方
- 章タイトル：計算の構造を見抜くためのAIへの聞き方
- 章ロール：認識転換＋概念理解（マインドセットの引き上げ）
- 章ゴール：受講者が「場当たり的な公式当てはめではなく、AIを壁打ち相手にして計算の構造を見抜く」という学習姿勢を説明でき、高校計算分野特有の3種プロンプト（解法検証・構造発見・型抽出）の使い分けを理解している状態になる
- 尺目安：8〜12分

### Ch2：実演・計算編（録画映像を差し込む）
- 章タイトル：実演・計算編
- 章ロール：実行体験（デモ）
- 章ゴール：【空白チャプター】録画映像差込。台本・スライド内容は生成しない
- 尺目安：10〜20分

### Ch3：NotebookLMで自分専用の数学コーチを作る
- 章タイトル：NotebookLMで自分専用の数学コーチを作る
- 章ロール：応用展開（外部ツール連携による自走環境構築）
- 章ゴール：受講者が「実演で作った解法マニュアルや教材PDFをNotebookLMにアップロードし、自分専用の数学コーチとして日常的に活用する方法」を理解している状態になる
- 尺目安：5〜8分

### Ch4：ここからはあなたの番
- 章タイトル：ここからはあなたの番
- 章ロール：統合定着＋自走スイッチ起動
- 章ゴール：受講者が「学んだ内容（構造を検証する姿勢＋3種プロンプト＋NotebookLM活用）を整理し、今日から自分で計算の構造をAIに検証させて体系化する意志を持つ」状態になる
- 尺目安：3〜5分

## 中学版（M-01-A）からの進化ポイント

| 観点 | 中学版 | 高校版（本講座） |
|------|--------|-----------------|
| AIとの対話レベル | 「教えてもらう」— なぜ？を聞く | 「検証する」— 自分の仮説をぶつける |
| プロンプト3種 | なぜ？ / 別の視点で / 検証して | 解法検証 / 構造発見 / 型抽出 |
| 求められる姿勢 | 「わからない→AIに聞く」 | 「仮説を立てる→AIに検証させる」 |
| 体系化の深度 | 学習メモのストック | 解法マニュアルとしての構造化 |
| 計算の捉え方 | 仕組みを理解する | 構造を見抜いて「型」として体系化する |

## NGチェック（7項目）

- [x] 各章ゴールが講座ゴールに貢献している → 全章が「構造を見抜き体系化して自走する」に収束
- [x] 各章ロールが明確に定義されている → 興味喚起/認識転換/実行体験/応用展開/統合定着
- [x] 章同士に内容重複がない → Ch1=考え方と3種プロンプト、Ch2=実演デモ、Ch3=ツール環境構築、Ch4=まとめ+エール
- [x] ゴール達成に必要な章が欠けていない → マインドセット転換→プロンプト習得→実演体験→環境構築→自走意志
- [x] 難易度順序が逆転していない → 概念→実践→応用→統合の順
- [x] 実行フェーズの章が存在する → Ch2（実演）+ Ch3（ツール連携）
- [x] 導入・認識フェーズの章が存在する → Ch0（興味）+ Ch1（認識転換）
