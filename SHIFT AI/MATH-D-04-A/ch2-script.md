# Ch2 実演台本 — MATH-D-04-A

## 実演の流れ

**例題**: x²−9 と x²−6x+9 の因数分解
**ツール**: Gemini（メイン）
**目的**: 3種プロンプト（解法検証→構造発見→型抽出）を1つの例題で完結させる

---

## シーン1: 導入（問題提示）

### 画面操作
- Geminiを開く（新しいチャット）

### ナレーション
「じゃあ実際にやってみよう。今回は因数分解を例にする。あえて基礎的な問題を選んでいる。なぜか。大事なのは問題の難しさじゃなくて、AIとの壁打ちの『やり方』を身につけることだからだ。このやり方さえ掴めば、数列でも三角関数でも、自分でどんどん深く聞いていける。ここに2つの因数分解がある。x²−9 と x²−6x+9。どっちも似た式に見えるけど、使う公式が違う。この『どっちの公式を使うか』の判断基準を、AIとの壁打ちで見抜いていく。」

---

## シーン2: 解法検証プロンプト

### 画面操作
Geminiに以下を入力して送信:
```
因数分解でどの公式を使うか迷っている。
まず x²−9 を因数分解したい。
この式は a²−b² の形だから「和と差の積」を使えばいいと思うんだけど、この方針で合ってる？
解答は言わずに、方針のメリット・デメリットだけ教えて。
```

### ナレーション
「まず解法検証プロンプト。冒頭の1文で『因数分解でどの公式を使うか迷っている』と状況を伝えている。これでAIはこちらの立ち位置を理解した上で返答してくれる。そして自分の仮説『和と差の積が使えそう』をぶつけて、『解答は言わずに、方針のメリット・デメリットだけ教えて』で答えをブロック。」

### Geminiの返答を待つ → 返答を読み上げながら解説
「AIが返してくれた。見てみよう。（返答の要点を読み上げ）方針は合っている。x² = x², 9 = 3² と見れば、a²−b² の形に当てはまる。メリットは一瞬で因数分解できること。デメリットは…特になし。シンプルな問題だからだ。じゃあ実際にやってみよう。(x+3)(x−3)。これが答えだ。」

---

## シーン3: 構造発見プロンプト（2問目との比較）

### 画面操作
Geminiに以下を入力して送信:
```
次は x²−6x+9 を因数分解したいんだけど、「和と差の積」ではうまくいかない。私は何につまずいている？
```

### ナレーション
「2問目。x²−6x+9。さっきの和と差の積を使おうとしても、うまくいかない。ここで構造発見プロンプトの出番だ。聞き方はシンプルに『私は何につまずいている？』。自分の思考の穴をAIに突かせる。答えをもらうんじゃなくて、自分のつまずきの構造を見抜いてもらう。」

### Geminiの返答を待つ → 返答を読み上げながら解説
「AIの返答を見てみよう。（返答の要点を読み上げ）視点1：項数で判断する。2項なら和と差の積、3項なら完全平方式を疑う。視点2：3項の場合、最初と最後の項がそれぞれ何かの2乗になっているかチェックする。x² = x², 9 = 3²。そして真ん中の項が 2×x×3 = 6x と一致するか。一致してる。だから x²−6x+9 = (x−3)²。」

---

## シーン4: 型抽出プロンプト

### 画面操作
Geminiに以下を入力して送信:
```
今の2問の「判断の流れ」をフローチャートにまとめて。次に別の因数分解が出てきてもこれを見れば迷わない、ってレベルで。
```

### ナレーション
「最後に型抽出プロンプト。2問解いて終わりじゃない。ここで『型』にする。聞き方はシンプルに『判断の流れをフローチャートにまとめて。これを見れば迷わない、ってレベルで』。たったこれだけで、AIは他の問題にも使える汎用フローチャートを作ってくれる。」

### Geminiの返答を待つ → 返答を読み上げながら解説
「これが返ってきた。（フローチャートを読み上げ）
①まず式の項数を数える。
②2項の場合 → a²−b² の形か確認 → 和と差の積。
③3項の場合 → 最初と最後が2乗か確認 → 真ん中が 2ab か確認 → 完全平方式。
④それ以外 → 共通因数をくくり出して再チャレンジ。
これが『型』だ。この型をNotebookLMにストックすれば、次に因数分解で迷ったとき、いつでも引き出せる。」

---

## シーン5: 締め

### ナレーション
「今の流れを振り返ろう。まず自分で仮説を立てて、解法検証で方針を確かめた。次に構造発見で2つの式の構造の違いを見抜いた。最後に型抽出で判断フローチャートを作った。このサイクルを1回やるだけで、暗記じゃなくて『構造の理解』で因数分解が攻略できるようになった。次の章では、この型をNotebookLMに蓄積して、いつでも引き出せる仕組みの作り方を見ていく。」
