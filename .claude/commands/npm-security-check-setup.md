# npm セキュリティチェック スキル セットアップガイド

## これは何？

Claude Code で `/npm-security-check` と入力するだけで、npmサプライチェーン攻撃の感染チェックを自動実行してくれるスキルです。

チェック内容：
- 汚染パッケージの混入確認
- 汚染版 axios の痕跡確認
- npm の脆弱性スキャン
- マルウェア痕跡の確認
- 不審な通信（攻撃者サーバーへの接続）の確認
- Claude Code のインストール方法の確認

Mac / Windows どちらでも使えます。

---

## セットアップ手順（3ステップ）

### ステップ 1: ファイルを受け取る

`npm-security-check.md` というファイルを受け取ってください（このガイドとは別のファイルです）。

### ステップ 2: ファイルを置く

受け取ったファイルを、以下のフォルダに保存してください。

**Windows の場合：**
```
C:\Users\（自分のユーザー名）\.claude\commands\
```

**Mac の場合：**
```
~/.claude/commands/
```

> `.claude` フォルダが見つからない場合は、隠しフォルダになっています。
>
> - **Windows**: エクスプローラーの「表示」→「隠しファイル」にチェックを入れる
> - **Mac**: Finder で `Cmd + Shift + .`（ドット）を押す
>
> `commands` フォルダがない場合は、新しく作成してください。

### ステップ 3: 使う

Claude Code を起動して、プロンプトに以下を入力するだけです。

```
/npm-security-check
```

あとは自動的にすべてのチェックが実行され、結果が表示されます。

---

## 実行結果の見方

チェック完了後、以下のような結果表が表示されます。

| 結果の表示 | 意味 |
|-----------|------|
| ✅ 安全 | 問題なし |
| ⚠️ 検出 | 問題が見つかった（指示に従って対応） |
| ⏭️ スキップ | 該当ファイルがないためチェック省略（問題なし） |

**⚠️ が表示された場合は、画面の指示に従ってください。**

## よくある質問

**Q: Claude Code を起動していればどのフォルダでも使える？**
A: はい。ただし、npm関連のチェック（チェック1〜3）はnpmプロジェクトのフォルダで実行した方がより正確です。

**Q: 何か壊れたりしない？**
A: このスキルはファイルの読み取りとコマンドの実行のみで、ファイルの変更や削除は一切行いません。安心してお使いください。

**Q: 「win32」と表示されるけど大丈夫？**
A: Windows 10 / 11 でも「win32」と表示されるのは正常です。プログラムの慣例で、Windows を示す識別子です。
